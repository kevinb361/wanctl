## Remaining Code-Adjacent Drift Not Changed

These were identified but not changed because the request was documentation-only:

| Item | Evidence | Why not changed |
| --- | --- | --- |
| `scripts/install.sh` embeds `VERSION="1.37.0"` while `pyproject.toml` is `1.39.0` | Installer script metadata | It is source/script behavior, not documentation. |
| `configs/spectrum.yaml` and `configs/att.yaml` contain `irtt.duration_ms` and `irtt.packet_size`, which current config parsing does not consume | Current docs already document `duration_sec`, `interval_ms`, and fixed payload size | Editing checked-in configs would be a config change, not documentation. |
| `configs/spectrum.yaml` has an alerting rule without `severity` | Config loader requires severity for alert rules | Editing checked-in configs would be a config change, not documentation. |
| `configs/steering.yaml` uses legacy threshold names not read by the daemon loader | Schema docs already describe current steering threshold names | Editing checked-in configs would be a config change, not documentation. |
