If absent, you must ADD them under the thresholds: section.

## Test Methodology (per D-03, D-06, D-07)

1. Set value A in YAML, SIGUSR1 reload, wait 5s for config to take effect
2. Run `flent rrul_be -H 104.200.21.31 -l 60`, note time of day
3. Record: ICMP median, p99, max latency; DL throughput avg; UL throughput avg; state distribution from health endpoint
4. Set value B, SIGUSR1 reload, wait 5s
5. Run same flent command, note time of day
6. Compare metrics. If delta < 5%, re-run both (per D-03)
7. Apply winner, move to next parameter

## Metrics Extraction from flent

After each run, flent produces a summary. Key metrics:
- `avg`, `median`, `p99`, `max` from the ICMP ping flow
- `avg` from TCP download flows
- `avg` from TCP upload flows

Also check health endpoint for state distribution:
```bash
curl -s http://127.0.0.1:9101/health | python3 -m json.tool
```
Look at download.state_distribution for GREEN/YELLOW/SOFT_RED/RED percentages.
