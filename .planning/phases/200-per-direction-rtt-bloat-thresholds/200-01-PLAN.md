<objective>
Replace the working-tree's value-derived `_upload_thresholds_explicit` flag with two per-key presence-based flags (`_upload_target_bloat_ms_explicit`, `_upload_warn_bloat_ms_explicit`) and gate `target_delta` / `warn_delta` writes in `_apply_threshold_param` independently. Also: register the two new UL leaf paths in `KNOWN_AUTORATE_PATHS` (Plan 03's silent-pass invariant depends on this), and emit a one-shot INFO log at WANController construction when either explicit flag is True so Plan 06's deploy verification can confirm the new thresholds are live without adding a new /health field (D-06).

Purpose: The current value-derived form silently fails in two cases — (1) operator coincidentally sets a UL key equal to the DL global value (flag becomes False, live-tuner overwrites the explicit operator intent), and (2) operator sets only one of the two UL keys (the other field is unprotected by design, but the current single flag protects neither or both). Per-key presence-based flags fix both. The validator paths and log emission close blocker gaps for Plans 03 and 06.

Output: Updated `autorate_config.py` (records two flags at `ul.get(...)` time), updated `wan_controller.py` (reads two flags via getattr; gates each write independently; emits one-shot INFO log on explicit thresholds), updated `check_config_validators.py` (KNOWN_AUTORATE_PATHS includes the two UL leaf paths), and 2 new tests in `tests/test_wan_controller.py`.
</objective>
