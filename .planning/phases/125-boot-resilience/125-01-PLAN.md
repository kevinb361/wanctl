Create `deploy/scripts/wanctl-nic-tuning.sh` (new file, deployed to `/usr/local/bin/wanctl-nic-tuning.sh` on target). Per D-02, follow the `wanctl-recovery.sh` pattern: `logger -t wanctl-nic-tuning` for all journal output.

Script structure:
1. Shebang: `#!/bin/bash`
2. Header comment: purpose, idempotency note, deployed path
3. `set -euo pipefail` (but individual NIC failures handled with `||` so set -e doesn't abort)
4. Constants:
   - `TAG="wanctl-nic-tuning"` (for logger -t)
   - `SPECTRUM_NICS=(ens16 ens17)` -- Spectrum bridge pair (i210)
   - `ATT_NICS=(ens27 ens28)` -- ATT bridge pair (i350)
   - `ALL_NICS=("${SPECTRUM_NICS[@]}" "${ATT_NICS[@]}")`
   - `SPECTRUM_CPU=0` -- IRQ affinity target for Spectrum NICs
   - `ATT_CPU=1` -- IRQ affinity target for ATT NICs
   - `RING_RX=4096`
   - `RING_TX=4096`
5. Helper function `tune_nic()` taking NIC name as argument:
   - Check NIC exists: `if ! ip link show "$nic" &>/dev/null; then logger -t "$TAG" "WARNING: $nic not found, skipping"; return 0; fi`
   - Ring buffers: `ethtool -G "$nic" rx "$RING_RX" tx "$RING_TX" 2>&1 || logger -t "$TAG" "WARNING: ring buffer set failed for $nic (may already be at max)"`
   - rx-udp-gro-forwarding: `ethtool -K "$nic" rx-udp-gro-forwarding on 2>&1 || logger -t "$TAG" "WARNING: rx-udp-gro-forwarding failed for $nic"`
   - Log success: `logger -t "$TAG" "$nic: ring buffers rx=$RING_RX tx=$RING_TX, rx-udp-gro-forwarding=on"`
6. Function `set_irq_affinity()` taking NIC name and CPU number:
   - Find IRQs: `irqs=$(grep "$nic" /proc/interrupts | awk '{print $1}' | tr -d ':')`
   - If no IRQs found, log warning and return 0
   - For each IRQ: `echo "$cpu" > /proc/irq/$irq/smp_affinity_list 2>/dev/null || logger -t "$TAG" "WARNING: IRQ $irq affinity failed for $nic"`
   - Log: `logger -t "$TAG" "$nic: IRQ affinity pinned to CPU $cpu"`
7. Main body:
   - `logger -t "$TAG" "Starting NIC tuning..."`
   - `errors=0`
   - Loop over ALL_NICS calling tune_nic; increment errors on failure
   - Loop over SPECTRUM_NICS calling `set_irq_affinity "$nic" "$SPECTRUM_CPU"`
   - Loop over ATT_NICS calling `set_irq_affinity "$nic" "$ATT_CPU"`
   - Final summary: `logger -t "$TAG" "NIC tuning complete (${#ALL_NICS[@]} NICs configured, $errors errors)"`
   - Exit 0 always (per D-01: warn but don't block boot chain)

Make the script executable: chmod +x.

IMPORTANT: The script must ALWAYS exit 0. Missing NICs or failed ethtool commands log warnings but never cause a non-zero exit. This ensures the boot chain proceeds even with degraded NIC config (per D-01 decision to prefer availability).
