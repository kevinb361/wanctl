<objective>
Build the time-aligned multi-source joiner that makes "bad p99 while GREEN" trivially visible at row N+18 of a per-second table. Given a `.flent.gz`, a 1Hz `/health` NDJSON file, a pre-pulled journal NDJSON, and an optional alerts list, the aligner emits one row per integer second spanning `[flent_t0 - pre_buf, flent_end + post_buf]`. Each row carries the flent pings that landed in that second (bucketed by `int(p['t'])` from `raw_values`), the matching `/health` row's projected fields, and any journal events for that second (±1s tolerance per Pitfall 4 clock-skew).

Purpose: MEAS-01 requires controller-state correlation; the aligned table is the single artifact every downstream classifier (214-04) reads.
Output: `scripts/phase214-align.py` module/CLI + tests + a synthesized health NDJSON fixture that drives the bucketing test and feeds the collapse-while-GREEN scenarios used by 214-04.
</objective>

class AlignmentError(RuntimeError):
    """Raised when required inputs are absent or unreadable."""

def _read_ndjson(path: Path) -> list[dict]:
    """One JSON object per non-blank line; missing file → []."""

def align_window(
    flent_path: Path,
    health_ndjson: Path,
    journal_lines: list[dict],   # pre-read journalctl --output=json
    alerts_window: list[dict],   # pre-pulled alerts rows; may be []
    flent_t0_unix: float,
    flent_end_unix: float,
    pre_buf_sec: int = 10,
    post_buf_sec: int = 10,
) -> list[dict]:
    """Return aligned rows, one per integer second across
       [int(flent_t0_unix) - pre_buf_sec, int(flent_end_unix) + post_buf_sec].
    """

# Journal-row time key: 'ts' (epoch float); bucket by int(ts) with ±1s tolerance.

#       --health-ndjson <path> \
#       --journal-ndjson <path>  (optional; default to []) \
