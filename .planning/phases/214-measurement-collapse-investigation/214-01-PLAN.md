<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| operator → wrapper CLI | Untrusted positional/flag input; risk of shell injection if interpolated into `bash -c` or `eval`. |
| wrapper → phase213-baseline-capture.sh | Trusted local subprocess; arguments are constants except `$DURATION` (numeric). |
| wrapper → `git diff` | Local read-only git invocation; output not interpreted. |
| wrapper → `jq -n` (sidecar write) | Local write to `evidence/RUN-<ts>/`; outside `src/wanctl/`. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-214-01-01 | Tampering | `src/wanctl/` modification during live run | mitigate | D-14 hard guard: `git diff --quiet "${PHASE214_BASE_SHA}..HEAD" -- src/wanctl/` returns 0 or wrapper exits 4 before delegating. |
| T-214-01-02 | Tampering | Shell injection via positional args | mitigate | Strict `case "$WINDOW" in off-peak|daytime|prime-time) ... *) usage; exit 64;;` — no interpolation of `$WINDOW` into shell commands except inside the validated `case` block. Numeric `$DURATION` is passed as a discrete argv element, never interpolated into a string. |
| T-214-01-03 | Elevation of Privilege | Window-hour bypass to fire an out-of-budget run | mitigate | `--test-hour` is only honored when `--dry-run` is also set (exit 7 otherwise), mirroring phase198:111-118. Production runs read live `date +%H`. |
| T-214-01-04 | Spoofing | Operator misreads dry-run as a real run and consumes the off-peak window | mitigate | `--dry-run` prefixes the planned command with literal `DRY-RUN:` and skips invocation; usage block describes the difference explicitly. |
| T-214-01-05 | Information Disclosure | Sidecar manifest leaks credentials | accept | Sidecar fields are static (`phase`, `window`, `flent_duration_sec`, timestamps, `git_head_sha`, `mutation_posture`); no env vars or secrets are emitted. Steering redaction is owned by `phase213-steering-snapshot.sh` (D-08, already enforced by the delegated orchestrator). |
| T-214-01-06 | Denial of Service | Failed orchestrator leaves stale `RUN-*` dir and burns the operator's window budget | accept | Phase 213 orchestrator's existing `trap` cleanup handles poller PIDs; partial RUN dirs are diagnostic, not destructive. Operator can re-run the window per D-02 (one retry allowed for invalid artifacts/netperf failure). |
</threat_model>
