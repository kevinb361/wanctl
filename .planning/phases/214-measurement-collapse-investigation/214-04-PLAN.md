def classify(aligned_rows: list[dict]) -> dict:
    """Apply all six drivers; return {primary, ranked, drivers, evidence_summary}.

    Returns:
      {
        "primary_driver": str | None,    # "reflector_loss" | "icmp_udp_divergence" | "stale_cached_rtt"
                                          #  | "steering_behavior" | "cake_queue_mismatch" | "external_path"
                                          #  | None
        "ranked": list[str],              # ordered by evidence_strength_score desc
        "drivers": {
            "<name>": {"fired": bool, "evidence": str, "score": int, "first_unix": int|None}
        },
      }

    primary_driver contract (revision 2026-05-27 — resolves checker Warning #4):
      - If aligned_rows (filtered to in_flent_window) is EMPTY → primary_driver = None
        (degenerate: no evidence to classify).
      - If aligned_rows is non-empty AND ≥1 driver fired → primary_driver = ranked[0]
        (highest-score driver among 1..5; lexicographic tiebreak).
      - If aligned_rows is non-empty AND NO driver fired → primary_driver = "external_path"
        (catch-all per Pattern 4 driver 6).
      The `None` and `external_path` cases are mutually exclusive — `None` means "no rows
      to evaluate", `external_path` means "evaluated rows, no signal driver matched".
    """

def verdict_for_window(latency: dict, drivers: dict) -> str:
    """Returns 'pass', 'fail', or 'ambiguous' per D-06 boundaries.

    Pass: latency.p99_ms < 500 AND drivers['reflector_loss']['fired'] is False
                                AND drivers['icmp_udp_divergence']['fired'] is False
    Fail: latency.p99_ms > 1000 AND any(d['fired'] for d in drivers.values())
    Otherwise: 'ambiguous'

    PASS SEMANTICS (Codex MED-6 — explicit contract, locked 2026-05-28):
      A `pass` verdict requires p99 BELOW the floor (default 500ms per D-06).
      Non-p99 quality signals (stale cycles, fusion suspension, cake queue
      mismatch, steering churn, external path) do NOT block pass when p99 is
      healthy — they are informational only in the pass case. The two drivers
      that DO block pass (reflector_loss, icmp_udp_divergence) are explicitly
      enumerated above and tested by `test_verdict_pass_with_subordinate_drivers_firing`.
      Rationale: D-06 defines the user-experience floor (p99 < 500ms = acceptable
      latency tail); the subordinate drivers describe controller-internal
      observability and would have prevented the bad UX if they had been
      problematic, so their firing while p99 is low is evidence the controller
      is working correctly, not evidence of a problem. Reflector_loss and
      icmp_udp_divergence are NOT subordinate — they directly undermine the
      `/health` trustworthiness that is the entire question of this phase, so
      they block pass even at low p99 (because that low p99 might itself be
      stale/cached and the measurement collapsed)."""
