<title>Write the read-only 3-run baseline capture wrapper</title>

<action>
Create `scripts/phase226-baseline-capture.sh` modeled on the conventions in
`scripts/phase198-rerun-flent-3run.sh` and `scripts/phase213-baseline-capture.sh`:
`set -euo pipefail`; `require_command` for ssh, curl, python3, sha256sum, flent (or the project's
flent helper); args `--output-dir <dir>` (required; refuse non-empty), `--ssh-host` (default
`cake-shaper`), `--health-url` (default `http://10.10.110.223:9101/health` per spectrum.yaml
health_check), `--local-bind` (default `10.10.110.226` Spectrum source bind, matching phase198),
`--runs` (default 3, D-08), `--duration` (default 60, D-08), `--router-iface` (default `spec-router`),
`--modem-iface` (default `spec-modem`), and `--dry-run` (run gates/prereqs only — no flent, no SSH
mutation-free reads still allowed but skip evidence writes). All target-host access is read-only:
`ssh -o BatchMode=yes "$SSH_HOST" "sudo -n <read-cmd>"`. Create `baseline-<UTC>/` (UTC stamp
`date -u +%Y%m%dT%H%M%SZ`) under `--output-dir` and, for each of `--runs` runs, capture into a
`run-NN/` subdir:
- `tc-qdisc-spec-router.before.txt` / `.during.txt` / `.after.txt` ← `sudo -n tc -s qdisc show dev <router-iface>` sampled at pre-load (T0), mid-load (during the flent window), and post-load (T2).
- `tc-qdisc-spec-modem.before.txt` / `.during.txt` / `.after.txt` ← same for `<modem-iface>`.
- `health.before.json` / `.during.json` / `.after.json` ← `curl -fsS --max-time 5 <health-url>` (Spectrum `/health` + state, D-09 cadence per AB-02 pre/during/post).
- `flent-rrul.<run>.{flent.gz,txt}` ← flent RRUL (bidirectional saturating + latency probe) bound to `--local-bind`, satisfying AB-02; reuse the phase198 flent invocation/host defaults.
- `ref-udp-unmarked.<run>.txt` and `ref-tcp-bulk-unmarked.<run>.txt` ← unmarked-UDP and unmarked-bulk-TCP reference flows alongside RRUL (D-07) — these pre-stage the Phase 227 marked-EF-vs-unmarked comparison; capture their latency/throughput summary, not just raw pcaps.
Honor an off-peak/window gate consistent with phase198 (`--force-window <reason>` / `--allow-extended-window` escape hatches) so the capture does not run during peak production load. Write `MANIFEST.md` (Captured UTC, Source Posture = read-only/no-mutation, runs, duration, local-bind, the exact `configs/spectrum.yaml` baseline state line `920/18 besteffort wash` with `diffserv`/`allow_wash`/ceilings/`docsis_mode` echoed from the live config, Artifacts list with sha256 of each file) following the phase224 MANIFEST style. End with a required-artifact assertion loop that exits non-zero on any missing/empty expected file. The script MUST NOT contain any `tc ... add/del/replace`, any nft `add/delete/flush/-f`, any CAKE-mode write (`diffserv`/`wash` mutation), any `/etc/wanctl` write, any service restart/deploy, or any reference to controller-path source files (`wan_controller`, `queue_controller`, `cake_signal`, `alert_engine`, fusion).
</action>
