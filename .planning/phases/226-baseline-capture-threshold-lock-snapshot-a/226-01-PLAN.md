- `health.window.ndjson` ← a CONTINUOUS health stream: a backgrounded poller loop that issues `curl -fsS --max-time 3 <health-url>` every `--health-interval` seconds (default 1) for the full pre→during→post window, appending one JSON object per line with a captured UTC timestamp (review HIGH#4). This windowed NDJSON — NOT 3 isolated point samples — is what lets Task 2 compute restart-rate (D-02), transition-rate (D-03), and floor-hit-cycle / SOFT_RED-dwell (D-04) deltas. - `flent-rrul.<run>.{flent.gz,txt}` ← flent RRUL (bidirectional saturating + latency probe) bound to `--local-bind`, satisfying AB-02; reuse the phase198 flent invocation/host defaults.
- `ref-udp-unmarked.<run>.txt` and `ref-tcp-bulk-unmarked.<run>.txt` ← unmarked-UDP (`iperf3 -u` at a declared `--ref-udp-rate`) and unmarked-bulk-TCP (`iperf3`) reference flows run CONCURRENTLY with the RRUL saturating window against a declared `--ref-host`/`--ref-port` (review MEDIUM — explicit schedule). Start them after RRUL ramp-up so they share the saturated window. Assert DSCP neutrality: the ref flows are emitted with DSCP/TOS 0 (no `--tos`/`--dscp` set) and the capture records the observed egress DSCP to prove they are unmarked. Write `MANIFEST.md` (Captured UTC, Source Posture = read-only/no-mutation, runs, duration, local-bind, the exact `configs/spectrum.yaml` baseline state line `920/18 besteffort wash` with `diffserv`/`allow_wash`/ceilings/`docsis_mode` echoed from the live config, Artifacts list with sha256 of each file) following the phase224 MANIFEST style.
