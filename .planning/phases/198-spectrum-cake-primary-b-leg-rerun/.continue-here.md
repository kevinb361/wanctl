<decisions_made>

- **Drop `--remote-user root` from harness default.** Harness wraps remote sqlite3 in `sudo -n`; root SSH was gratuitous and broke cron-context (no agent).
- **Capture `ATTEMPTED_AT_UTC` once at script start.** Was previously written at JSON-write time (mislabeled "attempted at"); ERR-trap path dropped it entirely.
- **Upsert semantics for ATTEMPT-LOG (not append-once).** Plan 198-06 MEDIUM-8 requires idempotent updates so re-running the helper after `mark_retry` mutates the JSON refreshes the rendered entry.
- **Strict input-derived attempt numbering.** Refuse on JSON/dirname disagreement rather than silently picking one ‚ÄĒ `max(existing)+1` misnumbers out-of-order backfills.
- **Three-source timestamp resolution.** Harness historically wrote `attempted_at_utc`; helper falls back to `start_utc` alias and to harness log filename pattern (`scheduled-attempt-YYYYMMDDTHHMMSSZ-runN.log`); 'unknown' as last resort.
- **Persistent systemd unit files (not transient).** `~/.config/systemd/user/` survives reboot; `/run/user/1000/systemd/transient/` does not.
- **Timer `Persistent=false`.** If dev is off at 02:30 and boots at 09:00, late firing would bash production WAN during business traffic ‚ÄĒ better to miss than to misfire.
- **No remote `/schedule` agent for tomorrow's check.** `.planning/` is gitignored so a cloud agent can't read ATTEMPT-LOG. Local `/gsd-progress` is the right tool.
