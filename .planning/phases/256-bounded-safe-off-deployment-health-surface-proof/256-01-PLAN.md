# Plan 256-01 — Operator-Gated Safe/Off Deploy/Restart + Rollback Proof

<objective>
Resolve the Phase 255 blocker only with explicit operator approval: create rollback anchors for the flat `/opt/wanctl` deployment and `/etc/wanctl/steering.yaml`, deploy route-management-capable steering code/config in safe/off or dry-run mode, restart only `steering.service`, and rollback on health failure. If approval is denied or missing, stop with a blocked/no-mutation outcome.
</objective>

<must_haves>
<truths>
- CHECKPOINT REQUIRED before any privileged read, backup write, config edit, code deploy, or service restart.
- Netwatch remains active/interim route owner.
- Phase 256 target config must be `mode: "off"` or `mode: "dry_run"`; `mode: "active"` is forbidden.
- Only `steering.service` may be restarted, and only after approval.
- No RouterOS route mutation. No Netwatch disablement. No CAKE/qdisc change. No route-owner flip.
</truths>
<outputs>
- Approval record.
- Rollback-anchor evidence for `/opt/wanctl` and `/etc/wanctl/steering.yaml`, or blocked/no-approval artifact.
- Deploy/restart transcript with rollback result if approved.
</outputs>
</must_haves>

<threat_model>
T-256-01 unauthorized live mutation: backups/config edits/restart happen without explicit approval. Mitigation: approval record with `APPROVED_PHASE256_DEPLOY: true` is mandatory before mutation; otherwise stop.

T-256-02 unrecoverable flat deploy overwrite: `/opt/wanctl` is overwritten without a restore source. Mitigation: create backup/manifest first and verify restore path before deploy.

T-256-03 route-owner safety breach: config accidentally enables active route mutation or disables Netwatch. Mitigation: target config validator, forbidden token scan, and explicit no-RouterOS/Netwatch/CAKE commands.
</threat_model>
