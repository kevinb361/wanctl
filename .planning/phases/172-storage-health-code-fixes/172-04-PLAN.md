<objective>
Fix the health endpoint contract gap: add a top-level `storage` field to the autorate
health response so the canary script can read storage file sizes.

Purpose: The canary-check.sh script reads `.storage.files.db_bytes` from the top-level
health JSON, but the autorate endpoint only places storage data under `wans[0].storage`.
The steering endpoint already has a top-level `storage` field. This gap makes the canary
report `storage status=unknown, db=0B` on production even though per-WAN storage data
exists.

Output: Fixed health_check.py with top-level storage hoisting + regression test.
</objective>
