The script must:

1. **Header and safety:**
   - `#!/bin/bash` with `set -euo pipefail`
   - Usage comment block: `./scripts/migrate-storage.sh [--ssh user@host]`
   - Support `--ssh user@host` for remote execution (same pattern as canary-check.sh)
   - Support `--dry-run` to show what would happen without making changes
   - Color output matching deploy.sh conventions (RED, GREEN, YELLOW, BLUE, NC)

2. **Pre-flight checks (run on target):**
   ```bash
   STATE_DIR="/var/lib/wanctl"
   LEGACY_DB="$STATE_DIR/metrics.db"
   ARCHIVE_NAME="$STATE_DIR/metrics.db.pre-v135-archive"
   SPECTRUM_DB="$STATE_DIR/metrics-spectrum.db"
   ATT_DB="$STATE_DIR/metrics-att.db"
   ```
   - Check if legacy DB exists. If not, print "Legacy DB not found, nothing to migrate" and exit 0 (idempotent).
   - Check if archive already exists. If so, print "Archive already exists, migration already completed" and exit 0 (idempotent).
   - Print legacy DB size in human-readable format (e.g., "Legacy DB: 925 MB").

3. **Stop services before migration:**
   ```bash
   echo "Stopping wanctl services..."
   sudo systemctl stop wanctl@spectrum.service wanctl@att.service 2>/dev/null || true
   # Wait for processes to exit
   sleep 2
   ```

4. **Retention purge on legacy DB (D-03 prerequisite):**
   Run a retention purge before VACUUM to delete old rows first, making VACUUM effective:
   ```bash
   echo "Purging data older than 24h from legacy DB..."
   CUTOFF=$(date -d '24 hours ago' +%s)
   sudo sqlite3 "$LEGACY_DB" "DELETE FROM metrics WHERE timestamp < $CUTOFF;"
   sudo sqlite3 "$LEGACY_DB" "DELETE FROM downsampled_metrics WHERE timestamp < $CUTOFF;"
   ROWS_DELETED=$(sudo sqlite3 "$LEGACY_DB" "SELECT changes();")
   echo "Deleted rows from downsampled_metrics: $ROWS_DELETED"
   ```

5. **VACUUM the legacy DB (D-03):**
   ```bash
   echo "Running VACUUM on legacy DB (this may take a moment)..."
   PRE_SIZE=$(sudo stat -c%s "$LEGACY_DB" 2>/dev/null || echo 0)
   sudo sqlite3 "$LEGACY_DB" "VACUUM;"
   POST_SIZE=$(sudo stat -c%s "$LEGACY_DB" 2>/dev/null || echo 0)
   SAVED=$(( (PRE_SIZE - POST_SIZE) / 1024 / 1024 ))
   echo "VACUUM complete: ${PRE_SIZE} -> ${POST_SIZE} bytes (saved ${SAVED} MB)"
   ```

6. **Archive the legacy DB (D-07):**
   ```bash
   echo "Archiving legacy DB..."
   sudo mv "$LEGACY_DB" "$ARCHIVE_NAME"
   # Also move WAL and SHM if they exist
   [ -f "${LEGACY_DB}-wal" ] && sudo mv "${LEGACY_DB}-wal" "${ARCHIVE_NAME}-wal"
   [ -f "${LEGACY_DB}-shm" ] && sudo mv "${LEGACY_DB}-shm" "${ARCHIVE_NAME}-shm"
   echo "Archived to: $ARCHIVE_NAME"
   ```

7. **Verify post-migration state:**
   ```bash
   echo ""
   echo "=== Post-Migration Verification ==="
   echo "Legacy DB: $([ -f "$LEGACY_DB" ] && echo 'STILL EXISTS (ERROR)' || echo 'Archived (OK)')"
   echo "Archive: $([ -f "$ARCHIVE_NAME" ] && echo "$(sudo stat -c%s "$ARCHIVE_NAME" | numfmt --to=iec) (OK)" || echo 'Missing (ERROR)')"
   echo "Spectrum DB: $([ -f "$SPECTRUM_DB" ] && echo "$(sudo stat -c%s "$SPECTRUM_DB" | numfmt --to=iec)" || echo 'Not yet created (will be created on service start)')"
   echo "ATT DB: $([ -f "$ATT_DB" ] && echo "$(sudo stat -c%s "$ATT_DB" | numfmt --to=iec)" || echo 'Not yet created (will be created on service start)')"
   echo ""
   echo "Next steps:"
   echo "  1. Start services: sudo systemctl start wanctl@spectrum wanctl@att"
   echo "  2. Wait 30s for initial data, then run: ./scripts/canary-check.sh --ssh user@host"
   echo "  3. Verify storage.status is 'ok' in canary output"
   ```

8. **SSH wrapper:**
   When `--ssh user@host` is provided, the script should SSH to the target and execute the migration commands remotely. Use a heredoc pattern:
   ```bash
   if [ -n "$SSH_TARGET" ]; then
       ssh "$SSH_TARGET" 'bash -s' <<'REMOTE_SCRIPT'
       # ... all the migration commands above ...
   REMOTE_SCRIPT
   else
       # ... run locally ...
   fi
   ```

   For simplicity, implement as two functions: `run_migration()` containing all the migration logic, and the main block that either runs locally or pipes to SSH.
