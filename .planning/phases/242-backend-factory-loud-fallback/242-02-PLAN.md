<objective>
Build the factory module — the only genuinely new code in the phase. It selects
icmplib vs fping from measurement.backend, falls back to icmplib LOUDLY at
construction time when fping is selected but absent, and returns an RttBackendHandle
(NOT a raw thread). The handle exposes a DEFERRED make_thread() builder that the
autorate caller invokes later (when the pool + hosts_fn exist) and whose output
satisfies the COMMON THREAD PROTOCOL WANController.measure_rtt() requires. For the
fping path, make_thread builds an in-module ADAPTER wrapping FpingThread so a raw
FpingThread is never assigned to _rtt_thread. This plan makes the Plan 01 factory
unit tests — including the thread-protocol and measure_rtt-no-AttributeError
regressions — GREEN.

Purpose: Centralize backend construction in one place with zero call-site branching
(D-04), make fallback observable per-WAN (FALL-02), and CLOSE the review's
load-bearing HIGH: the factory hands the call sites a protocol-satisfying thread
handle/adapter, not a raw FpingThread that would AttributeError in measure_rtt.
Output: src/wanctl/rtt_backend_factory.py with build_rtt_backend(), RttBackendHandle,
the fping adapter, the deferred make_thread builder, and the WARN-once + per-WAN
counter helpers; no call-site edits in this plan.

This plan creates a NEW module only — it does not edit any existing src/wanctl file,
so it does NOT touch the controller path beyond adding one allowlisted file. The
fping adapter lives HERE (inside rtt_backend_factory.py, already allowlisted), NOT
in the 241-frozen fping_measurement.py.
</objective>

  class RttBackendHandle:
      backend: <RttBackend Protocol object — RTTMeasurement | FpingMeasurement>

      and `if not (cycle_status and self._should_skip_scorer_update(cycle_status)):` — so
      returning None is SAFE and tolerated. (wan_controller.py:1152-1154,1171)
  get_profile_stats() -> dict
