   - Threshold defaults: `STALL_MAX_COUNT_PER_ARM = 2`, `STALL_MAX_RATE_PCT = 0.01`.

   - Rationale: fixed production arms showed isolated 1-2 stalls over >=36k cycles; 2/36k = ~0.0056%, below the 0.01% rate. Sustained or bursty cadence failure still blocks.

   - Replace `max_tasks <= first_icmplib_tasks + TASKS_BOUND` with backend-delta logic.
   - Threshold default: `TASKS_BACKEND_DELTA_BOUND = 1`.
   - Pass condition: `max_tasks(fping_hygiene) <= max_tasks(icmplib_hygiene) + TASKS_BACKEND_DELTA_BOUND` for the same WAN/load pair.
   - Optional extra check if cheap: `max_tasks(fping_hygiene) - min_tasks(fping_hygiene) <= TASKS_BACKEND_DELTA_BOUND` to catch fping task growth within the arm; do not fail solely because the icmplib control arm naturally has more tasks.
   - Rationale: fixed evidence shows fping max=5 and icmplib max=7-8, so the old gate penalized control-arm baseline behavior, not fping task growth.
