# Plan 257-01 â€” Dry-Run Observation + Canary Readiness Decision Packet

<objective>
Run a bounded read-only/dry-run observation from `cake-shaper`, compare wanctl route-management intent and health state against live Netwatch/default-route state, and produce one final readiness packet with verdict `ready-for-approval` or `not-ready`. Preserve SAFE-20: no active canary, no RouterOS route mutation, no Netwatch retirement/disablement, no CAKE/qdisc/threshold change, and no route-owner flip.
</objective>

<must_haves>
<truths>
- D-257-01: Use a bounded 10-15 minute dry-run observation window; a single snapshot is too weak and a 30-60 minute soak is out of scope.
- D-257-01: Every live command must come from a timestamped deterministic command file validated against an explicit read-only allowlist before execution.
- D-257-02: RouterOS REST currently fails closed for `/tool netwatch print detail`; if supported read-only ownership inspection cannot be proven, the verdict must be `not-ready`.
- D-257-03: `ready-for-approval` requires guard status ok/supported, reconciliation ok, circuit breaker closed, no intended/applied mutation, matching live route/Netwatch inventory, current rollback anchors, healthy bridge/state services, and clean SAFE-20 proof.
- D-257-03: Mixed or incomplete evidence resolves to `not-ready` with concrete blockers.
- Netwatch remains the active/interim route owner for all of v1.56.
- This plan produces a packet only; it must not request, imply, or chain into active canary approval.
</truths>
<outputs>
- `evidence/phase257-readonly-commands-<timestamp>.txt`
- `evidence/phase257-readonly-validator-<timestamp>.py`
- `evidence/phase257-observation-transcript-<timestamp>.md`
- `evidence/phase257-readiness-packet-<timestamp>.md`
- `257-01-SUMMARY.md`
</outputs>
</must_haves>

<threat_model>
T-257-01 live mutation through inspection channel: an SSH/RouterOS command mutates routes, Netwatch, scripts, CAKE/qdisc, or service config. Mitigation: write exact commands first, validate only `COMMAND:` lines against a tight allowlist, reject shell metacharacters and mutating RouterOS actions, and stop if any needed command is outside the allowlist.

T-257-02 false readiness from unsupported guard: route-management health looks otherwise healthy while ownership guard still fails closed because Netwatch cannot be inspected. Mitigation: final packet must require supported ownership inspection for `ready-for-approval`; unsupported guard forces `not-ready`.

T-257-03 endpoint confusion: bridge health on `10.10.110.223:9101` / `10.10.110.227:9101` is mistaken for route-management acceptance. Mitigation: accept route-management only from `127.0.0.1:9102/health` on `cake-shaper` and keep bridge health in a separate evidence section.

T-257-04 active canary creep: a readiness packet is treated as approval to flip route ownership. Mitigation: packet must state `APPROVED_ACTIVE_CANARY: false`, `NETWATCH_REMAINS_OWNER: true`, and `NO_ROUTE_OWNER_FLIP: true` regardless of verdict.
</threat_model>
