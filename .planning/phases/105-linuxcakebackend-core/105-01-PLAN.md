<objective>
Create LinuxCakeBackend: a complete RouterBackend implementation that controls CAKE qdiscs
via local tc subprocess commands. Handles bandwidth changes (tc qdisc change), statistics
collection with per-tin parsing (tc -j -s qdisc show), CAKE initialization (tc qdisc replace),
and parameter validation (readback verification). All operations are local subprocess calls
with zero new Python dependencies.

Purpose: This backend enables wanctl to control CAKE qdiscs on a Linux bridge VM instead of
MikroTik's built-in queues, unlocking full CAKE feature support and per-tin statistics.

Output: linux_cake.py (production backend) + test_linux_cake_backend.py (full test coverage)

```python
class RouterBackend(ABC):
    def __init__(self, logger: logging.Logger | None = None):
        self.logger = logger or logging.getLogger(__name__)

    @abstractmethod
    def set_bandwidth(self, queue: str, rate_bps: int) -> bool: ...
    @abstractmethod
    def get_bandwidth(self, queue: str) -> int | None: ...
    @abstractmethod
    def get_queue_stats(self, queue: str) -> dict | None: ...
    @abstractmethod
    def enable_rule(self, comment: str) -> bool: ...
    @abstractmethod
    def disable_rule(self, comment: str) -> bool: ...
    @abstractmethod
    def is_rule_enabled(self, comment: str) -> bool | None: ...
    def reset_queue_counters(self, queue: str) -> bool: return True
    def test_connection(self) -> bool: return True
    @classmethod
    def from_config(cls, config: Any) -> "RouterBackend": raise NotImplementedError
```

    **Internal helper** `_run_tc(self, args: list[str], timeout: float | None = None) -> tuple[int, str, str]`:
    - Builds `cmd = ["tc"] + args`
    - Calls `subprocess.run(cmd, capture_output=True, text=True, timeout=timeout or self.tc_timeout)` with `shell=False`
    - Returns `(result.returncode, result.stdout, result.stderr)`
    - Catches `subprocess.TimeoutExpired` -> returns `(-1, "", "timeout")`
    - Catches `FileNotFoundError` -> returns `(-1, "", "tc not found")`
    - Add `# noqa: S603` on subprocess.run line (per Pitfall 6, matching irtt_measurement.py pattern)

    **Internal helper** `_find_cake_entry(self, json_output: str) -> dict | None`:
    - `json.loads(json_output)` -> iterate list -> find `entry.get("kind") == "cake"` -> return entry
    - Returns None on JSONDecodeError, empty/invalid data, or no CAKE found

    **ABC methods:**

    `set_bandwidth(self, queue: str, rate_bps: int) -> bool` (BACK-01, per D-08):
    - `rate_kbit = rate_bps // 1000`
    - Runs `_run_tc(["qdisc", "change", "dev", self.interface, "root", "cake", "bandwidth", f"{rate_kbit}kbit"])`
    - Returns `True` on rc==0, logs WARNING and returns `False` on failure (per D-09)

    `get_bandwidth(self, queue: str) -> int | None`:
    - Runs `_run_tc(["-j", "qdisc", "show", "dev", self.interface])`
    - Uses `_find_cake_entry` to locate CAKE, reads `entry.get("options", {}).get("bandwidth")`
    - Returns `int(bandwidth)` (already bps in JSON per Pitfall 2), None on error

    `get_queue_stats(self, queue: str) -> dict | None` (BACK-02, BACK-04, per D-04, D-05):

    - Uses `_find_cake_entry`, then builds superset dict:

      - `tins`: iterate `cake_entry.get("tins", [])`, for each tin build dict with D-05 consumer names:
