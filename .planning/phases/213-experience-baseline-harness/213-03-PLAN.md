<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Dev VM → cake-shaper (SSH) | Read-only commands only. BatchMode=yes refuses password prompts; sudo -n refuses to wait. Failure mode is fail-fast, not hang. |
| cake-shaper → /var/lib/wanctl/metrics-*.db (live writer) | SQLite mode=ro URI form provides safe read against the live writer; never blocks the writer. |
| cake-shaper → /var/lib/wanctl/steering_state.json (root-readable) | sudo -n cat (per Phase 212 precedent). Raw output never committed; only D-08-redacted artifact lands in evidence. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-213-01 | I (Information disclosure) | steering_state.json raw output containing secrets | mitigate | Inline Python redactor applies the verbatim D-08 key pattern (password\|secret\|token\|credential\|auth\|key\|private, IGNORECASE) recursively before any artifact named *-state.redacted.json is written. The *-state.raw.json output is added to .gitignore so it never gets committed even by accident. |
| T-213-02 | T (Tampering) | cake-shaper SSH session for sudo -n sqlite3 / sudo -n cat | mitigate | BatchMode=yes prevents password-prompt hangs; sudo -n fails loudly if cached sudo absent (per Phase 212 precedent at threat T-213-02 in CONTEXT). Project uses SSH key auth exclusively. |
| T-213-03 | I (Information disclosure) | Concurrent SQLite read of live-writer DB | mitigate | URI form `file:DB?mode=ro` is the safe pattern per RESEARCH §Pattern 4. `immutable=1` explicitly forbidden in the verify gate (would give stale views). |
| T-213-05 | E (Elevation of privilege) | Steering threshold-field-name interpretation while v1.39 drift unresolved | mitigate | Script body contains no `if val < green_rtt_ms` or similar comparisons; only JSON capture. D-14 grep gate in verify enforces this. Classifier D-14 enforcement lives in Plan 04 and the test from Plan 01. |
</threat_model>
