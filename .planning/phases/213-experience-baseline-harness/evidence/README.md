# Phase 213 Evidence Command Index

Phase 213 evidence is generated by the one-command baseline harness. It is
evidence-only per D-09/D-10: dev-VM traffic generation and read-only production
queries are allowed; service/config/router mutations are not.

## Evidence Index

| Timestamp (UTC) | Source host | Command purpose | Redaction method | Output file | Mutation posture |
|---|---|---|---|---|---|
| Generated per run | dev VM | `bash scripts/phase213-baseline-capture.sh --bind-map ...` | D-08 for redacted JSON; command log excludes secrets | `RUN-<ts>/` | Evidence-only; dev-VM traffic plus read-only snapshots |

Per-RUN README files are generated by the orchestrator under `RUN-<ts>/README.md`
when applicable. `signal-sheet.json` and `signal-sheet.md` live inside each
`RUN-<ts>/` directory.

## Redaction Policy

- D-08 key pattern: `password|secret|token|credential|auth|key|private`.
- Secret-bearing values are replaced with `<REDACTED>` in `*.redacted.json`.
- `/etc/wanctl/secrets` is never read by the Phase 213 harness.
- Raw steering state stays in a process-local `/tmp` file and is removed by trap.

## Production Boundary

- Production evidence comes from `cake-shaper` read-only SSH, bound health
  endpoints, and live-writer SQLite reads opened with read-only URI mode.
- Dev VM artifacts are operator-run evidence, not production configuration.
- The harness does not restart services, edit `/etc/wanctl`, toggle steering,
  write RouterOS state, deploy code, or change controller/profiling settings.

## Artifact Location

The canonical run shape is:

```text
RUN-<ts>/manifest.json
RUN-<ts>/signal-sheet.json
RUN-<ts>/signal-sheet.md
RUN-<ts>/<wan>/<test>/manifest.json
RUN-<ts>/<wan>/<test>/health-spectrum.ndjson
RUN-<ts>/<wan>/<test>/health-att.ndjson
RUN-<ts>/<wan>/<test>/alerts-summary.json
RUN-<ts>/<wan>/<test>/steering-*-state.redacted.json
```
