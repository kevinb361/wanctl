# Plan 248.3-01 — Native Spectrum CAKE Parity for Fair fping Canary

## Objective

Align the native `wanctl@spectrum` CAKE operating point with the current external cake-autorate Spectrum production shape before any further native fping canary verdict.

Phase 248.2 proved the fping stale-window repair, but the native canary still used different CAKE parameters from external production:

- native canary: download ceiling 920Mbit and CAKE `rtt 100ms`
- external production: download 550Mbit base / 600Mbit max envelope and CAKE `rtt 25ms`

That mismatch means a native/fping keep verdict would conflate RTT backend behavior with unrelated shaper/qdisc configuration drift.

## Scope

Allowed changes:

1. Change native Spectrum config to match external cake-autorate trial envelope:
   - `continuous_monitoring.download.ceiling_mbps: 600`
   - keep `continuous_monitoring.download.floor_green_mbps: 550`
   - `cake_params.rtt: "25ms"`
2. Remove the global `cake_params.ack_filter: false` override so the existing direction-aware defaults apply:
   - download: `no-ack-filter`
   - upload: `ack-filter`
3. Add regression tests proving the native Spectrum config builds the expected qdisc shape:
   - download 550Mbit baseline / 600Mbit ceiling envelope
   - upload 18Mbit
   - `rtt 25ms`
   - download no ack-filter, upload ack-filter
   - wash on both directions
4. Deploy the parity config only for a bounded native fping canary with rollback armed.
5. Roll back live Spectrum to external cake-autorate + `icmplib` after the canary unless an explicit keep decision is made.

Out of scope:

- Permanent native fping keep/default flip.
- Startup first-sample fallback suppression.
- Changes to external cake-autorate config.
- RouterOS/MikroTik changes.
- Storage hygiene phases 249/250.

## Verification gates

Local gates:

- `tests/test_cake_params.py` passes.
- `tests/test_spectrum_cake_autorate_artifacts.py` passes.
- Config YAML parses.
- Ruff/mypy on touched Python test/module surfaces pass.
- `git diff --check` passes.

Live canary gates:

- Native `wanctl@spectrum.service` starts with fping and parity config.
- Health endpoint is healthy and attributable to native wanctl/fping.
- qdisc after startup shows:
  - `spec-router` near 550Mbit/600Mbit envelope, `rtt 25ms`, `no-ack-filter`
  - `spec-modem` 18Mbit, `rtt 25ms`, `ack-filter`
- No `RTT data stale` logs during the observation window.
- No unexpected systemd restarts.
- Rollback restores external cake-autorate/state bridge, `measurement.backend: "icmplib"`, and external qdisc shape.

## Deliverables

- `configs/spectrum.yaml` parity config.
- `tests/test_cake_params.py` parity regression tests.
- `248.3-01-SUMMARY.md` with verification and live canary evidence.
- `evidence/live-canary-20260619T*.md` with canary/rollback proof.
