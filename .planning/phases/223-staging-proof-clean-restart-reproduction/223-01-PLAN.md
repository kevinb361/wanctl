<objective>
Stand up the offline replay harness (PROOF-01). The harness drives the
v1.47-close `WANSteeringDaemon` decision pipeline through a fixture corpus
using a fake RouterOS transport and tempdir-redirected state files. It
produces per-fixture verdicts (`matches | diverges | inconclusive`) and the
operator-facing `evidence/replay-results.{json,md}` artifacts.

The harness must NOT touch the live router, NOT write to production state
file paths, and NOT mutate controller-path source (SAFE-12). The preferred
implementation strategy is constructor injection through existing seams in
`WANSteeringDaemon` (`SteeringConfig`, state-file paths, router transport).
Any new steering-source seam requires an explicit named, scoped edit
documented in `evidence/README.md` with a regression test under
`tests/steering/`.

Plan 02 owns the clean-restart-degraded fixture (PROOF-02) and appends its
results into the same `evidence/replay-results.json` artifact. Plan 03 owns
PROOF-03 spine evidence + SAFE-12 boundary check.
</objective>
