# Phase 223 Plan 01 Seam Inventory

**Choice:** option (a) FULL I/O SEAL.

The replay harness will drive `SteeringDaemon.run_cycle()` end-to-end rather than calling `update_state_machine()` directly. This preserves PROOF-03 daemon-path fidelity: every verdict is produced through the same baseline update, CAKE-read, RTT-read, state-machine, RouterOS-toggle, and state-save paths the daemon uses in production, with each live-I/O seam sealed by fakes or tempdir paths.

**Confidence posture:** production `configs/steering.yaml` has `mode.use_confidence_scoring: true`, `confidence.sustain_duration_sec: 2.0`, `confidence.recovery_sustain_sec: 3.0`, `confidence.hold_down_duration_sec: 30.0`. `ASSESSMENT_INTERVAL_SECONDS = 0.05` in `src/wanctl/steering/daemon.py:127`, so confidence-mode budgets are sustain `ceil(2.0 / 0.05) = 40` cycles, recovery `ceil(3.0 / 0.05) = 60` cycles, and hold-down `ceil(30.0 / 0.05) = 600` cycles. Plan 01 defaults fixtures to `hysteresis-only` mode by setting `SteeringConfig.use_confidence_scoring=False`; the corpus also includes one confidence-mode smoke fixture using these derived budgets.

## `run_cycle()` live-I/O verdict

Re-read of `SteeringDaemon.run_cycle()` (`daemon.py:2064-2103`) and `_run_steering_state_subsystem()` (`daemon.py:2105-2172`) found no additional live-I/O surface beyond the rows above. The exercised I/O paths are:

- `update_baseline_rtt()` → `BaselineLoader.load_baseline_rtt()` (sealed by `FixtureBaselineLoader`)
- `collect_cake_stats()` → `cake_reader.read_stats()` (sealed by `FakeCakeReader`)
- `_measure_current_rtt_with_retry()` → `load_live_rtt()` / `load_live_irtt_rtt()` (sealed by `FixtureBaselineLoader` plus urlopen/socket monkeypatches)
- `execute_steering_transition()` → `router.enable_steering()` / `router.disable_steering()` (sealed by `FakeRouterTransport`)
- `state_mgr.save()` (tempdir state file)
- `_record_steering_metrics()` (disabled `_metrics_writer`; Prometheus calls disabled via `metrics_enabled=False`)

**No steering-source edit required.** All seams are usable through constructor injection or post-construction assignment. Plan 01 may proceed with pure test-side wiring.
