# Plan 254-01 — Dry-Run Observation + Pre-Canary Approval Packet

<objective>
Run a read-only, deterministic dry-run observation prep path for wanctl route management: validate exact live inspection commands before use, collect live health/operator/RouterOS evidence without mutation, compare intended wanctl route-management state against Netwatch/live route ownership, and produce a pre-canary approval packet or a no-canary recommendation.
</objective>

<must_haves>
<truths>
- Netwatch remains the interim active WAN route owner throughout this plan.
- This plan may run read-only live inspection only after a timestamped command file passes a deterministic allowlist validator.
- This plan must not enable, disable, set, add, remove, run, import, reset, or otherwise mutate RouterOS routes, Netwatch entries, scripts, production configs, systemd units, CAKE/qdisc, or production defaults.
- No active canary is implied by dry-run observation success.
- No-mutation proof must scan only explicit issued-command transcript lines prefixed with `COMMAND:`.
</truths>
<outputs>
- Timestamped read-only command file and validation result artifact.
- Dry-run observation evidence bundle containing health JSON route-management section, operator summary output, live Netwatch/script/route inventory, and no-mutation proof.
- Pre-canary approval packet containing evidence summary, rollback proof checklist, exact proposed canary scope, stop criteria, and a recommendation: proceed-to-approval-gate or keep Netwatch owner.
</outputs>
</must_haves>

<threat_model>
T-254-01 accidental-live-mutation: an observation step issues route/Netwatch/script mutation commands. Mitigation: write exact command file first, validate against a read-only allowlist, reject mutating command actions/metacharacters, and abort before live execution if validation fails.

T-254-02 false-no-mutation-proof: raw RouterOS output contains words such as `write` or script source commands and causes confusing proof. Mitigation: evidence transcript prefixes only issued commands with `COMMAND:` and no-mutation scan inspects only those lines.

T-254-03 dry-run-treated-as-approval: dry-run observation success flows directly into active route mutation. Mitigation: this plan only creates an approval packet; Plan 254-02 has a separate explicit operator approval checkpoint.

T-254-04 rollback-drift: Snapshot-A no longer matches live route/Netwatch state. Mitigation: collect a read-only live inventory and compare to Snapshot-A before recommending canary approval.

T-254-05 SAFE-19-breach: production config/systemd/CAKE/defaults are changed during observation. Mitigation: plan acceptance criteria and transcript scans explicitly forbid those actions; any needed mutation is deferred to operator-gated Plan 254-02.
</threat_model>
