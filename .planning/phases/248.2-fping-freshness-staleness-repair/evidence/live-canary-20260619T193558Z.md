## First canary attempt pitfall

A first bounded attempt started native ownership but rolled back early because the health parser used the invalid pattern:

```bash
curl ... | python3 - <<'PY'
```

The heredoc consumed Python stdin, so the JSON parser saw empty input. Services and qdisc rolled back, but the config key initially remained `fping`; this was immediately corrected and verified back to `icmplib` before retrying.

The reusable canary reference was patched with two lessons:

1. parse health JSON via `curl -o /tmp/health.json` plus `python3 -c ... /tmp/health.json`, or pipe into `python3 -c`, not a heredoc;
2. rollback should force-normalize `measurement.backend` back to `icmplib` and parse YAML after rollback, not rely solely on backup copy semantics.
