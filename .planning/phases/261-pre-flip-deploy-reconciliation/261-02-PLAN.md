WAN_NAME=att (deploy.sh guards). The src/wanctl rsync runs in BOTH (idempotent on the 2nd).
deploy.sh rsync = -av --delete --exclude=__pycache__ --exclude=*.pyc (this is what makes repo==prod
provable). The Phase 201 predeploy gate runs only for WAN_NAME=spectrum.

deploy.sh restart behavior (witnessed statically in Plan 01, re-witnessed dynamically here):
deploy_steering_systemd and deploy_{spectrum,att}_cake_autorate scp config/units + run
`systemctl daemon-reload` ONLY — they do NOT execute any `systemctl restart` of steering or the
shaper units (those are print_next_steps manual guidance). So deploy.sh returns control to the
operator WITHOUT restarting steering or the shaper proper. This is what makes the restore-before-
restart ordering enforceable AND the shaper non-restart provable.

LANDMINE (Pitfall 1, breaks RECON-03 downstream if unhandled): repo configs/steering.yaml has NO
route_management block; the live /etc/wanctl/steering.yaml has a host-only
route_management:{enabled:true, mode:"dry_run"} block (hand-added Phase 256, never committed).
--with-steering scp's the repo config over it -> mode reverts to off. RESOLUTION FOR THIS PHASE
(option 1, conservative, SAFE-22-clean, no repo source edit): back up the host steering.yaml BEFORE
the deploy, restore it (sudo install -m0640 -o root -g wanctl) AFTER the steering scp and BEFORE the
steering restart. Do NOT edit repo configs/steering.yaml in Phase 261 (committing the block is a
deferred reviewed source edit, out of scope).

Restart scope (Pitfall 6 / SAFE-22): restart ONLY cake-autorate-{spectrum,att}-state-bridge.service +
steering.service (LAST). Do NOT restart cake-autorate-{spectrum,att}.service (the shaper proper —
bouncing it disturbs live production shaping AND violates SAFE-22). The state-bridges + steering are
the wanctl-side consumers of the deployed /opt/wanctl code.

:9102 restart gate (Pitfall 5): after each restart, poll :9102 until status in (healthy,degraded)
AND ownership_inspection.inspector_status=="ok" with a fresh last_inspected_at, with timeout — a
restart transiently returns {"status":"starting"} with no route_management section; asserting too
early is a false FAIL. The smoke script's --min-inspected-after (Plan 01) enforces freshness.

Monotonic non-restart witness (HIGH-1/HIGH-2): Plan 01 captured the pre-deploy
ActiveEnterTimestampMonotonic of cake-autorate-spectrum.service, cake-autorate-att.service, and
steering.service. systemctl show -p ActiveEnterTimestampMonotonic is monotonic-clock microseconds
that change ONLY on a (re)start of the unit. daemon-reload alone does NOT change it.
