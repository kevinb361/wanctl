<objective>
Consolidate duplicated config loading boilerplate into BaseConfig and switch setup_logging() to RotatingFileHandler.

Purpose: Both daemon Config classes duplicate identical logging and lock field SCHEMA entries and loading methods. Extracting to BaseConfig eliminates this boilerplate. Simultaneously, switching FileHandler to RotatingFileHandler prevents unbounded log growth on production containers.

Output: BaseConfig with common field loading, both daemons simplified, log rotation active with safe defaults.
</objective>

   ```python
   logging_section = self.data.get("logging", {})
   self.main_log = logging_section["main_log"]
   self.debug_log = logging_section["debug_log"]
   self.max_bytes = logging_section.get("max_bytes", 10485760)  # 10MB default
   self.backup_count = logging_section.get("backup_count", 3)

   # Common lock fields (shared by all daemons)
   self.lock_file = Path(self.data["lock_file"])
   self.lock_timeout = self.data["lock_timeout"]
   ```
4. Add max_bytes and backup_count to the BaseConfig docstring Attributes section.
5. Add constants for the defaults above the BASE_SCHEMA:
   ```python
   DEFAULT_LOG_MAX_BYTES = 10_485_760  # 10MB per log file
   DEFAULT_LOG_BACKUP_COUNT = 3         # Keep 3 rotated copies
   ```
