  <action>
Create `scripts/burst-soak-check.sh` following the soak-monitor.sh pattern. The script:

1. **Header/usage:** `#!/bin/bash`, `set -euo pipefail`, usage message showing `--start` (record soak start baseline) and `--check` (check current state and compare) modes. Default: `--check`.

2. **Configuration constants:**
   - `VM_HOST="kevin@10.10.110.223"`
   - `HEALTH_URL="http://127.0.0.1:9101/health"`
   - `SOAK_STATE_FILE="/tmp/burst-soak-baseline.json"` (stores start-of-soak snapshot)

3. **Color constants and helper functions:** Match check-tuning-gate.sh pattern (RED, GREEN, YELLOW, BLUE, NC, print_pass, print_fail, print_info).

4. **query_health() function:** SSH to VM_HOST, curl the health endpoint, return JSON. Exit with error if SSH fails or empty response.

5. **--start mode:**
   - Query health endpoint.
   - Extract and display: version, uptime_seconds, total_bursts, burst_responses_total, status, burst_detection.enabled, burst_detection.burst_response_enabled.
   - Verify burst detection is enabled (both enabled and burst_response_enabled = true). Exit 1 if not.
   - Save snapshot to `$SOAK_STATE_FILE` as JSON: `{"start_time": "$(date -Iseconds)", "start_uptime": $uptime, "start_bursts": $total_bursts, "start_responses": $total_responses, "version": "$version"}`.
   - Print: "Soak baseline recorded. Run `burst-soak-check.sh --check` after 24h."

6. **--check mode:**
   - Read `$SOAK_STATE_FILE`. Exit with error if not found ("Run --start first").
   - Query health endpoint.
   - Extract current: uptime_seconds, total_bursts, burst_responses_total, status, version.
   - Compute: `soak_hours = (current_uptime - start_uptime) / 3600` (using awk for float math).
   - Compute: `new_bursts = current_total_bursts - start_bursts`.
   - Compute: `new_responses = current_total_responses - start_responses`.
   - Check version matches start version (warn if different -- service was redeployed).
   - Check uptime > start_uptime (if lower, service restarted -- warn that counters reset).
   - Print structured report:
     ```
     Soak Duration: Xh Ym
     Version: 1.30.0
     Service Status: healthy
     New Burst Triggers: 0
     New Burst Responses: 0
     Service Restarts: 0 (uptime continuous)
     ```
   - Verdict: PASS if new_bursts == 0 AND status == "healthy" AND soak_hours >= 24. WARN if soak_hours < 24 ("Soak not yet 24h"). FAIL if new_bursts > 0.

7. **No --watch mode needed** -- Kevin can use `watch -n 300 bash scripts/burst-soak-check.sh --check` externally if desired.

IMPORTANT constraints:
- Use `kevin@10.10.110.223` (direct IP), not SSH aliases.
- Match soak-monitor.sh style for health endpoint queries (jq with fallback note).
- jq is available on dev machine for local parsing. The health JSON is fetched via SSH but parsed locally.
