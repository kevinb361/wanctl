## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| deployed YAML (over SSH) → gate-script env preflight | gate trusts what the operator declares as expected ceiling; mismatch with deployed value must abort |
| flent artifact → extractor | untrusted/possibly-collapsed measurement data crosses into gate scoring |
| evidence artifacts → git | config/health snapshots may carry secret-shaped fields |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-215-01 | Tampering | gate verdict from stale deployed config | mitigate | Env-preflight reads deployed ceiling_mbps over SSH and ABORTs (exit 2) if it != expected; a stale-config false PASS cannot be emitted. |
| T-215-02 | Information Disclosure | secrets in committed gate script / evidence scaffold | mitigate | Gate reads config KEYS for comparison only, never echoes ${ROUTER_PASSWORD}/${DISCORD_WEBHOOK_URL}; acceptance grep asserts no secret tokens; evidence dir is empty scaffold only. |
| T-215-03 | Spoofing/Repudiation | false "pass" on a collapsed measurement window | mitigate | D-09 VOID path: extractor fails closed (FlentExtractionError) and the gate emits verdict="void" on sustained signal_outlier_rate / measurement_state=collapsed — a regressing or unmeasurable window is never scored pass. |
| T-215-04 | Denial of Service | gate consuming live production endpoint during tests | accept | Test path is fully offline (fixtures, no SSH/curl); live sampling only occurs in Plan 03 against the read-only /health endpoint. Low risk. |
</threat_model>
