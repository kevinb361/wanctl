# Plan 248.4-01 — fping Startup First-Sample Readiness

## Objective

Fix the remaining native fping canary blocker: the 50ms control loop runs before the first fping background burst has published a cached sample, so `measure_rtt() is None` is immediately treated as ICMP failure and triggers gateway/TCP fallback checks.

This is not a real connectivity failure. It is startup readiness for a slower background producer.

## Scope

Allowed changes:

1. Add a bounded fping-only first-sample startup grace in `WANController`.
2. During that grace, skip `handle_icmp_failure()` and freeze/save the startup cycle instead of running fallback checks.
3. Preserve existing icmplib behavior: if icmplib has no sample, it still follows the historical fallback path.
4. Add regression tests for both fping and icmplib behavior.
5. Deploy the patched controller to `cake-shaper`, run a bounded native fping canary, and roll back live ownership/config to external cake-autorate + `icmplib`.

Out of scope:

- Permanent native fping keep/default flip.
- Changes to fping cadence or reflector set.
- Further CAKE/qdisc tuning.
- Storage hygiene Phase 249.

## Root cause

`WANController._run_rtt_measurement()` treats any `measure_rtt() is None` as ICMP failure. For fping startup, `None` can simply mean the first background fping burst is still in flight and no cached `RttSample` exists yet.

Before this phase, startup produced warning storms such as:

```text
No RTT data available (background thread starting)
All ICMP pings failed - running fallback checks
ICMP unavailable - using TCP RTT=... as fallback
```

## Implementation design

- Track `_rtt_thread_started_ts` when the background RTT thread is started.
- Add `_initial_rtt_sample_pending()`:
  - true only when active backend is `fping`;
  - no cached RTT sample exists yet;
  - current time is within `max(5s, cadence_sec * 1.5)` from thread start.
- In `measure_rtt()`, log one INFO message while waiting instead of repeated WARNs.
- In `_run_rtt_measurement()`, if initial fping sample is pending, save/freeze and early-return without `handle_icmp_failure()`.
