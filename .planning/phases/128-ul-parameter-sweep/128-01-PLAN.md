## UL Context (per D-06)

- UL bandwidth ~38 Mbps (vs DL 940 Mbps)
- UL uses 3-state machine (GREEN/YELLOW/RED) -- no SOFT_RED
- UL params may need different tuning than DL (confirmed on REST: UL factor_down=0.85 while DL was 0.90)
- DL params from Phase 127 are already applied. UL tests run with the new DL config active (per D-08)

## Test Methodology (per D-01, D-03)

1. Set value A in YAML (upload section), SIGUSR1 reload, wait 5s
2. Run `flent rrul_be -H 104.200.21.31 -l 60`, note time of day
3. Record: ICMP median, p99, max latency; DL throughput avg; UL throughput avg; state distribution from health endpoint
4. Set value B, SIGUSR1 reload, wait 5s
5. Run same flent command, note time of day
6. Compare metrics. If delta < 5%, re-run both (per D-03)
7. Apply winner, move to next parameter

## Sed Safety for UL Section

The upload section sits between `upload:` and `thresholds:` in spectrum.yaml. To scope sed to only the UL section:

```bash
# Safe UL-scoped sed pattern:
sudo sed -i '/^\s*upload:/,/^\s*thresholds:/ s/factor_down: 0.85/factor_down: 0.90/' /etc/wanctl/spectrum.yaml

# ALWAYS verify after sed -- check BOTH sections:
sudo sed -n '/download:/,/upload:/p' /etc/wanctl/spectrum.yaml   # DL section
sudo sed -n '/upload:/,/thresholds:/p' /etc/wanctl/spectrum.yaml  # UL section
```

If sed feels risky, use `sudo vi /etc/wanctl/spectrum.yaml` instead.
