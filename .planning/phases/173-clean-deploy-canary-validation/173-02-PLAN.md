Accept/reject criteria for config diffs (addresses MEDIUM review concern):
- **Accept (proceed):** Exit 0 (no diff), or diff is only whitespace/comments
- **Accept with note:** Diff shows known production tuning overrides (tuning_params live in metrics.db, not YAML)
- **REJECT (stop and fix):** Diff shows changed thresholds, endpoints, IP addresses, or db_path values that the operator did not expect. Resolve the diff before continuing.

**Step 7: Wait for Spectrum health readiness (addresses MEDIUM fixed-sleep concern)**

Poll health endpoint until it responds with uptime >= 10s (canary threshold):

```bash
for i in $(seq 1 15); do
  HEALTH=$(ssh kevin@10.10.110.223 "curl -sf http://127.0.0.1:9101/health" 2>/dev/null) && \
  UPTIME=$(echo "$HEALTH" | python3 -c "import sys,json; print(json.load(sys.stdin).get('uptime_seconds',0))" 2>/dev/null) && \
  if [ "$(echo "$UPTIME >= 10" | bc -l 2>/dev/null || echo 0)" = "1" ]; then
    echo "Spectrum ready: uptime=${UPTIME}s"
    break
  fi
  echo "Waiting for Spectrum readiness... (attempt $i, uptime=${UPTIME:-unknown}s)"
  sleep 2
done
```

**Step 8: Validate Spectrum health (per-WAN spot check -- canary-check.sh cannot do per-WAN)**

print('status:  ', d.get('status','MISSING'))
print('storage: ', d.get('storage',{}).get('status','MISSING'))
print('download:', d.get('download',{}).get('state','MISSING'))
print('upload:  ', d.get('upload',{}).get('state','MISSING'))

\""

Required results:
- version: 1.35.0 (release gate)
- storage: ok (fresh per-WAN DB must be ok -- warning is unexpected and must be investigated)
- download: GREEN or YELLOW (not unknown)
- upload: GREEN or YELLOW (not unknown)
- VERDICT: PASS

| Boundary | Description |
|----------|-------------|
| dev machine -> cake-shaper VM | SSH with key auth, sudo on remote |
| deploy.sh rsync -> /opt/wanctl | Code deployment overwrites production binaries |
| health endpoint -> validation | Trusts health JSON from localhost; version is a release gate |

| T-173-04 | D (Denial of Service) | migration stops ATT | accept | migrate-storage.sh stops both services at line 114 as unavoidable side effect of archiving shared DB; ATT restart deferred to Plan 03; documented in action step 5 |
