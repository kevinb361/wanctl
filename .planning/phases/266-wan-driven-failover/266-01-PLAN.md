# Wave 1: FailoverBridge Code

**Phase**: 266
**Wave**: 1
**Scope**: New `FailoverBridge` class in `src/wanctl/steering/failover_bridge.py`

## Tasks

1. Create `src/wanctl/steering/failover_bridge.py`:
   - `FailoverBridge` class with configurable hysteresis thresholds
   - `update(congestion_state: str)` method — feeds RED/YELLOW/GREEN, returns optional action
   - State machine: track consecutive RED/GREEN counts
   - Return `None` when no threshold crossed, `"disable"` when RED threshold crossed, `"enable"` when GREEN threshold crossed
   - YELLOW resets both counters (intermediate state = no decision)

2. Configuration shape in steering.yaml:
   ```yaml
   route_management:
     failover:
       enabled: false          # off by default, opt-in per WAN
       wan: "spectrum"         # which WAN's congestion state to watch
       red_cycles: 3           # consecutive RED cycles before failover
       green_cycles: 5         # consecutive GREEN cycles before recovery
   ```

3. Unit tests (`tests/steering/test_failover_bridge.py`):
   - Hysteresis: N RED cycles → disable, fewer than N → no action
   - Recovery: M GREEN cycles → enable, fewer than M → no action
   - YELLOW resets counters
   - State transitions: RED→YELLOW→GREEN (counter reset), GREEN→YELLOW→RED (counter reset)
   - Edge cases: threshold exactly at boundary, immediate state reversal
