```python
def _run_spike_detection(self) -> None:
    """EWMA spike detection: rate-of-change acceleration for sudden RTT spikes."""
    delta_accel = self.load_rtt - self.previous_load_rtt
    if delta_accel > self.accel_threshold:
        self._spike_streak += 1
        if self._spike_streak >= self.accel_confirm:
            self.logger.warning(...)
            self.download.red_streak = 1
            self.download.green_streak = 0
            self.download.soft_red_streak = 0
    else:
        self._spike_streak = 0
    self.previous_load_rtt = self.load_rtt
```

  <action>
Create `src/wanctl/burst_detector.py` with:

1. Module-level docstring explaining: detects multi-flow burst ramps via RTT acceleration
   (second derivative). Called once per cycle with the current load_rtt (EWMA-smoothed).
   stdlib-only, follows SignalProcessor pattern.

2. `BurstResult` frozen dataclass (slots=True):
   - `acceleration: float` -- second derivative: velocity(t) - velocity(t-1), in ms/cycle^2
   - `velocity: float` -- first derivative: load_rtt(t) - load_rtt(t-1), in ms/cycle
   - `is_burst: bool` -- True if burst confirmed (streak >= confirm_cycles)
   - `consecutive_accel: int` -- current streak of above-threshold acceleration cycles
   - `warming_up: bool` -- True until 3+ samples received (first 2 cycles)

3. `BurstDetector` class:
   - `__init__(self, wan_name: str, accel_threshold_ms: float, confirm_cycles: int, logger: logging.Logger)`:
     Stores params. Initializes `_prev_load_rtt: float | None = None`,
     `_prev_velocity: float | None = None`, `_accel_streak: int = 0`,
     `_total_bursts: int = 0`.

   - `update(self, load_rtt: float) -> BurstResult`:
     Cycle 1 (no prev_load_rtt): store load_rtt, return warming_up=True result.
     Cycle 2 (no prev_velocity): compute velocity, store it, return warming_up=True result with velocity.
     Cycle 3+: compute velocity and acceleration. If acceleration > threshold, increment streak.
     Else reset streak to 0. If streak >= confirm_cycles, set is_burst=True, increment total_bursts,
     log at WARNING: "{wan_name}: BURST detected! accel={accel:.2f}ms/cycle^2 ({streak} consecutive,
     threshold={threshold}ms/cycle^2) (detection only, response not yet enabled)".
     Return BurstResult with all computed values.

   - Properties for runtime mutation (SIGUSR1 reload):
     `accel_threshold` (getter/setter for `_accel_threshold`)
     `confirm_cycles` (getter/setter for `_confirm_cycles`)

   - Read-only property: `total_bursts -> int`

   - `reset(self) -> None`: clears _prev_load_rtt, _prev_velocity, _accel_streak to initial state.
     Does NOT reset _total_bursts (lifetime counter).

Note the WARNING log message includes "(detection only, response not yet enabled)" per
Research Pitfall 3. This will be removed in Phase 152 when response is wired.

Do NOT add any rate control actions. This is detection-only (Phase 151).
