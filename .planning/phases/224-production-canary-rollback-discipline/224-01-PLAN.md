## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Local workstation → SSH → cake-shaper | Snapshot capture and rollback execute commands as `kevin` then sudo on a production-ish host. SSH keys only (per CLAUDE.md). |
| Phase 224 scripts → /health endpoint | Read-only over HTTP. Bound (10.10.110.223:9101) and loopback (127.0.0.1:9102) surfaces. Trusted internal network. |
| Phase 224 scripts → /etc/wanctl/steering.yaml | Snapshot script reads via `sudo -n cat`; rollback writes via `sudo -n tee`. Sudoers grant assumed pre-existing. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-224-01 | Information disclosure | Snapshot artifacts (config + state) may contain router password, secrets | mitigate | Two-tier capture: redacted artifacts under `--output-dir` (committed evidence, all secrets replaced with `REDACTED`); raw artifacts under `--raw-dir` (operator-private path OUTSIDE the git tree, mode 0600, NEVER committed). Snapshot script refuses to write `--raw-dir` inside the git working tree. Rollback consumes raw artifacts; reports consume redacted artifacts. |
| T-224-02 | Tampering | Snapshot script accidentally mutates target host | mitigate | Snapshot script uses `sudo -n cat` (read-only); shellcheck-clean; no `sudo tee`, no `sudo cp`, no `systemctl` calls in snapshot path. |
| T-224-01b | Repudiation | Rollback restore source is unrecoverable (e.g., redacted YAML used) | mitigate | Rollback script hard-refuses to run without `--raw-dir`; checks that `<raw-dir>/deployed-steering.yaml` exists and is non-empty before any mutation; never references `<snapshot>/deployed-steering.redacted.yaml` as a restore source (only as comparison evidence). |
| T-224-03 | Denial of service | Rollback restart exceeds WatchdogSec=30s, watchdog fires loop | mitigate | Rollback script polls `/health.status == "healthy"` with bounded timeout (30s); staging rehearsal validates restart-to-healthy is inside that window before production touch. |
| T-224-04 | Repudiation | Snapshot timestamp drift between capture and deploy | mitigate | Snapshot directory name includes UTC TS; deploy plan (Plan 03) cites the exact snapshot TS in deploy log evidence. |
| T-224-SC | Supply chain | New external packages | accept | No new packages required — stdlib + bash + curl + jq + ssh only. No npm/pip/cargo install in this plan. |
</threat_model>
