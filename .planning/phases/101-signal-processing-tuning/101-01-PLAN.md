    ```python
    """Signal processing tuning strategies.

    Optimizes Hampel filter parameters and load EWMA time constant per-WAN from
    production metrics. All strategies are pure StrategyFn callables.

    SIGP-01: Hampel sigma from outlier rate analysis (target-based approach)
    SIGP-02: Hampel window from jitter-based noise level proxy
    SIGP-03: Load EWMA time constant from settling time analysis

    NOTE on SIGP-03: This strategy outputs parameter="load_time_constant_sec"
    (range 0.5-10s), NOT "alpha_load" directly. The applier converts tc to alpha
    via alpha = 0.05 / tc in _apply_tuning_to_controller. This avoids Pitfall 3
    from research: clamp_to_step rounds to 1 decimal (destroying alpha precision
    like 0.025 -> 0.0), and the trivial change filter (abs < 0.1) blocks tiny
    alpha deltas. Time constants in the 0.5-10s range survive both filters.
    """
    from __future__ import annotations
    import logging
    from statistics import median, mean
    from wanctl.tuning.models import SafetyBounds, TuningResult

    logger = logging.getLogger(__name__)
    MIN_SAMPLES = 60  # ~1 hour of 1m data
    ```

    **SIGP-01: tune_hampel_sigma**
    Signature: `def tune_hampel_sigma(metrics_data: list[dict], current_value: float, bounds: SafetyBounds, wan_name: str) -> TuningResult | None`
    Constants:
    - `TARGET_OUTLIER_RATE_MIN = 0.05` (5%)
    - `TARGET_OUTLIER_RATE_MAX = 0.15` (15%)
    - `SIGMA_STEP = 0.1` (adjustment step per cycle)
    - `SAMPLES_PER_MINUTE = 1200` (20Hz * 60s)

    Algorithm:
    1. Extract `wanctl_signal_outlier_count` values by timestamp into a dict `{ts: count}`.
    2. Sort timestamps, compute deltas between consecutive minutes: `delta_i = count[i] - count[i-1]`.
    3. Convert each delta to rate: `rate_i = delta_i / SAMPLES_PER_MINUTE`. Clamp rate_i to [0.0, 1.0] (guard against counter resets).
    4. If fewer than MIN_SAMPLES deltas, return None.
    5. Compute `mean_outlier_rate = mean(rates)`.
    6. If `TARGET_OUTLIER_RATE_MIN <= mean_outlier_rate <= TARGET_OUTLIER_RATE_MAX`: return None (converged).
    7. If `mean_outlier_rate > TARGET_OUTLIER_RATE_MAX`: candidate = current_value - SIGMA_STEP (more filtering).
    8. If `mean_outlier_rate < TARGET_OUTLIER_RATE_MIN`: candidate = current_value + SIGMA_STEP (less filtering).
    9. Confidence = `min(1.0, len(rates) / 1440.0)`.
    10. Return `TuningResult(parameter="hampel_sigma_threshold", old_value=current_value, new_value=round(candidate, 1), confidence=confidence, rationale=f"outlier_rate={mean_outlier_rate:.1%} {'above' if mean_outlier_rate > TARGET_OUTLIER_RATE_MAX else 'below'} target range {TARGET_OUTLIER_RATE_MIN:.0%}-{TARGET_OUTLIER_RATE_MAX:.0%}", data_points=len(rates), wan_name=wan_name)`.

    CRITICAL PITFALL (Pitfall 1 from RESEARCH.md): `wanctl_signal_outlier_count` is a monotonically increasing counter. The 1m-downsampled AVG gives a mid-bucket value, NOT a rate. Must compute deltas between consecutive timestamps. Negative deltas (counter reset on daemon restart) should be discarded (skip that pair).

    **SIGP-02: tune_hampel_window**
    Signature: `def tune_hampel_window(metrics_data: list[dict], current_value: float, bounds: SafetyBounds, wan_name: str) -> TuningResult | None`
    Constants:
    - `JITTER_LOW_THRESHOLD = 1.0` (ms -- below this, signal is stable)
    - `JITTER_HIGH_THRESHOLD = 5.0` (ms -- above this, signal is noisy)
    - `MIN_WINDOW = 5` (matches bounds.min default)
    - `MAX_WINDOW = 15` (matches bounds.max default)

    Algorithm:
    1. Extract `wanctl_signal_jitter_ms` values from metrics_data.
    2. If fewer than MIN_SAMPLES values, return None.
    3. Compute `median_jitter = median(jitter_values)`.
    4. Linear interpolation: if jitter <= JITTER_LOW_THRESHOLD, candidate = MAX_WINDOW. If jitter >= JITTER_HIGH_THRESHOLD, candidate = MIN_WINDOW. Otherwise: `candidate = MAX_WINDOW - (MAX_WINDOW - MIN_WINDOW) * (median_jitter - JITTER_LOW_THRESHOLD) / (JITTER_HIGH_THRESHOLD - JITTER_LOW_THRESHOLD)`.
    5. Confidence = `min(1.0, len(jitter_values) / 1440.0)`.
    6. Return `TuningResult(parameter="hampel_window_size", old_value=current_value, new_value=round(candidate, 1), confidence=confidence, rationale=f"median_jitter={median_jitter:.2f}ms -> window={candidate:.0f}", data_points=len(jitter_values), wan_name=wan_name)`.

    The returned new_value is a float (e.g., 9.0). Conversion to int happens in `_apply_tuning_to_controller` per research recommendation (Open Question 4).

    **SIGP-03: tune_alpha_load (outputs load_time_constant_sec)**
    Signature: `def tune_alpha_load(metrics_data: list[dict], current_value: float, bounds: SafetyBounds, wan_name: str) -> TuningResult | None`

    CRITICAL DESIGN DECISION (Pitfall 3 from research): This function receives `current_value` as a time constant in seconds (NOT raw alpha), and outputs `parameter="load_time_constant_sec"` with a time constant value (range 0.5-10s). The applier converts tc to alpha via `alpha = 0.05 / tc` in `_apply_tuning_to_controller`. This is necessary because:
    - `clamp_to_step()` returns `round(clamped, 1)` which destroys alpha precision (0.025 -> 0.0)
    - The trivial change filter `abs(delta) < 0.1` blocks tiny alpha deltas
    - Time constants (0.5-10s) survive both `round(..., 1)` and the 0.1 trivial filter
    - The `current_params` dict in Plan 02 will provide `load_time_constant_sec` (= 0.05 / alpha_load)
    - Safety bounds should be specified as tc bounds: SafetyBounds(min_value=0.5, max_value=10.0)

    Constants:
    - `TARGET_SETTLING_SEC = 2.0` (target settling time for 95% response)
    - `SETTLING_TOLERANCE = 0.5` (within +/-0.5s of target is converged)
    - `STEP_DETECTION_MULTIPLIER = 2.0` (step = delta > 2x median jitter)
    - `SETTLING_THRESHOLD_PCT = 0.05` (5% of step magnitude)
    - `CYCLE_INTERVAL = 0.05` (50ms)
    - `MIN_STEPS = 3` (need at least 3 step events for reliable analysis)

    Algorithm:
    1. Extract `wanctl_rtt_ms` and `wanctl_rtt_load_ewma_ms` by timestamp into aligned dicts.
    2. Also extract `wanctl_signal_jitter_ms` for step detection threshold.
    3. Sort by timestamp. Compute consecutive deltas of raw RTT: `abs(rtt[i] - rtt[i-1])`.
    4. Compute `median_jitter` from jitter_ms values (or from consecutive deltas if jitter metric insufficient).
    5. Detect steps: where consecutive raw RTT delta > `STEP_DETECTION_MULTIPLIER * median_jitter` and the absolute delta is >= 2.0ms (minimum meaningful step).
    6. For each step at timestamp t_step: scan forward from t_step to find when `abs(ewma[t] - rtt[t_step])` < `SETTLING_THRESHOLD_PCT * step_magnitude`. The settling time is `(t_settle - t_step)` seconds. If EWMA never settles within 10 minutes, discard that step.
    7. If fewer than MIN_STEPS detected, return None.
    8. Compute `avg_settling_sec = mean(settling_times)`.
    9. If `abs(avg_settling_sec - TARGET_SETTLING_SEC) <= SETTLING_TOLERANCE`: return None (converged).
    10. Derive target time constant: `target_tc = TARGET_SETTLING_SEC / 3.0` (~0.667s for 2s settling).
    11. Move current tc 20% toward target: `candidate_tc = current_value + 0.2 * (target_tc - current_value)`. (Conservative: do NOT jump to target in one cycle. current_value IS a time constant.)
    12. Confidence = `min(1.0, len(settling_times) / 10.0)`.
    13. Return `TuningResult(parameter="load_time_constant_sec", old_value=current_value, new_value=round(candidate_tc, 1), confidence=confidence, rationale=f"settling={avg_settling_sec:.1f}s (target {TARGET_SETTLING_SEC}s), {len(settling_times)} steps", data_points=len(settling_times), wan_name=wan_name)`.
