Fix the FD leak in NetlinkCakeBackend._reset_ipr() and wire LinuxCakeAdapter to use NetlinkCakeBackend when pyroute2 is available.

Purpose: Eliminates ~5ms/cycle fork overhead by switching from subprocess tc to netlink for all bandwidth changes. Fixes FD leak that would exhaust ulimit in ~25 minutes of netlink flapping.
Output: netlink_cake.py with FD-safe reset, linux_cake_adapter.py creating NetlinkCakeBackend instances, tests for both changes.
</objective>

    **Fix (GREEN):** Change `_reset_ipr()` in `netlink_cake.py` (line 127-129) from:
    ```python
    def _reset_ipr(self) -> None:
        """Null the IPRoute reference to force reconnect on next call."""
        self._ipr = None
    ```
    To:
    ```python
    def _reset_ipr(self) -> None:
        """Close and null the IPRoute reference to force reconnect on next call."""
        if self._ipr is not None:
            try:
                self._ipr.close()
            except Exception:
                pass
        self._ipr = None
    ```

    This matches the pattern already used in `close()` at line 437-448 but is more targeted (called from error paths where we want to force reconnect).
