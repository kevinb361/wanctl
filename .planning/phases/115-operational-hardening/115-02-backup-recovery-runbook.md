# wanctl Backup and Recovery Runbook

**Created:** 2026-03-26
**Target:** cake-shaper VM 206 (10.10.110.223)
**Scope:** Configuration, metrics data, VM snapshots, and rollback procedures

---

## 1. Backup Inventory

### What to Back Up

| Category | Path | Owner | Contents | Priority |
|----------|------|-------|----------|----------|
| Config (YAML) | `/etc/wanctl/*.yaml` | root:wanctl (0640) | spectrum.yaml, att.yaml, steering.yaml | CRITICAL |
| Secrets | `/etc/wanctl/secrets` | root:wanctl (0640) | Router password, webhook URLs | CRITICAL |
| Metrics DB | `/var/lib/wanctl/metrics.db` | wanctl:wanctl (0644) | Historical performance data (SQLite) | HIGH |
| State files | `/var/lib/wanctl/*_state.json` | wanctl:wanctl (0600) | Runtime state (spectrum, att, steering) | MEDIUM |
| Service units | `/etc/systemd/system/wanctl@.service` | root:root | Template unit for spectrum/att | HIGH |
| Service units | `/etc/systemd/system/steering.service` | root:root | Steering daemon unit | HIGH |
| Service units | `/etc/systemd/system/wanctl-nic-tuning.service` | root:root | NIC tuning oneshot | HIGH |
| Application | `/opt/wanctl/` | root:root (0755) | Python code (deployed from git tags) | LOW (reproducible) |
| SSH keys | `/etc/wanctl/ssh/` | root:wanctl (0750) | Router SSH keys (if SSH transport used) | HIGH |

### What NOT to Back Up

- `/var/lib/wanctl/metrics.db-wal` and `metrics.db-shm` -- WAL journal files, regenerated by SQLite
- `/var/lib/wanctl/metrics.db.corrupt` -- dead corrupt DB (284MB), safe to delete
- `/opt/wanctl/autorate_continuous.py.bak` -- dead backup file (178KB), safe to delete
- `/var/log/wanctl/*.log` -- logs are in journald; file logs are rotated and non-critical
- `/run/wanctl/` -- runtime directory, recreated on service start

## 2. Quick Backup Commands

### Full Backup (configs + metrics + units)

```bash
# From your workstation:
BACKUP_DATE=$(date +%Y%m%d)

# Create tarball on the VM
ssh kevin@10.10.110.223 "sudo tar czf /tmp/wanctl-backup-${BACKUP_DATE}.tar.gz \
  /etc/wanctl/ \
  /var/lib/wanctl/metrics.db \
  /var/lib/wanctl/spectrum_state.json \
  /var/lib/wanctl/att_state.json \
  /var/lib/wanctl/steering_state.json \
  /etc/systemd/system/wanctl@.service \
  /etc/systemd/system/steering.service \
  /etc/systemd/system/wanctl-nic-tuning.service"

# Copy to workstation
scp kevin@10.10.110.223:/tmp/wanctl-backup-${BACKUP_DATE}.tar.gz ~/backups/

# Clean up remote temp file
ssh kevin@10.10.110.223 "rm /tmp/wanctl-backup-${BACKUP_DATE}.tar.gz"
```

### Config-Only Backup (fast, for pre-change snapshots)

```bash
ssh kevin@10.10.110.223 "sudo tar czf /tmp/wanctl-config-$(date +%Y%m%d-%H%M%S).tar.gz /etc/wanctl/ /etc/systemd/system/wanctl@.service /etc/systemd/system/steering.service /etc/systemd/system/wanctl-nic-tuning.service"
```

### Metrics DB Backup (for data preservation)

```bash
# Use SQLite online backup (safe while services are running)
ssh kevin@10.10.110.223 "sudo -u wanctl sqlite3 /var/lib/wanctl/metrics.db '.backup /tmp/metrics-backup.db'"
scp kevin@10.10.110.223:/tmp/metrics-backup.db ~/backups/metrics-$(date +%Y%m%d).db
```

### Dependency Lock Refresh

```bash
ssh kevin@10.10.110.223 "pip freeze" > requirements-production.txt
```

## 3. Proxmox VM Snapshot

### Via Web UI

1. Navigate to: **Datacenter > odin > VM 206 (cake-shaper)**
2. Click **Snapshots** tab
3. Click **Take Snapshot**
4. Name: `pre-<reason>-YYYYMMDD` (e.g., `pre-v122-hardening-20260326`)
5. Description: Brief reason for the snapshot
6. Check **Include RAM** only if you need to preserve running state (slower)

### Via CLI (from Proxmox host)

```bash
# From odin (Proxmox host):
qm snapshot 206 pre-v122-hardening-20260326 --description "Before Phase 115 operational hardening"

# List snapshots:
qm listsnapshot 206

# Delete old snapshot (when no longer needed):
qm delsnapshot 206 <snapshot-name>
```

### When to Snapshot

- **Before** any systemd unit file changes
- **Before** any config file changes
- **Before** version upgrades or code deployments
- **Before** Phase 115 operational hardening (D-13)
- **Before** any network bridge or NIC configuration changes

## 4. Recovery Procedures

### 4a. Config Restore

```bash
# Stop all services first
ssh kevin@10.10.110.223 "sudo systemctl stop wanctl@spectrum wanctl@att steering"

# Restore from tarball
ssh kevin@10.10.110.223 "sudo tar xzf /path/to/wanctl-backup-YYYYMMDD.tar.gz -C /"

# Verify permissions
ssh kevin@10.10.110.223 "ls -la /etc/wanctl/"
# Expected: 0640 root:wanctl for .yaml and secrets files

# Restart services
ssh kevin@10.10.110.223 "sudo systemctl start wanctl@spectrum wanctl@att steering"
```

### 4b. Metrics DB Restore

```bash
# Stop services (they hold the DB open)
ssh kevin@10.10.110.223 "sudo systemctl stop wanctl@spectrum wanctl@att steering"

# Replace metrics DB
ssh kevin@10.10.110.223 "sudo rm -f /var/lib/wanctl/metrics.db /var/lib/wanctl/metrics.db-wal /var/lib/wanctl/metrics.db-shm"
scp ~/backups/metrics-YYYYMMDD.db kevin@10.10.110.223:/tmp/metrics-restore.db
ssh kevin@10.10.110.223 "sudo mv /tmp/metrics-restore.db /var/lib/wanctl/metrics.db && sudo chown wanctl:wanctl /var/lib/wanctl/metrics.db && sudo chmod 0644 /var/lib/wanctl/metrics.db"

# Restart services
ssh kevin@10.10.110.223 "sudo systemctl start wanctl@spectrum wanctl@att steering"
```

### 4c. Service Unit Restore

```bash
# Restore unit files from backup
ssh kevin@10.10.110.223 "sudo cp /path/to/backup/wanctl@.service /etc/systemd/system/wanctl@.service"
ssh kevin@10.10.110.223 "sudo cp /path/to/backup/steering.service /etc/systemd/system/steering.service"
ssh kevin@10.10.110.223 "sudo cp /path/to/backup/wanctl-nic-tuning.service /etc/systemd/system/wanctl-nic-tuning.service"

# Reload systemd and restart
ssh kevin@10.10.110.223 "sudo systemctl daemon-reload"
ssh kevin@10.10.110.223 "sudo systemctl restart wanctl@spectrum wanctl@att steering"
```

### 4d. Code Restore (from git tag)

```bash
# Application code is deployed from git tags
ssh kevin@10.10.110.223 "cd /opt/wanctl && sudo git fetch --tags && sudo git checkout v1.XX"

# Restart services to pick up code changes
ssh kevin@10.10.110.223 "sudo systemctl restart wanctl@spectrum wanctl@att steering"
```

### 4e. Full VM Rollback (Proxmox Snapshot)

**WARNING:** This reverts ALL changes since the snapshot -- configs, data, code, everything.

```bash
# From Proxmox host (odin):

# Stop VM first (required for rollback)
qm stop 206

# Rollback to snapshot
qm rollback 206 <snapshot-name>

# Start VM
qm start 206
```

After VM rollback, verify:
- Network connectivity: `ping 10.10.110.223`
- Services running: `ssh kevin@10.10.110.223 "systemctl is-active wanctl@spectrum wanctl@att steering"`
- Health check: `ssh kevin@10.10.110.223 "curl -s http://127.0.0.1:9101/health"`

### 4f. From-Scratch Rebuild

If the VM is unrecoverable, rebuild from scratch:

1. **Create VM on Proxmox (odin):**
   - VM ID: 206, Name: cake-shaper
   - OS: Debian 12 (bookworm)
   - CPU: 2 cores, Memory: 4GB, Disk: 32GB
   - Network: PCIe passthrough NICs (see v1.21 deployment docs)

2. **Base OS setup:**
   ```bash
   apt update && apt upgrade -y
   apt install -y python3 python3-pip git ethtool sqlite3
   ```

3. **Create wanctl user and directories:**
   ```bash
   useradd -r -s /usr/sbin/nologin -d /var/lib/wanctl wanctl
   mkdir -p /etc/wanctl /var/lib/wanctl /var/log/wanctl /run/wanctl /opt/wanctl
   chown root:wanctl /etc/wanctl && chmod 0750 /etc/wanctl
   chown wanctl:wanctl /var/lib/wanctl && chmod 0750 /var/lib/wanctl
   chown wanctl:wanctl /var/log/wanctl && chmod 0750 /var/log/wanctl
   chown wanctl:wanctl /run/wanctl && chmod 0750 /run/wanctl
   ```

4. **Deploy code:**
   ```bash
   cd /opt && git clone https://github.com/kevinb361/wanctl.git
   cd /opt/wanctl && git checkout v1.XX  # desired version tag
   pip install -r requirements-production.txt
   ```

5. **Restore configs from backup:**
   ```bash
   tar xzf wanctl-backup-YYYYMMDD.tar.gz -C /
   # Verify: ls -la /etc/wanctl/  (should show 0640 root:wanctl)
   ```

6. **Install service units:**
   ```bash
   cp /path/to/backup/wanctl@.service /etc/systemd/system/
   cp /path/to/backup/steering.service /etc/systemd/system/
   cp /path/to/backup/wanctl-nic-tuning.service /etc/systemd/system/
   systemctl daemon-reload
   systemctl enable wanctl@spectrum wanctl@att steering wanctl-nic-tuning
   systemctl start wanctl-nic-tuning
   systemctl start wanctl@spectrum wanctl@att steering
   ```

7. **Configure network bridges** (see v1.21 VM infrastructure docs for bridge setup)

8. **Verify** (see Section 6 below)

## 5. Rollback from Phase 115 Changes

Phase 115 (Operational Hardening) modifies systemd units and NIC tuning. If changes cause problems:

### 5a. Revert systemd Hardening (Plan 01)

Pre-Phase 115 backups should be saved as `.bak.pre115` files on the VM.

```bash
# Revert wanctl@ template
ssh kevin@10.10.110.223 "sudo cp /etc/systemd/system/wanctl@.service.bak.pre115 /etc/systemd/system/wanctl@.service"

# Revert steering service
ssh kevin@10.10.110.223 "sudo cp /etc/systemd/system/steering.service.bak.pre115 /etc/systemd/system/steering.service"

# Reload and restart
ssh kevin@10.10.110.223 "sudo systemctl daemon-reload && sudo systemctl restart wanctl@spectrum wanctl@att steering"
```

### 5b. Revert NIC Tuning Changes

```bash
# Revert NIC tuning service
ssh kevin@10.10.110.223 "sudo cp /etc/systemd/system/wanctl-nic-tuning.service.bak.pre115 /etc/systemd/system/wanctl-nic-tuning.service"

# Reload and re-run the tuning oneshot
ssh kevin@10.10.110.223 "sudo systemctl daemon-reload && sudo systemctl restart wanctl-nic-tuning"
```

### 5c. Revert Resource Limits

If MemoryMax/TasksMax/LimitNOFILE cause service failures:

```bash
# Edit unit files to remove resource limit lines
ssh kevin@10.10.110.223 "sudo sed -i '/^MemoryMax=/d; /^TasksMax=/d; /^LimitNOFILE=/d' /etc/systemd/system/wanctl@.service /etc/systemd/system/steering.service"

# Reload and restart
ssh kevin@10.10.110.223 "sudo systemctl daemon-reload && sudo systemctl restart wanctl@spectrum wanctl@att steering"
```

### 5d. Nuclear Option -- Full VM Rollback

If individual rollbacks fail or multiple changes interact badly:

```bash
# From Proxmox host (odin):
qm stop 206
qm rollback 206 pre-v122-hardening-20260326
qm start 206
```

This reverts the VM to the exact state before Phase 115 began. All Phase 115 changes are lost.

## 6. Verification After Recovery

Run these checks after any recovery or rollback:

### Service Status

```bash
ssh kevin@10.10.110.223 "systemctl is-active wanctl@spectrum wanctl@att steering"
# Expected: active active active
```

### Health Check

```bash
ssh kevin@10.10.110.223 "curl -s http://127.0.0.1:9101/health | python3 -m json.tool"
# Expected: healthy status with cycle timing data
```

### Journal Errors

```bash
ssh kevin@10.10.110.223 "journalctl -u wanctl@spectrum --since '2 min ago' --no-pager | tail -20"
# Expected: no ERROR lines, normal cycle logging
```

### NIC Tuning

```bash
ssh kevin@10.10.110.223 "ethtool -k ens16 | grep rx-udp-gro-forwarding"
# Expected: rx-udp-gro-forwarding: on

ssh kevin@10.10.110.223 "ethtool -g ens16 | grep -A1 'Current'"
# Expected: RX: 4096, TX: 4096
```

### Metrics DB Integrity

```bash
ssh kevin@10.10.110.223 "sudo -u wanctl sqlite3 /var/lib/wanctl/metrics.db 'PRAGMA integrity_check;'"
# Expected: ok
```

### Circuit Breaker

```bash
ssh kevin@10.10.110.223 "systemctl show wanctl@spectrum -p NRestarts"
# Expected: NRestarts=0 (or low number)
```
