   - Add `secondary_measurement` section under `route_management.failover.att`
   - Fields: `ping_host` (ATT gateway), optional `interval_seconds`
   - Default: use same interval as primary (2s)

2. Build secondary RTT backend:
   - Reuse `build_rtt_backend` factory from `rtt_backend_factory`
   - Point at ATT gateway (10.123.48.1) or configurable ping target
   - Measure each cycle alongside primary RTT

3. Assess ATT congestion:
   - Maintain separate baseline RTT for ATT (initialized at startup)
   - Use same `assess_congestion_state` function with ATT thresholds
   - Feed result to ATT bridge via `group.update("att", congestion_state)`

4. Daemon cycle integration:
   - Add ATT RTT measurement in `run_cycle` (after primary RTT)
   - Compute ATT congestion state
   - Replace single bridge update with group-based updates
