# Wave 1: FailoverBridgeGroup + Config Refactor

**Phase**: 267
**Wave**: 1
**Scope**: Refactor FailoverBridge into a group manager + per-WAN config

## Tasks

1. Create `FailoverBridgeGroup` class:
   - Dict of `FailoverBridge` instances keyed by WAN name
   - `update(wan_name, congestion_state)` — feed state to specific bridge
   - `get_decisions()` — return list of `FailoverDecision` from all bridges
   - `snapshot()` — return dict of per-WAN bridge status for health

2. Config schema change:
   - `route_management.failover` becomes a dict: `spectrum: {...}, att: {...}`
   - Each entry: `enabled`, `wan` (route key), `red_cycles`, `green_cycles`
   - Backward compat: if old flat format detected, wrap into dict with `spectrum` key

3. Update daemon:
   - Replace `self.failover_bridge` with `self.failover_group`
   - `_process_failover_bridge` iterates `get_decisions()` instead of single decision
   - Config reload: rebuild entire group on failover config change

4. Tests:
   - Test group creation from config dict
   - Test multiple bridges with independent state
   - Test backward compat with flat config
   - Test empty group (no failover enabled)
