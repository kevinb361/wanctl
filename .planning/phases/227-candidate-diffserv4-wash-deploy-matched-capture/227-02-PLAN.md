remote_read() { ssh -o BatchMode=yes "$SSH_HOST" "sudo -n $cmd"; }
Defaults that MUST match the capture harness so the gate and capture see the same qdisc:
  SSH_HOST=cake-shaper  ROUTER_IFACE=spec-router  MODEM_IFACE=spec-modem

`tc -s qdisc show dev <iface>` output for CAKE includes a token such as `diffserv4` or
`besteffort` on the `qdisc cake` line. Match the mode token with a word-boundary grep on the
qdisc cake line; do NOT bare-`grep -c` an unfiltered file (grep gate hygiene).

**Fail-closed parsing contract (REVIEW HIGH 227-02 — Codex; tokenize only the `qdisc cake`
line, bounded timeout, explicit proof state):**
- Isolate the single line that begins (after leading whitespace) with `qdisc cake`. There must
  be exactly ONE such line per NIC. Zero such lines → `no_cake`. More than one → `ambiguous`.
- On that one line, find exactly one allowed mode token from the closed set
  {`besteffort`, `diffserv3`, `diffserv4`, `precedence`, `diffserv8`}. Zero allowed tokens or
  more than one → `ambiguous`.
- Empty / NIC-not-present output (`Cannot find device`, empty stdout) → `missing`.
- SSH connect/exec failure or timeout → `ssh_failed`.
- Prefer Python (or strict shell tokenization) over bare grep so comments, stats lines, prior
  shell output, or unrelated text on stdout cannot false-pass.
- Bound the SSH read: `ssh -o BatchMode=yes -o ConnectTimeout=5 -o ServerAliveInterval=2
  -o ServerAliveCountMax=2 "$SSH_HOST" "timeout 8 sudo -n tc -s qdisc show dev <iface>"`
  (concrete values — connect ≤5s, total exec ≤8s) so a hang cannot stall the off-peak window.
- The per-NIC result is one of: `<mode-token>` | `missing` | `ssh_failed` | `no_cake` |
  `ambiguous`. The gate exits 0 ONLY when BOTH NICs resolve to the expected mode token; every
  other state (including any non-expected proof state) exits non-zero. This is the fail-closed
  contract: a false pass poisons the entire candidate dataset.
</interfaces>
