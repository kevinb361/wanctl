    **2. Create `src/wanctl/alert_engine.py`:**

    Module-level logger: `logger = logging.getLogger(__name__)`

    Class `AlertEngine`:
    - Constructor: `__init__(self, enabled: bool, default_cooldown_sec: int, rules: dict[str, dict], writer: MetricsWriter | None = None)`
      - `self._enabled = enabled`
      - `self._default_cooldown_sec = default_cooldown_sec`
      - `self._rules = rules` (map of type_name -> {enabled, cooldown_sec, severity})
      - `self._writer = writer` (MetricsWriter instance, for SQLite; None disables persistence)
      - `self._cooldowns: dict[tuple[str, str], float] = {}` (key=(type, wan), value=monotonic timestamp of last fire)
      - `self.logger = logger`

    - Method: `fire(self, alert_type: str, severity: str, wan_name: str, details: dict[str, Any]) -> bool`
      - If not `self._enabled`: return False
      - Check rule: `rule = self._rules.get(alert_type, {})`. If rule exists and `rule.get("enabled") is False`: return False
      - Check cooldown: call `self._is_cooled_down(alert_type, wan_name)`. If cooled down (too recent): return False
      - Record cooldown: `self._cooldowns[(alert_type, wan_name)] = time.monotonic()`
      - Persist: call `self._persist_alert(alert_type, severity, wan_name, details)`
      - Log at INFO: `f"Alert fired: {alert_type} [{severity}] on {wan_name}"`
      - Return True

    - Method: `_is_cooled_down(self, alert_type: str, wan_name: str) -> bool`
      - key = (alert_type, wan_name)
      - last_fire = self._cooldowns.get(key)
      - If last_fire is None: return False (never fired, not cooled down)
      - cooldown_sec = self._rules.get(alert_type, {}).get("cooldown_sec", self._default_cooldown_sec)
      - return (time.monotonic() - last_fire) < cooldown_sec

    - Method: `_persist_alert(self, alert_type: str, severity: str, wan_name: str, details: dict[str, Any]) -> None`
      - If self._writer is None: return (no persistence)
      - timestamp = int(time.time())
      - details_json = json.dumps(details)
      - Use writer.connection to INSERT INTO alerts (timestamp, alert_type, severity, wan_name, details)
      - Wrap in try/except, log warning on failure (never crash the daemon -- follow @handle_errors philosophy)

    - Method: `get_active_cooldowns(self) -> dict[tuple[str, str], float]`
      - Return dict of (type, wan) -> seconds remaining for active cooldowns
      - Used by health endpoint (Phase 80) and debugging
