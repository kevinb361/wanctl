  <action>
Create scripts/check_function_lines.py -- a standalone AST-based tool that:

1. Walks all .py files under a given directory (default: src/wanctl/)
2. For each function/method, counts lines EXCLUDING docstrings (per D-08: blank lines and comments DO count)
3. Reports all functions exceeding a configurable threshold (default: 50)
4. Exits with code 0 if no violations, 1 if any violations found
5. Supports --threshold N flag to override the 50-line default
6. Supports --show-all flag to list ALL functions with their line counts
7. Uses ast module to parse, handles nested classes and methods correctly
8. Output format: one line per violation: "filepath:lineno FUNCTION_NAME (N lines)"

The docstring exclusion logic: if the first statement in a function body is an ast.Expr containing an ast.Constant (string), subtract its line count (end_lineno - lineno + 1) from the total.

Make the script executable (chmod +x). Add a shebang line: #!/usr/bin/env python3

Add a Makefile target: `check-lines` that runs `.venv/bin/python scripts/check_function_lines.py src/wanctl/ --threshold 50`
