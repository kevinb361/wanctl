  <behavior>
    Episode Detection (_detect_recovery_episodes):
    - Returns empty list when fewer than 2 timestamps in metrics_data
    - Detects a recovery episode when state transitions from >= 2.0 to 0.0 in consecutive 1m samples
    - Tracks peak severity (max state value) during each episode
    - Captures pre-congestion and post-recovery rate (wanctl_rate_{direction}_mbps) at episode boundaries
    - Multiple episodes in one lookback window are all returned

    tune_step_up (RTUN-01 -- dl_step_up_mbps / ul_step_up_mbps):
    - Returns None when fewer than MIN_SAMPLES (60) wanctl_state data points
    - Returns None when zero recovery episodes detected
    - When recovery episodes exist with low re-trigger rate (< 30%): recovery is safe but may be slow, propose step_up increase
    - When recovery episodes exist with high re-trigger rate (> 50%): step_up is too aggressive, propose decrease
    - When recovery speed is fast AND re-trigger rate is low: step_up is near optimal, return None (no change)
    - Direction parameter selects which rate metric to use (wanctl_rate_download_mbps vs wanctl_rate_upload_mbps)

    tune_factor_down (RTUN-02 -- dl_factor_down / ul_factor_down):
    - Returns None when fewer than MIN_SAMPLES (60) wanctl_state data points
    - Returns None when zero congestion episodes detected
    - When congestion episodes resolve quickly (short duration): factor_down is effective, may be too aggressive (propose increase toward 1.0)
    - When congestion episodes resolve slowly (long duration): factor_down is too gentle, propose decrease toward 0.0
    - Bounds keep factor_down in reasonable range (0.5-0.98 typical)

    tune_green_required (RTUN-03 -- dl_green_required / ul_green_required):
    - Returns None when fewer than MIN_SAMPLES (60) wanctl_state data points
    - Returns None when zero recovery episodes detected
    - When re-trigger rate is high (recovery followed by quick re-entry to congestion): green_required is too low, propose current + 1
    - When re-trigger rate is low AND green_required > minimum: green_required may be too conservative, propose current - 1
    - Always proposes integer values (not fractional)
