# Wave 1: RTT Failure → RED Bridge

**Phase**: 268
**Wave**: 1
**Scope**: Treat consecutive RTT measurement failures as RED for failover

## Tasks

1. Add RTT failure tracking to daemon cycle:
   - Per-WAN counter: `rtt_fail_count` for spectrum and att
   - Increment on each RTT failure (None/timeout), reset on success
   - Config: `rtt_failure_cycles` — consecutive failures before emitting RED (default: 3)

2. Bridge integration:
   - When `rtt_fail_count >= rtt_failure_cycles`, emit RED to the bridge for that WAN
   - Log: `[FAILOVER] RTT failure for {wan} — emitting RED to bridge`
   - On RTT success after failures: emit GREEN to bridge (recovery)

3. Tests:
   - Synthetic: 3 consecutive RTT None → RED → bridge fails over
   - RTT recovers after failures → GREEN → bridge recovers
   - Intermittent failures (fail, success, fail, success) → no false trigger

4. Dry-run verification:
   - Deploy to cake-shaper, enable, observe for 1h
   - Verify no false triggers under normal conditions
