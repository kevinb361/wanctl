<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| process -> filesystem metadata | Health/metrics export reads DB/WAL and procfs state from the local host, which may be missing or transient. |
| daemon state -> operator surface | Internal counters and profiler values are transformed into public health and Prometheus payloads. |
| config/runtime path -> pressure helper | Runtime paths and DB locations come from existing config and must not create unsafe or noisy inspection behavior. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-168-01 | I | `runtime_pressure.py` | mitigate | Read only metadata needed for bounded summaries; do not expose full procfs dumps or unbounded filesystem detail. |
| T-168-02 | D | `metrics.py` scrape path | mitigate | Refresh pressure signals at scrape/low-rate cadence only; no per-cycle filesystem inspection or SQLite writes. |
| T-168-03 | T | health summary builders | mitigate | Reuse existing bounded storage/cycle telemetry and normalize unavailable inputs to explicit status values instead of partial raw payloads. |
| T-168-04 | R | pressure status classification | accept | Classification is advisory visibility only in this phase; record deterministic tests for healthy/warn/unavailable branches. |
</threat_model>
