# Plan 255-01 — Deploy Shape Proof + Safe/Off Config Contract + Rollback Plan

<objective>
Prove the live `cake-shaper` steering deployment shape read-only, define and validate the safe/off route-management production config contract, and write the Phase 256 operator-gated deploy/restart proposal with explicit rollback and stop criteria. Do not deploy, restart, reload, edit live config, mutate RouterOS, disable Netwatch, touch CAKE/qdisc, or flip route ownership.
</objective>

<must_haves>
<truths>
- Netwatch remains the active/interim WAN route owner throughout Phase 255.
- Phase 255 may inspect `cake-shaper` only after writing and validating a timestamped read-only command file.
- Phase 255 must not run `systemctl restart`, `systemctl reload`, `systemctl enable`, `systemctl disable`, `systemctl start`, `systemctl stop`, `systemctl daemon-reload`, `git pull`, `git reset`, `git checkout`, `git clean`, `rsync`, `scp`, package installs/removals, live file writes, RouterOS route/Netwatch/script mutation, CAKE/qdisc mutation, or production route-owner flip.
- Bridge health on `:9101` is not route-management health; steering health from `cake-shaper` localhost `:9102` is the route-management acceptance surface for Phase 256.
- No-mutation proof scans only issued command lines prefixed with `COMMAND:`.
</truths>
<outputs>
- Timestamped read-only command file and validation artifact.
- Deploy-shape proof artifact with live service/unit/process/code/config/health/rollback evidence or a clear blocker.
- Safe/off route-management config contract artifact with offline validation result or blocker.
- Phase 256 deploy/restart proposal with approval gate, rollback layers, stop criteria, and health checks.
</outputs>
</must_haves>

<threat_model>
T-255-01 accidental-service-mutation: a read-only inspection phase accidentally restarts/reloads/enables/disables live services. Mitigation: command file validation rejects systemd mutation verbs and execution runs only validated commands.

T-255-02 live-config-mutation: config proof edits `/etc/wanctl/steering.yaml` or writes backups during Phase 255. Mitigation: Phase 255 permits read-only config inspection only; proposed config is documented in evidence and validated offline/local without live write.

T-255-03 false-deploy-shape-assumption: planner assumes git checkout or flat rsync layout without proof. Mitigation: deploy-shape proof must capture live unit, process argv, code path, git/marker state, and config path from `cake-shaper` before proposing Phase 256.

T-255-04 bridge-health-confusion: healthy cake-autorate bridge `:9101` is mistaken for route-management health. Mitigation: artifacts explicitly distinguish bridge health from `steering.service` route-management health on `:9102`.

T-255-05 route-owner-safety-breach: planning a safe/off deploy implicitly approves active route mutation. Mitigation: config contract and Phase 256 proposal state active mutation/canary is out of scope and requires a future explicit gate; SAFE-20 remains in force.
</threat_model>
