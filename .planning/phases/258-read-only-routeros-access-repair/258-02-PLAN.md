<objective>
Make ALL THREE RouterOS reads the daemon guard depends on succeed over the daemon's real transport
(REST) and harden the read-only enforcement layer (ACCESS-02, ACCESS-03, SAFE-21).

Three surgical changes:
1. Add a read-only `_handle_netwatch_print` GET handler to `RouterOSREST` mirroring
   `_handle_route_print`, plus a dispatch branch.
2. Add a read-only `_handle_script_print` GET handler — REQUIRED, not optional (review HIGH-1).
   `RouteOwnershipGuard.inspect()` reads `/tool netwatch print detail` THEN
   `/system script print detail` (route_ownership_guard.py lines 69-76), each fail-closed. If only
   the netwatch handler is added, the guard moves from "failed to read netwatch" to "failed to read
   script" — status=error, the v1.56 blocker is NOT cleared. Both handlers are needed to clear the
   guard path. An integration test proves the guard returns a non-error result over a mocked REST
   returning route + netwatch + script.
3. Carry forward and GENERALIZE the Phase 257 read-only allowlist validator (D2 keeper). Keep its
   FORBIDDEN_SUBSTRINGS / FORBIDDEN_ROUTEROS_ACTIONS / negative self-test verbatim; replace the
   brittle exact-literal ALLOWED_COMMANDS with an ANCHORED read-object prefix predicate (after
   whitespace/slash normalization) covering /ip/route/print + /tool/netwatch/print + /system/script/print.

Purpose: ACCESS-02 needs the route + netwatch + script reads working over one supported path so the
guard clears. ACCESS-03/SAFE-21 enforce read-only PROCEDURALLY at the validator layer for the
evidence/proof command file (D2) — the validator gates the proof-command file, NOT the live run_cmd
transport (which has no RBAC limit since steering creds are reused). The GET-only handlers enforce
read-only on the transport by construction. This is the code that makes the live D4 proof (Plan 03)
possible.

Output: modified `routeros_rest.py`, new `readonly_validator.py`, unit tests for both, and a guard
integration test.
</objective>
