<objective>
Make both required RouterOS reads succeed over the daemon's real transport (REST) and harden the
read-only enforcement layer (ACCESS-02, ACCESS-03, SAFE-21).

Two surgical changes:
1. Add a read-only `_handle_netwatch_print` GET handler to `RouterOSREST` mirroring the existing
   `_handle_route_print`, plus a dispatch branch. This fixes the daemon-path blocker: a
   `/tool netwatch print` currently falls through to "Unsupported command for REST API" -> None ->
   rc=1 -> `RouteOwnershipGuard` status=error. Read-only by construction (GET only).
2. Carry forward and GENERALIZE the Phase 257 read-only allowlist validator (D2 keeper). Keep its
   FORBIDDEN_SUBSTRINGS / FORBIDDEN_ROUTEROS_ACTIONS / negative self-test verbatim; replace the
   brittle exact-literal ALLOWED_COMMANDS with a read-only verb+object predicate covering
   /ip/route/print + /tool/netwatch/print (+ /system/script/print for forward compatibility).

Purpose: ACCESS-02 needs both reads working over one supported path; ACCESS-03/SAFE-21 need the
read-only guarantee enforced at the validator layer (D2, since reused steering creds are not
RouterOS-RBAC-limited). This is the code that makes the live D4 proof (Plan 03) possible.

Output: modified `routeros_rest.py`, new `readonly_validator.py`, and unit tests for both.
</objective>
