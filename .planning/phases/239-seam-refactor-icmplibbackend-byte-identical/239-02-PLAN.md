<objective>
Make the existing `RTTMeasurement` satisfy the `RttBackend` Protocol by adding a thin, additive `probe()` method that wraps the existing ping + aggregation and returns an `RttSample | None`. This satisfies SEAM-01 (no second silo — the existing class IS the icmplib backend) and SEAM-02 (byte-identical: the publish boundary and `measure_rtt()` are untouched).

Purpose: Wire the contract from Plan 01 onto the real icmplib implementation without changing any measurement math, the `_cached` type, the publish boundary, or any downstream consumer. The hot-path slice is the byte-identity regression proof.

SEAM-01 interpretation (binding): this phase makes RTTMeasurement *conform* to RttBackend. Steering (`SteeringDaemon`) and autorate (`autorate_continuous.py`) already construct and hold a concrete RTTMeasurement, so they now consume an RttBackend structurally — no consumer is re-typed or rewired this phase (that would widen the SAFE-17 allowlist). The isinstance() test in Task 2 is the proof that "the Protocol is consumed by both steering and autorate."

Zero-success contract (HIGH-2): probe() returns None when no host yields a successful RTT — mirroring the existing all-fail behavior (BackgroundRTTThread._run keeps stale _cached on zero-success; WANController.measure_rtt returns None on "All pings failed"). A returned RttSample always carries a real rtt_ms aggregated by the existing rule from the successful hosts only.

Output: an additive `probe()` on `RTTMeasurement` in `src/wanctl/rtt_measurement.py`, plus probe() coverage in `tests/test_rtt_measurement.py`. RTTSnapshot, the aggregation rule, BackgroundRTTThread._run publish, and wan_controller.py are all byte-unchanged.
</objective>
