       class WanPanel(textual.widget.Widget):
           - wan_name: str (constructor param, used in panel title)
           - _data: dict | None (last known WAN data from wans[] element)
           - _online: bool (True when endpoint reachable)
           - _degraded: bool (True when status == "degraded")
           - _last_seen: datetime | None

           - update_from_data(self, wan_data: dict | None, status: str | None = None, last_seen: datetime | None = None):
             If wan_data is None: set _online=False, keep _data for frozen display
             If wan_data is dict: update _data, set _online=True, check status for degraded
             Call self.refresh() to trigger re-render

           - render(self) -> Rich renderable:
             Build Rich Text/Table with:
             - Title bar: WAN name + status badge (ONLINE/DEGRADED/OFFLINE)
             - Congestion state: largest text element, color-coded:
               GREEN -> green, YELLOW -> yellow, SOFT_RED -> dark_orange, RED -> bold red
             - Download: "DL {current_rate_mbps} Mbps" -- note: rate limits come from autorate top-level config, not health endpoint. Show current rate only if limit not available in WAN data.
               Actually, looking at the health JSON, download only has current_rate_mbps. The CONTEXT says "245/300 Mbps" but the limit comes from config, not health endpoint. Show just the current rate for now: "DL 245.3 Mbps [GREEN]". The rate limit display will need either a separate field or config knowledge. Per user decision "DL/UL current rates + limits (e.g., 245/300 Mbps)" -- if the health endpoint doesn't expose limits, show just rate with state. Document this gap.
             - Upload: same pattern
             - RTT: "RTT {baseline_rtt_ms:.1f} -> {load_rtt_ms:.1f} D{delta:.1f}ms" where delta = load - baseline
             - Router: small badge "[Router OK]" or "[Router UNREACHABLE]"
             - Offline: dim all text, show "Last seen: HH:MM:SS"

       Color theme per CONTEXT: GREEN=green, YELLOW=yellow, SOFT_RED=orange/bright red, RED=bold red. Use Rich markup: [green]GREEN[/], [yellow]YELLOW[/], [dark_orange]SOFT_RED[/], [bold red]RED[/].
