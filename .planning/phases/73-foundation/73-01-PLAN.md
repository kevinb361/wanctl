    - Test: EndpointPoller.last_data holds the most recent successful response
    - Test: EndpointPoller handles HTTP 500 as failure (returns None, goes offline)
  </behavior>
  <action>
    Create src/wanctl/dashboard/poller.py:

    class EndpointPoller:
        - __init__(self, name: str, base_url: str, normal_interval: float = 2.0, backoff_interval: float = 5.0)
        - Properties: is_online (bool), last_seen (datetime | None), last_data (dict | None), current_interval (float)
        - async poll(self, client: httpx.AsyncClient) -> dict | None:
          GET {base_url}/health with timeout=2.0
          On success: update last_data, last_seen, set online, reset interval to normal
          On failure (ConnectError, TimeoutException, HTTPStatusError): set offline, switch to backoff interval
          Returns parsed JSON dict or None

    Each poller instance tracks its own state independently -- no shared state between pollers. The DashboardApp (Plan 03) will create one per endpoint and call poll() on each via set_interval timers.

    Use httpx.AsyncClient passed in (not created per-call) -- the app will manage client lifecycle. This keeps poller testable without needing to mock httpx.AsyncClient creation.

    Create tests/test_dashboard/test_poller.py:
    - Use unittest.mock to mock httpx.AsyncClient
    - Create mock Response objects with .json(), .raise_for_status()
    - Test each behavior listed above
    - Use pytest-asyncio or manual event loop for async tests (check if pytest-asyncio is available, if not use asyncio.run in tests)

    Note: httpx must be installed in dev venv for tests. Run `uv pip install httpx textual` if not present.
