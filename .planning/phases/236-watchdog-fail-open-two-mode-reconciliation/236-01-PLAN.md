<objective>
Reconcile the stale `wanctl@`-coupled Silicom watchdog template to the external cake-autorate two-mode reality and wire the operator `arm`/`disarm` CLI verbs (WDOG-01, WDOG-03).

Purpose: The generic `silicom-bypass-watchdog@.service` hardcodes `After=/Wants=wanctl@%i.service` and is fed `.env` files naming the now-disabled `wanctl@<wan>.service`. An armed Spectrum watchdog would read its watched unit as inactive on a healthy system and drive the pair into bypass. This plan decouples the template, points the watched-unit name at the live controller via env, ships the watchdog artifacts off-by-default, and adds operator-gated `arm`/`disarm` verbs.

Output: Reconciled watchdog template + env examples; SYSTEMCTL-seamed petter; `arm`/`disarm` verbs in `silicom-bypass`; deploy.sh shipping watchdog scripts/template/`.env` install-if-absent and installed-not-enabled; extended offline test harness (Wave 0 fake-bpctl watchdog verbs + fake systemctl) with static and behavior tests.

This is the one phase that intentionally touches cake-shaper bypass FAILURE behavior (units/scripts/tests). It does NOT touch `src/wanctl` controller logic (SAFE-16).
</objective>
