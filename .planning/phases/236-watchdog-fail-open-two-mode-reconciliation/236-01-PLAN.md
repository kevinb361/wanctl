<objective>
Reconcile the stale `wanctl@`-coupled Silicom watchdog template to the external cake-autorate two-mode reality and wire OPERATIONALLY-SAFE `arm`/`disarm` CLI verbs (WDOG-01, WDOG-03), incorporating the cross-AI review findings through cycle 2.

Purpose: The generic `silicom-bypass-watchdog@.service` hardcodes `After=/Wants=wanctl@%i.service` and is fed `.env` files naming the now-disabled `wanctl@<wan>.service`. An armed Spectrum watchdog would read its watched unit as inactive on a healthy system and drive the pair into bypass. This plan decouples the template, points the watched-unit name at the live controller via env, ships the watchdog artifacts off-by-default, and adds operator-gated `arm`/`disarm` verbs that are SAFE: disarm never fires bypass and is fail-safe on all exit paths (HIGH-1); re-arming an active pair is clean and never bypasses it (HIGH-A); the timeout actually reaches the armed hardware timer via an atomic, key-validated env write (HIGH-2); the mutual-exclusion guarantee against the retired ATT variant is scoped to `@att` ONLY via a drop-in so arming Spectrum cannot knock ATT into bypass (HIGH-5 mechanism / HIGH-B); and `cmd_arm` preflights the env to refuse arming against a stale `wanctl@...` name in cake mode (HIGH-4 residual).

Output: Reconciled watchdog template (decoupled; NO template-wide `Conflicts=`); an `@att`-scoped `conflicts.conf` drop-in carrying the `Conflicts=` guard; env examples naming the live controller; sentinel-aware ExecStop bypass script (clean disarm); SYSTEMCTL-seamed petter; `arm`/`disarm` verbs in `silicom-bypass` with an atomic per-pair timeout write + key validation, an env preflight, a positive-integer timeout gate, a clean re-arm sequence (no raw restart), a double-petter precondition, and a fail-safe sentinel-based clean disarm; deploy.sh shipping watchdog scripts/template/drop-in/`.env` install-if-absent and installed-not-enabled; extended offline test harness (fake-bpctl watchdog verbs + fake systemctl with sanitized per-unit keys) with static and behavior tests covering every HIGH-fix behavior.

This is the one phase that intentionally touches cake-shaper bypass FAILURE behavior (units/scripts/tests). It does NOT touch `src/wanctl` controller logic (SAFE-16).
</objective>
