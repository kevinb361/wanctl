<objective>
Reconcile the stale `wanctl@`-coupled Silicom watchdog template to the external cake-autorate two-mode reality and wire OPERATIONALLY-SAFE `arm`/`disarm` CLI verbs (WDOG-01, WDOG-03), under ONE enforced global invariant that closes the recurring fail-open class identified across review cycles 1-3.

INVARIANT (W-INV) — stated verbatim, enforced as a gate:
"No fail-open watchdog unit is ever stopped or disabled (systemctl stop / disable / disable --now / restart) anywhere — CLI, deploy, rollback, or test — without first writing the operator-disarm sentinel, and the sentinel write + the stop/disable must be under a shell EXIT trap that guarantees a known sentinel state (sentinel normalized/removed) on both success AND failure, even on interrupt or set -e abort."

Purpose: The three prior replans fixed each named footgun INDIVIDUALLY and the SAME failure CLASS (an unsentineled or non-trap-protected stop/disable of a unit whose ExecStop fail-opens) kept resurfacing at the next edge. This plan stops patching sites one-by-one. It introduces ONE sanctioned helper — `sentineled_stop <iface> <systemctl-verb...>` in `scripts/silicom-bypass` — as THE ONLY sanctioned way to stop/disable/restart a fail-open watchdog unit: it writes `/run/wanctl/bpctl-watchdog/<iface>.disarm`, installs `trap 'rm -f "$sentinel"' EXIT` so the sentinel is normalized on every exit path (success, failure, interrupt, set -e abort), runs the stop/disable under non-fatal handling, and verifies the unit inactive. Every watchdog stop/disable in `cmd_disarm` AND the `cmd_arm` active re-arm path routes through this one helper. A STATIC GLOBAL-COMPLIANCE TEST (`-k invariant`) greps the whole watchdog-touching surface and FAILS on any raw `systemctl stop|disable|restart` of a `silicom-bypass-watchdog` unit outside the sanctioned helper — so a future new stop site that forgets the trap is the ONLY way to regress, and the test catches it. This single discipline closes the re-arm sentinel leak (HIGH-1-NEW) and underpins the Plan 02 rollback re-point and retired-variant retirement.

The plan also: decouples the template from `wanctl@`, points the watched-unit name at the live controller via env, ships the watchdog artifacts off-by-default; makes ExecStop sentinel-aware so an intentional disarm/re-arm restores inline (HIGH-1); writes the timeout atomically + key-validated so it reaches the armed timer (HIGH-2); scopes the mutual-exclusion `Conflicts=` to `@att` ONLY via a drop-in so arming Spectrum cannot knock ATT into bypass (HIGH-5 / HIGH-B); and preflights the env to refuse arming against a stale `wanctl@...` name in cake mode (HIGH-4 residual).

Output: Reconciled watchdog template (decoupled; NO template-wide `Conflicts=`); an `@att`-scoped `conflicts.conf` drop-in; env examples naming the live controller; sentinel-aware ExecStop bypass script; SYSTEMCTL-seamed petter; the sanctioned `sentineled_stop` trap-guarded helper plus `arm`/`disarm` verbs that route EVERY watchdog stop through it; deploy.sh shipping watchdog artifacts install-if-absent and installed-not-enabled; an extended offline test harness whose fake systemctl actually runs ExecStop on stop/disable (non-vacuous no-bypass proofs) with a SEPARATE disable-failure rc, plus the static global-compliance invariant test.

This is the one phase that intentionally touches cake-shaper bypass FAILURE behavior (units/scripts/tests). It does NOT touch `src/wanctl` controller logic (SAFE-16).
</objective>
