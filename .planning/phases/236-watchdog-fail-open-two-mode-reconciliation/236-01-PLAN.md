<objective>
Reconcile the stale `wanctl@`-coupled Silicom watchdog template to the external cake-autorate two-mode reality and wire OPERATIONALLY-SAFE `arm`/`disarm` CLI verbs (WDOG-01, WDOG-03), incorporating the five HIGH cross-AI review findings.

Purpose: The generic `silicom-bypass-watchdog@.service` hardcodes `After=/Wants=wanctl@%i.service` and is fed `.env` files naming the now-disabled `wanctl@<wan>.service`. An armed Spectrum watchdog would read its watched unit as inactive on a healthy system and drive the pair into bypass. This plan decouples the template, points the watched-unit name at the live controller via env, ships the watchdog artifacts off-by-default, and adds operator-gated `arm`/`disarm` verbs that are SAFE: disarm never fires bypass (HIGH-1), the timeout actually reaches the armed hardware timer (HIGH-2), and `@att` cannot run alongside the retired ATT variant (HIGH-5).

Output: Reconciled watchdog template (decoupled + `Conflicts=` guard); env examples naming the live controller; sentinel-aware ExecStop bypass script (clean disarm); SYSTEMCTL-seamed petter; `arm`/`disarm` verbs in `silicom-bypass` with a real per-pair timeout write, an integer/positive timeout gate, a double-petter precondition, and a sentinel-based clean disarm; deploy.sh shipping watchdog scripts/template/`.env` install-if-absent and installed-not-enabled; extended offline test harness (fake-bpctl watchdog verbs + fake systemctl) with static and behavior tests covering all four HIGH-fix behaviors.

This is the one phase that intentionally touches cake-shaper bypass FAILURE behavior (units/scripts/tests). It does NOT touch `src/wanctl` controller logic (SAFE-16).
</objective>
