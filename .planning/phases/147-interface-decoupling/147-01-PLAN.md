  <action>
Create `scripts/check_private_access.py` -- an AST-based tool that detects cross-module private attribute access per D-05.

The script must:
1. Accept a directory path as CLI argument: `python scripts/check_private_access.py src/wanctl/`
2. Walk all `.py` files in the directory recursively
3. Parse each file with `ast.parse()` and walk the AST
4. For each `ast.Attribute` node where `node.attr.startswith("_")`:
   - SKIP if `node.value` is `ast.Name` with `id == "self"` (same-class access)
   - SKIP if `node.value` is `ast.Name` with `id == "cls"` (classmethod access)
   - SKIP if `node.attr.startswith("__")` (dunder methods)
   - SKIP if the access matches an entry in the allowlist
   - Otherwise, report as a violation: `{filepath}:{lineno}: {accessor}.{attr}`
5. Include an ALLOWLIST dict at the top of the script mapping `(filename_stem, attr_name)` tuples. Initially populate with ALL known violations from the coupling census (~113 entries) so the script passes immediately. As Plans 02-04 eliminate violations, entries are removed from the allowlist.
6. Print summary: `{N} violations found ({M} in allowlist, {K} new)`
7. Exit code 0 if no NEW violations (allowlisted ones are expected). Exit code 1 if any new violation found.
8. Use the shebang `#!/usr/bin/env python3` and include a module docstring

For the allowlist, organize by boundary (comments matching the coupling census boundaries 1-6 from RESEARCH.md). The allowlist entries should be tuples of `(source_file_stem, private_attr_name)` -- for example `("autorate_continuous", "_profiling_enabled")`.

Create `tests/test_boundary_check.py` with parametrized tests that:
1. Write temporary Python snippets to a tmpdir
2. Run `check_file()` on them
3. Assert violations are or are not detected per the behavior spec above

2. Add `check-boundaries` to the `ci:` recipe after `check-deps`
3. The updated ci line should be: `ci: lint type coverage-check dead-code check-deps check-boundaries`
