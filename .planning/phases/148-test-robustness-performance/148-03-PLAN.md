```python
class RateLimiter:
    def __init__(self, max_changes: int = 10, window_seconds: float = 60):
        self.max_changes = max_changes
        self.window_seconds = window_seconds
        self.change_times: deque = deque()
    
    def can_change(self) -> bool:
        self._clean_old()
        return len(self.change_times) < self.max_changes
    
    def record_change(self) -> None:
        self._clean_old()
        self.change_times.append(time.monotonic())  # <-- THIS is what to mock
    
    def _clean_old(self) -> None:
        now = time.monotonic()  # <-- AND THIS
        cutoff = now - self.window_seconds
        while self.change_times and self.change_times[0] < cutoff:
            self.change_times.popleft()
```

**Priority 1: test_rate_limiter.py (4.9s savings — 88% of all real sleep time)**

Replace all 6 `time.sleep()` calls with mocked `time.monotonic`. The RateLimiter class uses `time.monotonic()` in `record_change()` and `_clean_old()` to track sliding windows.

For each test that uses `time.sleep(N)`:
1. Add `from unittest.mock import patch` if not already imported
2. Wrap the test body in `with patch("wanctl.rate_utils.time") as mock_time:` (or more precisely, `patch("wanctl.rate_utils.time.monotonic")`)
3. Set `mock_time.monotonic.return_value = 100.0` at the start
4. Where the test had `time.sleep(1.1)`, instead advance the mock: `mock_time.monotonic.return_value = 101.1`
5. Remove the `import time` at the top of the file if no longer needed

**Example transformation for test at line 82-86:**
```python
# BEFORE:
def test_window_expiration(self):
    limiter = RateLimiter(max_changes=2, window_seconds=1)
    limiter.record_change()
    limiter.record_change()
    time.sleep(1.1)
    assert limiter.can_change() is True

# AFTER:
def test_window_expiration(self):
    with patch("wanctl.rate_utils.time") as mock_time:
        mock_time.monotonic.return_value = 100.0
        limiter = RateLimiter(max_changes=2, window_seconds=1)
        limiter.record_change()
        limiter.record_change()
        mock_time.monotonic.return_value = 101.1  # 1.1s later
        assert limiter.can_change() is True
```

Apply this pattern to all 6 sleep calls in test_rate_limiter.py. Note: some tests use `time.sleep(0.5)` or `time.sleep(0.6)` — these need proportional mock advances (e.g., 100.0 -> 100.5, 100.0 -> 100.6).

**IMPORTANT:** The mock must be on `wanctl.rate_utils.time` (where RateLimiter imports time), not on `time` directly (which would only affect the test module's time import).
