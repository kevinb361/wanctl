**Step 1: Audit and fix xdist isolation in tests/conftest.py**

Per D-06: Each xdist worker gets unique temp dirs. All DB-touching fixtures must use tmp_path.

Check the root tests/conftest.py for:
- `temp_dir` fixture: If it uses `tempfile.TemporaryDirectory()` instead of `tmp_path`, it may share across workers. Verify it uses pytest's `tmp_path` (which is per-worker unique in xdist).
- MetricsWriter singleton: Verify `_reset_instance()` is called in setup AND teardown of any singleton-using fixture.
- Prometheus registry: Verify autouse reset fixtures exist in files touching metrics.

Per D-18: Enforce Prometheus registry reset fixtures. Verify every test file that imports from `wanctl.metrics` or `wanctl.prometheus_metrics` has an autouse reset fixture OR inherits one from conftest. If any file is missing this, add it.

Scan for missing resets:
```bash
# Files importing metrics modules
grep -rl "from wanctl.metrics\|import wanctl.metrics\|from wanctl.prometheus" tests/ | sort
# Files with reset fixtures
grep -rl "metrics.reset\|reset_metrics\|_reset_instance" tests/ | sort
```
Add missing autouse reset fixtures where needed.

**Step 2: Add timeout exemptions for HTTP server tests**

Per D-09 and RESEARCH.md Pitfall 5: Tests that start real HTTP servers need more timeout headroom.

Add `@pytest.mark.timeout(5)` marker to test classes/methods in:
- `tests/steering/test_steering_health.py`: Classes that start steering health HTTP server (have `time.sleep(0.15)` for thread sync)
- `tests/test_health_check.py`: Classes that start autorate health HTTP server (have `time.sleep(0.1)`)
- `tests/test_metrics.py`: Test methods that start Prometheus HTTP server (have `time.sleep(0.05)`)

Only add the marker to specific classes/methods that start real HTTP servers, not entire files.

**Step 3: Add timeout exemptions for RTT timeout tests**

The tests in `tests/test_rtt_measurement.py` that use `time.sleep(10)` inside a side_effect function (lines 76 and 571) are designed to be interrupted by a timeout. These should NOT need a marker because the timeout is 0.01s. However, verify they pass with `--timeout=2`. If they fail, add `@pytest.mark.timeout(5)`.

**Step 4: Verify full suite with xdist + timeout**

Run the full suite: `.venv/bin/pytest tests/ -n auto --timeout=2 -x -q`

If any tests fail due to timeout:
- If the test legitimately needs > 2s (HTTP server startup): add `@pytest.mark.timeout(5)`
- If the test is slow due to real sleep: note it for Plan 03 (sleep elimination)
- If the test is slow due to computation: investigate and optimize in place
