**Create `scripts/check_test_brittleness.py`.**

This script counts cross-module private patches per test file and fails if any file exceeds a threshold (default 3, target 0 per D-11). Per D-12, counting is per test file, not per class/function.

**Detection approach:**
The script scans test files for two patterns:

1. **String-based patch:** `patch("wanctl.MODULE._attr")` where MODULE is a different module than the test file's target module. A test file `tests/test_foo.py` legitimately patches `wanctl.foo._private` (same-module). But `tests/test_foo.py` patching `wanctl.bar._private` is cross-module.

   Module extraction: from the patch string `"wanctl.MODULE._attr"` or `"wanctl.MODULE.SUBMODULE._attr"`, extract the module path. Compare against the test file's implied target (e.g., `test_check_cake.py` targets `check_cake`). If the patched module path differs, it's cross-module.

   Special case: `test_calibrate.py` patches `wanctl.calibrate._step_*` — these are same-module, acceptable. `test_systemd_utils.py` patches `wanctl.systemd_utils._*` — same-module, acceptable. `test_router_client.py` patches `wanctl.router_client._*` — same-module, acceptable.

2. **patch.object targeting private methods cross-module:** `patch.object(imported_obj, "_method")` where the object's class is defined in a different module. This is harder to detect statically — use a simpler heuristic: if the patch.object target string starts with `_` and the variable name doesn't match `self` or a same-module variable, flag it.

**Implementation structure (model after check_private_access.py):**

```python
#!/usr/bin/env python3
"""Check for cross-module private mock patches in test files.

Scans test files for patch() calls that target private attributes (_prefixed)
in modules OTHER than the module under test. Per D-10, cross-module is defined
as: patching a _attr belonging to a different module than the code under test.

Same-module patches (test_foo.py patching wanctl.foo._private) are acceptable.

Usage:
    python scripts/check_test_brittleness.py tests/
    python scripts/check_test_brittleness.py tests/ --threshold 3
    python scripts/check_test_brittleness.py tests/ --verbose

Returns exit code 0 if all files are within threshold.
Returns exit code 1 if any file exceeds threshold.
"""
```

Key function: `def count_cross_module_patches(filepath: Path) -> list[str]`
- Parse file with `ast` module
- Walk AST for `ast.Call` nodes where `func` is `patch` (from unittest.mock)
- For string-based patches: extract module path from the first argument (a string constant)
- Determine test file's target module: strip `test_` prefix from filename, map to wanctl module
  - `test_check_cake.py` -> targets `check_cake` AND `check_cake_fix` (sibling modules count as cross for this check per research Open Question 2 recommendation)
  - Wait, correction: per D-10, cross-module = different file = different module. So `test_check_cake.py` patching `check_cake_fix._save_snapshot` IS cross-module. Only `test_check_cake.py` patching `check_cake._create_audit_client` is same-module.
- For each patch string matching `wanctl.{module}._*`, compare `{module}` against the test file's target module list
- If different -> cross-module violation
- Collect per-file counts and violation details

Main function:
```python
def main() -> int:
    parser = argparse.ArgumentParser(...)
    parser.add_argument("directory", type=Path)
    parser.add_argument("--threshold", type=int, default=3)
    parser.add_argument("--verbose", "-v", action="store_true")
    args = parser.parse_args()
    
    violations_by_file = {}
    for test_file in sorted(args.directory.rglob("test_*.py")):
        violations = count_cross_module_patches(test_file)
        if violations:
            violations_by_file[test_file] = violations
    
    # Report
    exceeds = 0
    for filepath, violations in violations_by_file.items():
        count = len(violations)
        status = "FAIL" if count > args.threshold else "WARN"
        if status == "FAIL":
            exceeds += 1
        if args.verbose or status == "FAIL":
            print(f"  [{status}] {filepath.name}: {count} cross-module patches")
            for v in violations:
                print(f"    - {v}")
    
    total = sum(len(v) for v in violations_by_file.values())
    print(f"\n{total} cross-module private patches across {len(violations_by_file)} files")
    if exceeds:
        print(f"FAIL: {exceeds} file(s) exceed threshold of {args.threshold}")
    return 1 if exceeds > 0 else 0
```
