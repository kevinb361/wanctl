  <behavior>
    - Test 1: Script detects a cross-module private patch like `patch("wanctl.other_module._func")` in a test file and counts it
    - Test 2: Script does NOT count same-module patches (test_foo.py patching wanctl.foo._func is acceptable per D-10)
    - Test 3: Script does NOT count `patch.object(self_instance, "_method")` for same-module objects
    - Test 4: Script counts `patch.object(imported_obj, "_method")` when imported_obj is from a different module
    - Test 5: Script exits 0 when all files have <= threshold (3 per D-12) cross-module patches
    - Test 6: Script exits 1 when any file exceeds threshold
    - Test 7: conftest.py files are exempt from the check (Claude's discretion -- conftest provides shared fixtures)
  </behavior>
  <action>
Create `scripts/check_test_brittleness.py` that scans test files for cross-module private patches. Per D-12, count per test file (not per class or function). Per D-11, target zero; per D-12, threshold is 3 as safety valve.

The script should:
1. Accept a directory argument (e.g., `tests/`)
2. For each .py file in the directory (recursively), determine the "module under test" from the filename (test_foo.py -> foo)
3. Scan for `patch("wanctl.MODULE._attr")` strings where MODULE != the module under test
4. Also scan for `patch.object(obj, "_method")` where obj comes from a different module (harder to detect statically -- use AST if possible, otherwise regex with known patterns)
5. Count cross-module patches per file
6. Exit 0 if all files have <= 3 cross-module patches, exit 1 otherwise
7. Print summary: "N files checked, M cross-module private patches found, K files exceeding threshold"
8. Skip conftest.py files

Implementation reference: Model after `scripts/check_private_access.py` structure (argparse, Path-based scanning, exit codes).

Update `Makefile` `check-boundaries` target to also run the test brittleness check:
```makefile
check-boundaries:
	.venv/bin/python scripts/check_private_access.py src/wanctl/
	.venv/bin/python scripts/check_test_brittleness.py tests/ --threshold 3
```

This means `make ci` automatically enforces the brittleness threshold (since ci depends on check-boundaries).

Per D-13: Coverage floor stays at 90%. No changes to coverage config.
