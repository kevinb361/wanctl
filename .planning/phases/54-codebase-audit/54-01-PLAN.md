<task type="auto">
  <name>Task 2: Clean up steering __init__.py conditional imports</name>
  <files>src/wanctl/steering/__init__.py</files>
  <action>
Simplify the CONFIDENCE_AVAILABLE conditional import block in `src/wanctl/steering/__init__.py`. The current pattern:

1. Sets `CONFIDENCE_AVAILABLE = False`
2. Does `try: from . import steering_confidence as _sc` with bare `except ImportError: pass`
3. Then at module level (outside class/function): conditionally assigns `ConfidenceController = _sc.ConfidenceController` etc.
4. Dynamically extends `__all__`

This is confusing because `steering_confidence.py` always exists in the package -- the `ImportError` can never fire. The try/except suggests it might be an optional dependency, but it is not.

Replace with a direct import approach:
- Remove the try/except block
- Import confidence symbols directly: `from .steering_confidence import ConfidenceController, ConfidenceSignals, ConfidenceWeights, TimerState, compute_confidence`
- Set `CONFIDENCE_AVAILABLE = True` as a constant (it is always available)
- Add confidence symbols directly to `__all__` list (no dynamic mutation)
- Remove the conditional `if CONFIDENCE_AVAILABLE:` block at module bottom

Keep the same public API surface -- all previously exported symbols remain exported. The only change is removing the false optionality.

Run `ruff check src/wanctl/steering/__init__.py` and `ruff format src/wanctl/steering/__init__.py` after changes.
