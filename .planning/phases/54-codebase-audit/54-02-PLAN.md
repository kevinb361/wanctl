**1. Add `record_cycle_profiling()` to `src/wanctl/perf_profiler.py`:**

Create a standalone function (not a method) that consolidates the duplicated `_record_profiling` pattern from both daemons:

```python
def record_cycle_profiling(
    profiler: OperationProfiler,
    timings: dict[str, float],  # {"label": duration_ms, ...}
    cycle_start: float,
    cycle_interval_ms: float,
    logger: logging.Logger,
    daemon_name: str,  # "autorate" or "steering" for log prefix
    overrun_count: int,
    profiling_enabled: bool,
    profile_cycle_count: int,
    report_interval: int = 1200,
) -> tuple[int, int]:
    """Record subsystem timing, detect overruns, emit structured log.

    Returns:
        (updated_overrun_count, updated_profile_cycle_count)
    """
```

The function:
- Computes `total_ms` from `cycle_start` using `time.perf_counter()`
- Records each timing in `timings` dict to profiler via `profiler.record(label, ms)`
- Also records `{daemon_name}_cycle_total` with total_ms
- Detects overrun (total_ms > cycle_interval_ms), increments overrun_count
- Rate-limited WARNING on 1st, 3rd, every 10th overrun
- Structured DEBUG log with all timing fields + overrun bool
- Periodic profiling report when enabled and cycle count reaches interval
- Returns updated (overrun_count, profile_cycle_count)
