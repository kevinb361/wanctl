# Plan 248.1-01 — Operator-Gated fping Controlled Canary

## Objective

Run a controlled Spectrum fping backend canary only after explicit operator approval. The canary must be reversible, observable, and scoped to the native `wanctl@spectrum.service` backend path.

Phase 248 cleared fping for canary eligibility. This plan turns that verdict into an approval-gated live procedure. It does not authorize a blind production default flip.

## Read-only preflight finding

Evidence artifact:

- `.planning/phases/248.1-fping-controlled-canary/evidence/preflight-20260619T1958Z.json`

Observed live shape:

- `cake-autorate-spectrum.service`: active
- `cake-autorate-spectrum-state-bridge.service`: active
- `steering.service`: active
- `wanctl@spectrum.service`: inactive
- `/etc/systemd/system/cake-autorate-spectrum.service` has `Conflicts=wanctl@spectrum.service`
- `/etc/wanctl/spectrum.yaml` currently has `measurement.backend: "icmplib"`
- `/opt/wanctl` is a flat deploy, not a git checkout
- Health endpoint source is `cake-autorate-state-bridge`

Implication: editing `/etc/wanctl/spectrum.yaml` to `measurement.backend: "fping"` alone will not affect the active Spectrum shaper while external cake-autorate owns the path. A real canary requires switching live Spectrum ownership from external cake-autorate to native `wanctl@spectrum.service` for the canary window, then reverting or intentionally keeping native ownership.

## Scope

Allowed after approval:

1. Back up `/etc/wanctl/spectrum.yaml`.
2. Change only `measurement.backend` from `"icmplib"` to `"fping"`.
3. Stop/yield `cake-autorate-spectrum.service` and its state bridge as required by the unit conflict.
4. Start `wanctl@spectrum.service`.
5. Observe for the approved canary window.
6. Roll back immediately on gate failure or operator request.

Not allowed in this phase:

- RouterOS/MikroTik mutations.
- Controller-path code changes.
- Queue algorithm, rate, threshold, or CAKE qdisc tuning changes.
- Do not reuse `CYCLE_P99_ABS_CEILING_MS=10.0` as a hard rollback gate.
- Leaving production in an undocumented state.

## Approval checkpoint

Stop before any command that changes live service state or live config.

Required operator approval wording should include:

- The canary will briefly move Spectrum from external cake-autorate to native `wanctl@spectrum.service`.
- Rollback returns Spectrum to external cake-autorate and restores `measurement.backend: "icmplib"` in `/etc/wanctl/spectrum.yaml`.
- Steering remains dependent on the state file/health endpoint; verify it after ownership switch.

## Proposed live canary commands

These commands are a plan, not executed until approval.

### Pre-mutation snapshot

```bash
ssh cake-shaper 'set -e
  date -Is
  systemctl is-active cake-autorate-spectrum.service cake-autorate-spectrum-state-bridge.service steering.service wanctl@spectrum.service 2>/dev/null || true
  sudo -n -u wanctl sed -n "145,158p" /etc/wanctl/spectrum.yaml
  curl -fsS http://10.10.110.223:9101/health
  sudo -n tc qdisc show dev spec-router | head -5
  sudo -n tc qdisc show dev spec-modem | head -5
'
```

### Apply canary

```bash
ssh cake-shaper 'set -euo pipefail
  ts=$(date -u +%Y%m%dT%H%M%SZ)
  sudo -n cp -a /etc/wanctl/spectrum.yaml /etc/wanctl/spectrum.yaml.pre-fping-canary-$ts
  sudo -n python3 - <<"PY"
from pathlib import Path
p = Path("/etc/wanctl/spectrum.yaml")
s = p.read_text()
old = "measurement:\n  backend: \"icmplib\""
new = "measurement:\n  backend: \"fping\""
if old not in s:
    raise SystemExit("expected icmplib backend block not found")
p.write_text(s.replace(old, new, 1))
PY
  sudo -n python3 - <<"PY"
from pathlib import Path
import yaml
cfg = yaml.safe_load(Path("/etc/wanctl/spectrum.yaml").read_text())
assert cfg["measurement"]["backend"] == "fping"
print("backend=fping")
PY
  sudo -n systemctl stop cake-autorate-spectrum-state-bridge.service cake-autorate-spectrum.service
  sudo -n systemctl start wanctl@spectrum.service
  sleep 5
  systemctl is-active wanctl@spectrum.service
  curl -fsS http://10.10.110.223:9101/health
'
```

### Observe canary

Suggested first canary window: 15 minutes visible foreground monitoring, extend only if clean.

```bash
ssh cake-shaper 'for i in $(seq 1 30); do
  date -Is

```

## Canary gates

Fail and roll back immediately if any occur:

- `wanctl@spectrum.service` fails to start or enters restart loop.
- Health endpoint becomes unavailable or source/shape is not attributable to native wanctl.
- Unexpected systemd restart count increases during the canary window.
- RTT/loss/drop telemetry is missing, stale, or nonsensical.
- Spectrum qdisc rate state is missing or inconsistent with expected native wanctl ownership.
- Steering consumes stale state or reports unhealthy state after ownership switch.
- Operator requests rollback.

Pass conditions for the first canary window:

- `wanctl@spectrum.service` remains active for the approved window.
- `steering.service` remains active.
- Health endpoint is healthy and current.
- Backend attribution confirms fping/native wanctl where available.
- RTT/loss/drop fields are sane.
- No unexpected restarts.
- Rollback path remains proven and available.
