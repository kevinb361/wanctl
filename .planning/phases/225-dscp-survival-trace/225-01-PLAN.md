<objective>
Produce a read-only, evidence-backed trace map of where DSCP is set, preserved, or stripped along
the Spectrum download/upload path to CAKE ingress, satisfying DSCP-01. Deliver a capture script
that reads the live `bridge qos` nftables rule counters and CAKE-interface qdisc/filter state on
`cake-shaper`, and a `DSCP-TRACE.md` narrative documenting each stage (CRS hardware QoS trust →
Ruckus QoS mirroring → cake-shaper bridge `spectrum_dl` chain → CAKE egress on spec-router/spec-modem).
No external network gear is mutated; no CAKE mode change.
</objective>

<context>
The cake-shaper bridge `bridge-qos.nft` sets DSCP at `hook forward priority -10` on download
flows (`iif spec-modem oif spec-router` → chain `spectrum_dl`) BEFORE the CAKE egress qdisc on
`spec-router`. Its `ip dscp set ef|af41|cs1` rules carry live packet/byte counters that directly
quantify how much traffic is being marked non-BestEffort right now — the single cheapest read-only
signal for the survival question. CRS trust maps and Ruckus QoS mirroring affect endpoint-set marks
upstream and are documented (read-only RouterOS REST reference at 10.10.99.1 is allowed for the
narrative only, never mutated). Production CAKE mode is `besteffort` (one tin), so per-tin CAKE
counters cannot answer survival — this plan captures the bridge marking signal and the raw qdisc
state, and the DSCP distribution at enqueue is handled in 225-02.
</context>

<task id="225-01-1" type="execute">
<title>Write the read-only DSCP trace capture script</title>
<action>
Create `scripts/phase225-dscp-trace.sh` modeled on `scripts/phase224-snapshot-a.sh` conventions:
`set -euo pipefail`; args `--output-dir <dir>` (required) and `--ssh-host` (default `cake-shaper`);
`require_command` for ssh, python3, sha256sum; refuse a non-empty `--output-dir`. All target-host
access is read-only via `ssh -o BatchMode=yes "$SSH_HOST" "sudo -n <read-cmd>"`. Capture, each to a
named file under `--output-dir`:
- `nft-bridge-qos.txt` ← `sudo -n nft list table bridge qos` (full ruleset WITH counters).
- `tc-qdisc-spec-router.txt` ← `sudo -n tc -s qdisc show dev spec-router`.
- `tc-qdisc-spec-modem.txt` ← `sudo -n tc -s qdisc show dev spec-modem`.
- `tc-filter-spec-router.txt` ← `sudo -n tc filter show dev spec-router` (and `... root`/`... ingress` variants as available).
- `tc-filter-spec-modem.txt` ← `sudo -n tc filter show dev spec-modem`.
- `ip-link.txt` ← `sudo -n ip -d link show` (interface/bridge topology evidence).
Parse the `spectrum_dl` chain counters out of `nft-bridge-qos.txt` into a small redacted summary
`bridge-mark-counters.txt` listing each `ip dscp set ef|af41|cs1` rule with its packets/bytes.
Write a `MANIFEST.md` (Captured timestamp, Source Posture = read-only, Artifacts list, sha256 of
each file) following the phase224 manifest style. End with a required-artifact assertion loop that
exits non-zero if any expected file is missing or empty. The script MUST NOT contain any nft
`add`/`delete`/`flush`/`-f`, any `tc ... add/del/replace`, any CAKE-mode write, or any reference to
controller-path source files.
</action>

- Script contains a required-artifact assertion loop that exits non-zero on any missing/empty output file.
