<objective>
Turn the 225-01 trace evidence and 225-02 ingress-DSCP distribution into a single gated DSCP-03
verdict that either short-circuits the A/B (marks don't survive → v1.44 confirmed → milestone may
close negative) or unblocks Phase 226 (marks survive → proceed). Compute the verdict against the
pre-registered decision logic so it cannot be reverse-fitted, and record the SAFE-13 boundary check
proving zero controller-path diff vs the v1.48 close.
</objective>

<context>
The decision logic is pre-registered in `225-RESEARCH.md` (## DSCP-03). MARKS DO NOT SURVIVE if the
bridge `ip dscp set` rule counters are negligible under representative load AND the deliberately-marked
EF probe does not arrive intact at the CAKE enqueue interface. MARKS DO SURVIVE if the bridge counters
show meaningful non-BestEffort marking AND/OR the EF probe arrives intact. Ambiguous/partial results
record as a qualified proceed, deferring the volume question to the GATE-01 "useful non-BestEffort tin
separation" threshold in Phase 226. The SAFE-13 idiom is the v1.48 Phase 224 (SAFE-12) boundary check
reused verbatim with the `v1.48` git tag as the anchor: `git diff --name-only v1.48 HEAD -- <controller
files>` must be empty, and `configs/att.yaml` byte-identical. Phase 225 is trace/evidence + scripts
only, so SAFE-13 is expected to hold trivially; the value is the recorded proof.
</context>
