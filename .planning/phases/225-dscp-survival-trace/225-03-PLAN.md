<objective>
Turn the 225-01 trace evidence and 225-02 ingress-DSCP distribution into a single gated DSCP-03
verdict that either short-circuits the A/B (marks don't survive → v1.44 confirmed → milestone may
close negative) or unblocks Phase 226 (marks survive → proceed). Compute the verdict against the
pre-registered decision logic so it cannot be reverse-fitted, and record the SAFE-13 boundary check
proving zero controller-path diff vs the v1.48 close.
</objective>

<context>
The decision logic is pre-registered in `225-RESEARCH.md` (## DSCP-03), hardened after the cycle-1
review: thresholds are numeric, the branches are mutually exclusive and evaluated top-to-bottom, and
`unknown`/`invalid` channels ALWAYS map to the QUALIFIED fail-safe (never to the negative branch).
MARKS_DO_NOT_SURVIVE fires only on a logical AND of two VALID negatives (negligible DL histogram
<1.0% AND DL EF probe stripped <10%, both above the sample floor). MARKS_SURVIVE fires on meaningful
DL marking (≥5.0%) OR a surviving DL EF probe (≥90%). Everything else — ambiguous bands, degraded
probes, absent bridge counters, unproven wash ordering, quiet windows, UL-only positives — is
MARKS_SURVIVE_QUALIFIED, deferring the volume question to Phase 226 GATE-01.

The SAFE-13 boundary check must match the FULL v1.48 Phase 224 (SAFE-12) standard, NOT the weaker
`git diff --name-only` the original draft used (which misses staged/unstaged/untracked state). The
reference artifact is the v1.48 `safe12-boundary-check.json`, which recorded `per_path_diff` (per-file
added/removed/lines), `per_file_sha256_equal`, `dirty_tree` (unstaged/staged/untracked/status_porcelain),
`dirty_tree_clean`, `committed_clean`, and commit anchors. Phase 225 is trace/evidence + scripts only,
so SAFE-13 is expected to hold trivially; the value is the recorded proof at the SAFE-12 standard.
</context>

<action>
Create `scripts/phase225-safe13-boundary-check.sh` (`set -euo pipefail`) modeled on the v1.48
`safe12-boundary-check.json` artifact (NOT the weaker name-only draft). Args `--anchor` (default
`v1.48`) and `--out` (default
`.planning/phases/225-dscp-survival-trace/evidence/safe13-boundary-check.json`). Protected controller
paths: `src/wanctl/wan_controller.py`, `src/wanctl/queue_controller.py`, `src/wanctl/cake_signal.py`,
`src/wanctl/alert_engine.py`, `src/wanctl/fusion_healer.py`, `src/wanctl/backends/`. Plus ATT config
`configs/att.yaml`. For EACH protected path run all three git state channels:
- committed diff: `git diff --numstat <anchor> HEAD -- <path>` (capture added/removed lines per file);
- staged diff: `git diff --numstat --staged -- <path>`;
- dirty tree: `git status --porcelain -- <path>` AND `git ls-files --others --exclude-standard -- <path>`
  (untracked).
Also compute per-file sha equality vs the anchor: compare `git rev-parse <anchor>:<file>` blob hash
(or `git hash-object` of the worktree file) to confirm byte-identity per protected file. Emit a JSON
record to `--out` matching the v1.48 shape: `anchor`, `baseline_commit` (`git rev-parse <anchor>`),
`head_commit` (`git rev-parse HEAD`), `protected_paths` (array), `per_path_diff` (object: per file
`{added, removed, lines}`), `per_file_sha256_equal` (object: per file bool), `dirty_tree`
(`{unstaged, staged, untracked, status_porcelain}`), `dirty_tree_clean` (bool), `committed_clean`
(bool), `controller_path_diff_count` (N), `att_config_diff_count` (M), `passed` (bool), `checked_at`
(iso8601). `passed` is true ONLY when committed diff == 0 across all protected paths AND
`dirty_tree_clean` is true AND every `per_file_sha256_equal` is true AND `att_config_diff_count` == 0
(fail-closed). Exit non-zero if `passed` is false. Keep it read-only on git (`git diff`,
`git status`, `git ls-files`, `git rev-parse`, `git hash-object`, `git cat-file` only — no checkout,
commit, reset, push, or add).
</action>
