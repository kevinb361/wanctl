# Plan 253-03 â€” Route Ownership Observability, Alerts, and Operator Output

<objective>
Expose route-owner mode, guard status, circuit-breaker status, startup reconciliation, last intended/applied route actions, and rollback readiness in steering health/operator/log/alert surfaces without changing existing payload semantics or enabling live mutation.
</objective>

<must_haves>
<truths>
- Operators must be able to see who currently owns route mutation during incidents.
- Health/operator output must make active route mutation blocked/allowed state visible.
- Observability must be additive; existing health payload keys and summary rows must not be removed or renamed casually.
- Logs/alerts for route decisions must include structured evidence, not just prose.
- Phase 253 still does not run live dry-run observation or active route canary.
</truths>
<outputs>
- Steering health JSON includes route ownership/management section.
- Compact `summary.rows` and `wanctl-operator-summary` show route owner/guard/circuit status.
- Route decision blocked/intended/applied events emit structured logs and alert records where alerting is enabled.
- Docs describe how to inspect route management status and confirm Netwatch remains interim owner until Phase 254.
</outputs>
</must_haves>

<threat_model>
T-253-11 invisible-owner-drift: during an incident, operators cannot tell whether Netwatch or wanctl is the route owner. Mitigation: health/operator output includes active owner and guard status.

T-253-12 hidden-blocked-active: config says active but guard/circuit blocks it, and operators see only generic healthy status. Mitigation: health output exposes `active_allowed`, `blocked_reason`, guard conflicts, reconciliation status, and circuit status.

T-253-13 alert-no-evidence: route decision alerts omit the signals that caused the decision. Mitigation: structured details include decision, route key/action, congestion state, RTT delta, drops, queue, guard status, and circuit status where available.

T-253-14 health-payload-breaking-change: existing health consumers break due to renamed keys. Mitigation: add new sections/fields only; preserve existing `steering`, `congestion`, `decision`, `summary` shape.

T-253-15 SAFE-19-observability-side-effect: adding observability triggers live route mutation or production config changes. Mitigation: observability reads in-memory/mock state only; no live RouterOS commands are added in this plan.
</threat_model>
