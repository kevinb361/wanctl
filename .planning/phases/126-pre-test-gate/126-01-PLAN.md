Create scripts/check-tuning-gate.sh -- a self-contained bash script that runs 5 gate checks. The script runs FROM the dev machine (not on the VM) and SSHes to cake-shaper as needed.

Per D-04: Script lives in the repo for reuse. Per D-06: 5 total checks.

**Constants at top of script:**
```
VM_HOST="kevin@10.10.110.223"
ROUTER_HOST="10.10.99.1"
ROUTER_USER="admin"
SPECTRUM_CONFIG="/etc/wanctl/spectrum.yaml"
HEALTH_URL="http://127.0.0.1:9101/health"
BRIDGE_NICS=(ens16 ens17 ens27 ens28)
```

**Script structure:**
1. Color variables matching deploy.sh pattern (RED, GREEN, YELLOW, BLUE, NC)
2. print_pass(), print_fail(), print_info() helper functions
3. Counter variables: PASS=0, FAIL=0, TOTAL=5

**Check 1 (GATE-01): CAKE qdiscs on all 4 bridge NICs**
- SSH to VM: `ssh $VM_HOST 'tc -s qdisc show'`
- For each NIC in BRIDGE_NICS: grep output for "qdisc cake" on that device
- PASS if all 4 found, FAIL with list of missing NICs otherwise

**Check 2 (GATE-02): No active CAKE on MikroTik router (per D-01)**
- SSH to VM to access router REST API (VM has network path to router):
  `ssh $VM_HOST "sudo grep -oP 'ROUTER_PASSWORD=\K.*' /etc/wanctl/secrets"` to get password
- Then from VM: `curl -sk -u admin:$PASS https://10.10.99.1/rest/queue/tree`
- Parse JSON: check if any queue tree entry has a queue type starting with "cake" AND is NOT disabled
- Also check: `curl -sk -u admin:$PASS https://10.10.99.1/rest/queue/type` for any active cake types
- Per D-03: Also check `curl -sk -u admin:$PASS https://10.10.99.1/rest/ip/firewall/mangle` for any mangle rules with "cake" in comment that are NOT disabled
- PASS if no active CAKE queue trees AND no active CAKE mangle rules, FAIL otherwise
- On FAIL: print the specific active entries found so user knows what to disable

**Check 3 (GATE-03): Rate change produces visible tc bandwidth change (per D-05)**
- Capture current CAKE bandwidth from `tc -s qdisc show dev ens16` (look for "bandwidth" line)
- Verify wanctl process is alive: `pgrep -f 'autorate_continuous.*spectrum'`
- Send SIGUSR1: `sudo kill -USR1 $(pgrep -f 'autorate_continuous.*spectrum')`
- Wait 1 second
- Confirm process still alive after SIGUSR1
- Capture bandwidth again from `tc -s qdisc show dev ens16`
- Verify bandwidth value is non-zero (meaning CAKE qdiscs are being actively shaped by wanctl)
- Print the observed bandwidth value for human review
- PASS if bandwidth is non-zero AND process survives SIGUSR1

**Check 4: Verify transport is linux-cake in spectrum.yaml**
- SSH to VM: `sudo grep 'transport:' $SPECTRUM_CONFIG`
- PASS if output contains "linux-cake"
- FAIL if output contains "rest" or "ssh" or anything else

**Check 5: Confirm health endpoint returns correct version**
- SSH to VM: `curl -s $HEALTH_URL`
- Parse JSON for "version" field
- PASS if version is present and non-empty (print the version for human review)
- FAIL if curl fails or no version in response

**Final summary:**
Print "GATE CHECK: X/5 passed" with appropriate color.
Exit 0 if all pass, exit 1 if any fail.

**Important bash patterns:**
- `set -euo pipefail` at top
- Use `#!/bin/bash` shebang
- `chmod +x` will be set by git
- Do NOT hardcode the router password -- read it from VM secrets file at runtime
- Use `jq` for JSON parsing (check availability with `command -v jq` on VM, fall back to python3 -c if not available)
