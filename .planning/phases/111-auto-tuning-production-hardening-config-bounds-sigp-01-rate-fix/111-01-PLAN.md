2. In `tune_hampel_sigma()`, replace lines 116-126 (the rate computation loop) with time-gap-aware calculation:

```python
    # 2. Sort timestamps, compute deltas between consecutive records
    sorted_ts = sorted(count_by_ts.keys())
    rates: list[float] = []

    for i in range(1, len(sorted_ts)):

```

Wait -- timestamps are integer keys. The correct code is:

```python
    sorted_ts = sorted(count_by_ts.keys())
    rates: list[float] = []
    samples_per_cycle = 1.0 / CYCLE_INTERVAL  # 20 at 50ms interval
    for i in range(1, len(sorted_ts)):
        delta = count_by_ts[sorted_ts[i]] - count_by_ts[sorted_ts[i - 1]]
        # Discard negative deltas (counter reset on daemon restart)
        if delta < 0:
            continue
        # Compute actual time gap in seconds between consecutive records
        time_gap = sorted_ts[i] - sorted_ts[i - 1]
        # Skip zero-gap records (duplicate timestamps)
        if time_gap <= 0:
            continue
        # Convert delta to rate: outliers per sample
        # Expected samples in this interval = time_gap * samples_per_cycle
        expected_samples = time_gap * samples_per_cycle
        rate = delta / max(expected_samples, 1.0)
        # Clamp to [0.0, 1.0] guard
        rate = max(0.0, min(1.0, rate))
        rates.append(rate)
```

This replaces the old `rate = delta / SAMPLES_PER_MINUTE` with `rate = delta / max(time_gap * samples_per_cycle, 1.0)`. The `samples_per_cycle` is computed from the existing `CYCLE_INTERVAL = 0.05` constant (1/0.05 = 20).
