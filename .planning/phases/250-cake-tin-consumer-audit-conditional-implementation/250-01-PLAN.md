# Plan 250-01 — CAKE Tin Consumer Audit + Conditional Implementation

## Objective

Audit every consumer of stored `wanctl_cake_tin_*` metrics and decide whether Phase B can safely change those metrics from continuous-series to sparse skip-on-unchanged series.

If every consumer is last-value-style, implement per-tin skip-on-unchanged and verify before/after write-rate behavior.

If any consumer needs continuous sampling, count-over-window, raw historical rows, or cumulative-counter delta semantics, defer TIN-02/TIN-03 to v1.55 and close this phase on the audit finding.

## Metrics in scope

- `wanctl_cake_tin_dropped`
- `wanctl_cake_tin_ecn_marked`
- `wanctl_cake_tin_delay_us`
- `wanctl_cake_tin_backlog_bytes`

## Classification rule

Classify each consumer as one of:

- producer — writes the stored metric;
- last-value-style — only needs most recent value;
- raw-history / continuous-series — displays or expects per-sample rows;
- count-over-window / delta-sensitive — computes or reasonably depends on sample continuity or cumulative counter deltas;
- test-only;
- doc-only.

## Gate

TIN-02 implementation is allowed only if every real consumer is last-value-style.

TIN-02/TIN-03 defer if any real consumer is raw-history, continuous-series, count-over-window, or delta-sensitive.

## Verification

Audit/defer path:

- Search output covers repo, docs, planning, JSON fixtures, scripts.
- Audit document cites producer and consumer paths.
- Relevant tests still pass unchanged.
- Live read-only checks prove current endpoint/CLI behavior shape.
- No source changes.

Implementation path, only if safe:

- Add cache keyed by metric/tin/labels.
- Always emit first sample.
- Emit subsequent samples only on value change.
- Add SimpleNamespace-style tests modeled after `TestSteeringEnabledFireOnChange`.
- Run before/after ingestion-rate measurement.
- Define rollback as code revert + service restart if deployed.

## Expected outcome

The likely outcome is defer, because `wanctl_cake_tin_dropped` and `wanctl_cake_tin_ecn_marked` are cumulative counter-shaped values and `wanctl-history --tins` exposes raw historical rows rather than last-value-before-T semantics.
