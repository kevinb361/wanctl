        `git diff --name-only "${ANCHOR_SHA}"..HEAD -- <file>` and accumulate any output;
        any nonempty result = fail

    Main flow:
      1. Parse args (--anchor, --out, --self-test)
      2. check_anchor_exists → resolve ANCHOR to ANCHOR_SHA via `git rev-parse "${ANCHOR}^{commit}"` (fail closed if missing)
      3. check_dirty_tree (across all 9 protected files)
      4. check_safe18 → capture changed file list across all 9 protected files
      5. If changed_files is empty: emit_evidence passed=true; exit 0
      6. Else: print changed files to stderr; emit_evidence passed=false; exit 1

    --self-test mode (copy Phase 245 worktree pattern — MUST use an isolated temp worktree;
    NEVER mutate the main worktree):
      - git worktree add to a temp dir at a detached HEAD
      - Make a one-line comment edit to src/wanctl/queue_controller.py in the temp worktree
      - git -C <worktree> commit -am "self-test: disallowed edit (will be discarded)"
      - Run the verifier against the temp worktree (adjust OUT to a temp path so the
        real evidence/safe18-boundary-247.json is not overwritten by the self-test)
      - Assert script exits non-zero (disallowed edit detected)
      - git worktree remove --force (always clean up, even on failure path)
      - Report PASS/FAIL
      - Exit 0 if the self-test correctly detected the violation (i.e., the verifier
        returned non-zero as expected); exit 1 if the verifier unexpectedly returned 0
        (failed to detect the violation)

    Script must be executable: chmod +x scripts/phase247-safe18-boundary-check.sh
