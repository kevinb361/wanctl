<objective>
Write the fping shadow capture script and its unit tests. The script runs standalone
on cake-shaper, importing FpingMeasurement/FpingThread directly from /opt/wanctl/src,
reading configs/spectrum.yaml for reflectors and source IP, and logging NDJSON per-sample
plus periodic probe_stats snapshots (every 100 polls) to avoid the OperationProfiler
max_samples=1200 rollover problem over a 12h soak.

Purpose: PROF-01 — run fping in shadow alongside the live icmplib backend without touching
any controller-path file. The NDJSON output is the evidence input for Phase 248 distribution
analysis. Two correctness requirements drive this plan: (1) the Spectrum config's root-level
`ping_source_ip` MUST be mapped to the `source_ip` constructor key FpingMeasurement expects
(wrong key → wrong source interface → invalid evidence), and (2) the periodic probe_stats
snapshot series MUST be sufficient to reconstruct the full 12h p99 distribution, because the
OperationProfiler rolling window only holds the last ~3.3h of samples.

Output:
  scripts/phase247-fping-shadow.py
  tests/test_phase247_shadow_script.py
