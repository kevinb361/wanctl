<objective>
Write the fping shadow capture script and its unit tests. The script runs standalone
on cake-shaper, importing FpingMeasurement/FpingThread directly from /opt/wanctl/src,
reading configs/spectrum.yaml for reflectors and source IP, and logging:
  1. A run_start metadata record on startup (provenance: source_ip, reflectors, config path,
     script version/commit, cadence settings).
  2. Per-burst probe_cycle records for every completed burst — success OR failure — providing
     the raw per-burst RTT distribution that Phase 248 needs for a full-window p99.
  3. Periodic probe_stats summary snapshots (every STATS_INTERVAL_PROBES bursts) as a
     convenience summary layer; these are NOT the authoritative p99 source.

Purpose: PROF-01 — run fping in shadow alongside the live icmplib backend without touching
any controller-path file. The NDJSON output is the evidence input for Phase 248 distribution
analysis.

The core correctness requirements driving this plan:
  (HIGH-fix) Per-burst probe_cycle records are the authoritative evidence: the OperationProfiler
  has max_samples=1200 (~3.3h at 10s cadence), so a 12h soak's hours 0-8.7 are NOT in the
  final rolling-window stats. probe_cycle records written per burst are always available for
  the full soak duration. Phase 248 MUST compute p99 from probe_cycle records, not from
  probe_stats snapshots.
  (MEDIUM-1) Failed/all-loss cycles must be logged as first-class probe_cycle records so
  failure periods do not bias the p99 interpretation.
  (MEDIUM-2) A run_start metadata record makes evidence provenance self-contained in the
  NDJSON file without requiring the config file to reconstruct what ran.
  (MEDIUM-3) cfg['ping_source_ip'] MUST be mapped to the 'source_ip' constructor key
  FpingMeasurement expects (wrong key → wrong source interface → invalid evidence).

    IMPORTANT: returns the CACHED last successful sample. Polling on cadence can return the

    The script must dedup by sample.timestamp to avoid double-counting.
    NOTE: FpingMeasurement.probe() returns None on all-loss — FpingThread caches only
    successful results. All-loss cycles are NOT reflected in get_latest() returning None
    after a prior success; the prior cached result remains. The script detects all-loss
    by tracking the expected next-burst wall time: if cadence_sec+grace has elapsed with
    no new sample.timestamp, a dropped/all-loss cycle is inferred.
  FpingThread.get_profile_stats() → dict with keys: count, min_ms, max_ms, avg_ms, p95_ms, p99_ms, samples
    NOTE: "count" here is the OperationProfiler ROLLING-WINDOW count (capped at max_samples=1200),
    NOT the cumulative number of probes since script start.
    IMPORTANT: This rolling window covers only the last ~3.3h at 10s cadence. For a 12h soak,
    get_profile_stats() at the end covers hours 8.7-12 only. Do NOT use probe_stats p99_ms
    as the full-soak p99 in Phase 248.
