  <behavior>
    ## Dwell Timer (HYST-01, HYST-03)

    - GREEN->YELLOW transition gated by dwell timer: requires `dwell_cycles` (default 3) consecutive above-threshold cycles
    - During dwell (delta > threshold but counter < dwell_cycles): zone remains GREEN, rates hold steady (no decay, per D-01)
    - Dwell counter `_yellow_dwell` increments each cycle delta exceeds threshold while in GREEN
    - Dwell counter resets to zero when delta drops below threshold mid-dwell (HYST-03)
    - Once `_yellow_dwell >= dwell_cycles`, zone transitions to YELLOW normally
    - RED transitions bypass dwell entirely -- immediate on 1 sample (D-02)
    - SOFT_RED keeps its existing `soft_red_required` sustain counter unchanged (D-02)

    ## Deadband Margin (HYST-02)

    - Asymmetric: only YELLOW->GREEN recovery uses deadband (D-03)
    - GREEN->YELLOW entry uses target_delta as-is (no deadband on entry)
    - YELLOW->GREEN recovery requires delta < (target_delta - deadband_ms) instead of delta <= target_delta
    - Default deadband_ms=3.0 creates hysteresis band: enter YELLOW at target_delta, exit at (target_delta - 3.0)
    - When delta is in the deadband range (target_delta - deadband_ms <= delta <= target_delta), zone stays YELLOW if already YELLOW

    ## Direction Parity (HYST-04)

    - adjust() (3-state, upload) and adjust_4state() (4-state, download) apply identical dwell+deadband logic
    - Each QueueController instance has its own _yellow_dwell counter (one per direction)

    ## Constructor Changes

    - New optional params: dwell_cycles: int = 3, deadband_ms: float = 3.0
    - New instance var: self._yellow_dwell: int = 0
    - Existing fixtures get new params via defaults (backward compatible)

    ## QueueController.adjust() changes (3-state):
    Replace the zone classification block (lines ~1385-1400) with state-dependent logic:

    1. Compute delta = load_rtt - baseline_rtt
    2. If delta > warn_delta: zone = RED, reset _yellow_dwell = 0, reset green_streak = 0
    3. Elif delta > target_delta:
       a. If _last_zone in ("GREEN",) or (_last_zone == "YELLOW" and this is dwell-held GREEN):
          - Increment _yellow_dwell
          - If _yellow_dwell >= dwell_cycles: zone = YELLOW (dwell satisfied)
          - Else: zone = GREEN (hold during dwell, rates hold steady per D-01)
       b. Else (_last_zone == "YELLOW"): zone = YELLOW (already transitioned)
       c. Reset green_streak = 0, red_streak = 0
    4. Elif _last_zone == "YELLOW" and delta > (target_delta - deadband_ms):
       - zone = YELLOW (deadband: stay YELLOW, delta hasn't dropped far enough)
       - Reset _yellow_dwell = 0, green_streak = 0
    5. Else: zone = GREEN, _yellow_dwell = 0, green_streak += 1, red_streak = 0

    Rate adjustment logic: When zone is GREEN due to dwell hold (step 3a else branch), the code falls through to "GREEN but not sustained -> hold steady" -- rates naturally hold. No special rate logic needed.

    ## QueueController.adjust_4state() changes (4-state):
    Mirror the same dwell+deadband logic at the GREEN->YELLOW boundary (lines ~1470-1508):

    1. raw_state classification stays the same (RED/SOFT_RED/YELLOW/GREEN by thresholds)
    2. Add dwell gate: when raw_state == "YELLOW" and _last_zone == "GREEN":
       - Increment _yellow_dwell
       - If < dwell_cycles: zone = GREEN (hold)
       - Else: zone = YELLOW
    3. Add deadband: when raw_state == "GREEN" and _last_zone == "YELLOW":

       - Else: zone = GREEN (recovered)
    4. RED and SOFT_RED paths unchanged (bypass dwell per D-02)
    5. Reset _yellow_dwell on RED, SOFT_RED, or GREEN (below deadband)
