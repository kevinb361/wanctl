# Phase 262, Plan 01 — Abort Scaffolding Code

**Goal:** Wire automatic abort-to-Netwatch in RouteManager, add trip condition checks to the daemon cycle, and provide a manual rollback entry point.

**Requirements:** ABORT-02 (auto-revert on trip), ABORT-04 (manual one-command rollback)

**SAFE-22:** May modify: route_manager.py, daemon.py, health.py, steering.yaml, scripts/. Must NOT modify: wan_controller.py, queue_controller.py, cake_signal.py, RTT backends, alert_engine.py, fusion.

## Task 1: Add auto-revert method to RouteManager

**What:** Add `abort_to_netwatch()` method to RouteManager that:
1. Sets mode from "active" → "dry_run"
2. For each configured route, re-enables the Netwatch-owned route and disables the wanctl-owned route
3. Resets circuit breaker to RouteCircuitBreaker() defaults
4. Records the abort as a RouteActionRecord (action="abort", reason=trip_condition)
5. Returns a RouteActionResult with evidence

**Files:** `src/wanctl/steering/route_manager.py`

**Steps:**
1. Add `abort_to_netwatch(trip_condition: str) -> RouteActionResult` method
2. The method iterates self.routes and for each route:
   - Constructs a "enable" command for the route (re-enabling Netwatch ownership)
   - Executes via self.router_client.run_cmd()
   - Records success/failure per route
3. Sets self.mode = "dry_run"
4. Resets self.circuit_breaker = RouteCircuitBreaker()
5. Records the overall abort as last_intended_action and last_applied_action
6. Returns RouteActionResult with abort evidence

**Verification:**
- Unit test the method with a mock RouteMutationClient
- Verify mode changes to "dry_run"
- Verify circuit breaker resets
- Verify route enable/disable commands are sent
- Verify abort is recorded in last_intended_action

## Task 2: Add trip condition check to daemon cycle

**What:** Add a method to the steering daemon that checks the 3 trip conditions and triggers abort if any fires:
1. Circuit breaker open: `self.route_manager.circuit_breaker.open`
2. Router unreachable: `not self.router_connectivity.is_reachable`
3. Netwatch contention: ownership_inspector detects Netwatch route re-enabled

**Files:** `src/wanctl/steering/daemon.py`

**Steps:**
1. Add `_check_abort_trip_conditions() -> str | None` method that returns the trip condition name or None
2. Add `_handle_abort(trip_condition: str)` method that calls `self.route_manager.abort_to_netwatch(trip_condition)` and logs the event
3. Integrate into run_cycle: after `_run_steering_state_subsystem`, check trip conditions when mode is "active"
4. Only check when route_management is enabled and mode is "active"

**Verification:**
- Test each trip condition independently
- Verify abort is triggered on any single condition
- Verify abort is NOT triggered when mode is "dry_run" or "off"
- Verify logging of trip condition and abort result

## Task 3: Add manual rollback entry point

**What:** Provide a manual one-command rollback. The simplest approach: a script that changes steering.yaml mode from "active" to "dry_run" and restarts steering. The daemon detects mode change and executes the revert sequence.

**Files:** `scripts/phase262-rollback.sh`

**Steps:**
1. Create `scripts/phase262-rollback.sh` — a standalone script that:
   - SSHs to the target host
   - Changes `mode: active` to `mode: dry_run` in `/etc/wanctl/steering.yaml`
   - Restarts `steering.service`
   - Verifies via :9102 that mode is "dry_run" and active_owner is "netwatch"
2. Add a daemon-side `_handle_mode_change()` that detects mode transition from "active" to "dry_run" and calls `abort_to_netwatch("manual_rollback")`
3. Make the script deployable by deploy.sh

**Verification:**
- Script is syntactically valid (bash -n)
- Script is ruff/bash-compliant
- Daemon mode change detection works
- Manual rollback triggers the same revert path as auto-abort
