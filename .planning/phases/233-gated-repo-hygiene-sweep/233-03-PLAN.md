<objective>
Make the Spectrum cake-autorate state-bridge systemd unit set its `WANCTL_EXTERNAL_*` identity,
path, and baseline env explicitly — mirroring the already-explicit ATT sibling unit — so Spectrum
identity stops being load-bearing implicit defaults in the shared bridge script. Values must equal
the current script defaults, so this is behavior-preserving with zero new abstraction (the only
exception is if the operator deliberately pins a different baseline, which is recorded as intentional drift).

Purpose: SWEEP-03 — strip Spectrum-only hardcoding where a generic `$wan` pattern already exists.
The two bridge scripts are byte-identical and fully env-driven; the only Spectrum-only hardcoding
is that the Spectrum unit leans on the script's Spectrum-flavored fallbacks while ATT is explicit.
Output: Spectrum unit made explicit (mirrors ATT); repo-only edit (live apply is out of scope and
requires an operator-gated redeploy + daemon-reload).
</objective>
