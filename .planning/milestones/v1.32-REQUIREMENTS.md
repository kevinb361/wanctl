# Requirements: wanctl v1.32

**Defined:** 2026-04-09
**Core Value:** Sub-second congestion detection with 50ms control loops

## v1.32 Requirements

Requirements for CAKE-Aware Congestion Detection milestone. Each maps to roadmap phases.

### Signal Infrastructure

- [ ] **CAKE-01**: CAKE qdisc stats are read every cycle via netlink with <1ms latency per direction
- [ ] **CAKE-02**: Drop rate is computed as EWMA-smoothed drops/sec with u32-safe delta math and cold start warmup
- [ ] **CAKE-03**: Per-tin stats are tracked separately (Bulk drops distinguished from BestEffort+ drops)
- [ ] **CAKE-04**: CAKE signal metrics (drop rate, backlog, peak delay) are exposed via health endpoint and stored in metrics DB
- [ ] **CAKE-05**: All CAKE signal features are independently toggleable via YAML config (default: disabled)

### Congestion Detection

- [ ] **DETECT-01**: Drop rate above threshold bypasses dwell timer to confirm congestion immediately
- [ ] **DETECT-02**: Queue backlog above threshold suppresses green_streak to prevent premature rate recovery
- [ ] **DETECT-03**: Refractory period prevents feedback loop oscillation after drop-triggered rate reduction
- [ ] **DETECT-04**: Only BestEffort and higher-priority tin drops drive rate decisions (Bulk tin drops excluded)

### Adaptive Recovery

- [ ] **RECOV-01**: Rate recovery uses exponential probing (1.5x step multiplier) guarded by CAKE signals
- [ ] **RECOV-02**: Probing reverts to linear step_up above 90% of ceiling to prevent overshoot
- [ ] **RECOV-03**: Probe multiplier resets immediately on any non-GREEN zone transition

## Out of Scope

- ECN-based rate control (DOCSIS CM/CMTS strips CE bits — unreliable)
- Weighted multi-signal fusion score (impossible to A/B test independently)
- Per-flow stats tracking (CAKE handles flow fairness internally)
- Peak delay as primary zone classifier (circular dependency with rate setting)
- Per-tin rate limiting (architecturally complex, defer to future milestone)
- Time-of-day tuning segmentation (independent feature, defer to v1.33)
