<objective>
Stand up the offline replay harness (PROOF-01). The harness drives the
v1.47-close `SteeringDaemon` decision pipeline through a fixture corpus
using a fake `RouterOSController` and tempdir-redirected state files. It
produces per-fixture verdicts (`matches | diverges | inconclusive`) and the
operator-facing `evidence/replay-results.{json,md}` artifacts.

**Class name correction (REVIEWS.md LOW):** The class is `SteeringDaemon` at
`src/wanctl/steering/daemon.py:1122` — NOT `WANSteeringDaemon`. Plan text
elsewhere may still say `WANSteeringDaemon`; the executor must use the
actual class name discovered during Task 01's seam inventory.

**Fake router API correction (REVIEWS.md HIGH #1):** The fake MUST mimic
the actual daemon-facing surface: `get_rule_status() -> bool | None`,
`enable_steering() -> bool`, `disable_steering() -> bool` (verified in
`src/wanctl/steering/daemon.py:805–920`). There is no `set_rule_status()`
method on `RouterOSController`. A fake with the wrong surface would test
an adapter rather than the daemon's real routing path.

**Confidence-mode timing correction (REVIEWS.md HIGH #2):** Production
`configs/steering.yaml` runs `use_confidence_scoring: true` with
`sustain_duration_sec: 2.0`, `recovery_sustain_sec: 3.0`, and
`hold_down_duration_sec: 30.0` against `ASSESSMENT_INTERVAL_SECONDS = 0.05`
(`daemon.py:127`). The harness MUST EITHER (a) explicitly disable
confidence scoring (`SteeringConfig.use_confidence_scoring=False`) and
author hysteresis-only fixtures with the cycle counts originally drafted,
OR (b) keep confidence scoring on and derive every fixture's cycle counts
from config (sustain → ≥40 cycles, recovery → ≥60 cycles, hold-down → 600
cycles). The mode choice is recorded in `evidence/README.md` and on every
fixture's `harness_mode` field. Hysteresis-only mode is the recommended
default because it isolates the spine-contract verdicts from confidence
controller timing.

The harness must NOT touch the live router, NOT write to production state
file paths, and NOT mutate controller-path source (SAFE-12). The preferred
implementation strategy is constructor injection through existing seams in
`SteeringDaemon` (`SteeringConfig`, `SteeringStateManager`,
`RouterOSController`, `BaselineLoader`). Any new steering-source seam
requires an explicit named, scoped edit documented in `evidence/README.md`
with a regression test under `tests/steering/`.

Plan 02 owns the clean-restart-degraded fixture (PROOF-02) and appends its
results into the same `evidence/replay-results.json` artifact. Plan 03 owns
PROOF-03 spine evidence + SAFE-12 boundary check.
</objective>
