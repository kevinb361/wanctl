| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-222-02-01 | T | per-commit contract verdict | mitigate | Every invariant verdict carries a `note` string naming the lines / function inspected; the verdict derivation rule (`preserves` / `breaks` / `ambiguous`) is enforced in the verify step. |

| T-222-02-03 | T | disposition assignment | mitigate | `breaks` MUST map to `no-go`; `behavior-preserving + preserves` MUST map to `go`; deviations from these rules are not permitted by the verify-step assertion. |
| T-222-02-04 | D | downstream phase ownership | mitigate | Every `mitigate` finding names the owner phase (typically 223 staging proof or 224 canary); ambiguous ownership is not allowed. |
</threat_model>
