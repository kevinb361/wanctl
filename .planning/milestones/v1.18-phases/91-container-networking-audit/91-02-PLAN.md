Steps:
1. Verify containers are reachable: ping -c 1 10.10.110.246 and ping -c 1 10.10.110.247
2. If both reachable, run: python scripts/container_network_audit.py --count 1000 --interval 0.01 --runs 5 --output docs/CONTAINER_NETWORK_AUDIT.md
3. If a container is unreachable, try SSH name instead: ping -c 1 cake-spectrum / ping -c 1 cake-att -- if DNS resolves differently, update the script CONTAINERS dict before running
4. The script takes approximately 2 minutes (5 runs x 10s per container x 2 containers + overhead)
5. After generation, read docs/CONTAINER_NETWORK_AUDIT.md and verify it contains all required sections

If containers are unreachable from the dev machine (e.g., different network segment), use --dry-run mode to generate a report with synthetic data and add a note at the top: "Generated with synthetic data -- re-run from a machine with container network access for real measurements."

Expected results based on preliminary measurements from RESEARCH.md:
- cake-spectrum: mean ~0.2ms, stdev ~0.06ms
- cake-att: mean ~0.17ms, stdev ~0.03ms
- Both well under 0.5ms threshold
- Jitter should be NEGLIGIBLE (container stdev / WAN idle jitter is far below 10%)
