    - Test: Drift below threshold does NOT fire (e.g., 30% drift with 50% threshold)

  <action>
    **Tests first** in `tests/test_anomaly_alerts.py`:
    Follow the same fixture pattern as `test_congestion_alerts.py`. Config dict includes alerting with rules:
    ```python
    "baseline_drift": {"enabled": True, "cooldown_sec": 600, "severity": "warning", "drift_threshold_pct": 50},
    ```

    **Implementation** in `src/wanctl/autorate_continuous.py`:

    1. Add `_check_baseline_drift(self)` method after `_check_congestion_alerts`:
       - Compare `self.baseline_rtt` against `self.baseline_rtt_initial`
       - Calculate `drift_pct = abs(self.baseline_rtt - self.baseline_rtt_initial) / self.baseline_rtt_initial * 100`
       - Get threshold from per-rule config: `self.alert_engine._rules.get("baseline_drift", {}).get("drift_threshold_pct", 50)`
       - If `drift_pct >= threshold`: fire baseline_drift with severity="warning"
       - Details: `{"current_baseline_ms": round(self.baseline_rtt, 2), "reference_baseline_ms": round(self.baseline_rtt_initial, 2), "drift_percent": round(drift_pct, 1)}`
       - No timer needed -- cooldown suppression handles re-fire naturally

    2. Call `self._check_baseline_drift()` in `run_cycle()` after `_check_congestion_alerts()` (around line 1651). This is a cheap comparison each cycle.

    Design note: Using `baseline_rtt_initial` (the config value) as the reference point. This is stable, never changes during runtime, and represents the ISP's expected baseline. The EWMA `baseline_rtt` is what drifts. Default threshold 50% is conservative -- for Spectrum at 37ms, that means 55.5ms+ triggers an alert. Per-rule override allows tighter thresholds.

    1. Add state variables in `WANController.__init__` after connectivity timer block:
       ```python
       # CONGESTION FLAPPING DETECTION (ALRT-07)
       self._dl_zone_transitions: collections.deque[float] = collections.deque()
       self._ul_zone_transitions: collections.deque[float] = collections.deque()

       ```

    2. Add `_check_flapping_alerts(self, dl_zone: str, ul_zone: str)` method:
       - For each direction (DL, UL):
         - If zone changed from previous: append `time.monotonic()` to transitions deque
         - Prune entries older than flap_window_sec (default 60s, per-rule override)
         - If len(transitions) >= flap_threshold (default 6, per-rule override): fire flapping_dl/flapping_ul
         - Update _prev_zone
       - flapping details: `{"transition_count": len(transitions), "window_sec": flap_window_sec, "current_zone": zone}`
       - Severity: "warning" (configurable via per-rule severity)

    Design note: This is a simplified version of the FlapDetector pattern from steering_confidence.py. The steering FlapDetector uses TimerState and applies penalty thresholds -- here we just need transition counting in a time window and alert firing. Using a simple deque of monotonic timestamps per direction is lighter weight and sufficient. Default 6 transitions in 60s means toggling every 10s or faster triggers the alert.
