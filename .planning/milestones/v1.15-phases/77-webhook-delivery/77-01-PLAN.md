<task type="auto" tdd="true">
  <name>Task 2: WebhookDelivery class with retry, rate-limit, threading, and schema migration</name>
  <files>src/wanctl/webhook_delivery.py, src/wanctl/storage/schema.py, tests/test_webhook_delivery.py</files>
  <behavior>
    - WebhookDelivery.deliver() submits delivery to a background thread (does not block caller)
    - HTTP POST to webhook_url with JSON payload from formatter
    - Retries on 5xx/timeout with exponential backoff (max 3 attempts, 2s initial, 2x factor)
    - Does NOT retry on 4xx (permanent failure)
    - Rate limiter rejects delivery when exceeding max_webhooks_per_minute (default 20)
    - Rate-limited deliveries logged as warning (dropped, not queued)
    - delivery_status updated in SQLite: "pending" on fire, "delivered" on success, "failed" on exhaustion
    - WebhookDelivery constructed with formatter, webhook_url, rate_limit config, writer (optional)
    - deliver() with empty/None webhook_url silently returns (no HTTP call, no error)
    - Delivery errors never crash -- all exceptions caught and logged as warnings
    - ALERTS_SCHEMA updated to include delivery_status TEXT DEFAULT 'pending'
  </behavior>
  <action>
Add to `src/wanctl/webhook_delivery.py`:

1. `WebhookDelivery` class:
   - Constructor: `(formatter: AlertFormatter, webhook_url: str, max_per_minute: int = 20, writer: MetricsWriter | None = None)`
   - Store a `RateLimiter(max_changes=max_per_minute, window_seconds=60)` internally
   - `deliver(alert_id: int | None, alert_type: str, severity: str, wan_name: str, details: dict)` method:
     a. If webhook_url is empty/None, return immediately (silent skip)
     b. If rate_limiter.can_change() is False, log warning "Webhook rate limited, dropping delivery for {alert_type}" and return
     c. Record the rate limit event: rate_limiter.record_change()
     d. Submit `_do_deliver(...)` to a `threading.Thread(daemon=True)` so it does not block the 50ms cycle
   - `_do_deliver(alert_id, alert_type, severity, wan_name, details)` internal method:
     a. Call `self._formatter.format(alert_type, severity, wan_name, details, mention_role_id=self._mention_role_id, mention_severity=self._mention_severity)` to get payload
     b. POST to webhook_url with `requests.post(url, json=payload, timeout=10)`
     c. Raise `requests.HTTPError` on 4xx/5xx via `response.raise_for_status()`
     d. On success: update delivery_status to "delivered" in SQLite (if writer and alert_id)
     e. On retryable failure (5xx/timeout): retry up to 3 times with backoff (use inline retry loop, not decorator -- cleaner for thread context)
     f. On permanent failure (4xx) or retry exhaustion: update delivery_status to "failed", log warning
     g. Increment `self._delivery_failures` counter (int, for health endpoint)
     h. All exceptions caught at outer level -- never propagate out of thread
   - `update_webhook_url(url: str)` method for SIGUSR1 reload -- validates https:// prefix, logs warning if invalid
   - `delivery_failures` property returning the counter value
   - `_update_delivery_status(alert_id: int, status: str)` helper -- UPDATE alerts SET delivery_status=? WHERE id=?. Catches exceptions, logs warning.

2. Update `src/wanctl/storage/schema.py`:
   - Add `delivery_status TEXT DEFAULT 'pending'` column to ALERTS_SCHEMA CREATE TABLE
   - This is safe for existing databases because CREATE TABLE IF NOT EXISTS won't alter existing tables, but new databases get the column. For existing databases, a migration helper or ALTER TABLE can be added if needed (mark as note for executor).

Also add constructor params: `mention_role_id: str | None = None`, `mention_severity: str = "critical"`.

Write tests first (RED), then implement (GREEN). Mock `requests.post` in tests. Use `unittest.mock.patch` for threading to test synchronously.
