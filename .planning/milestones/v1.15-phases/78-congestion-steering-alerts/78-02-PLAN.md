```python
details = {
```

```python
# Fire steering_recovered alert
duration_sec = None
if self._steering_activated_time is not None:
    duration_sec = round(time.monotonic() - self._steering_activated_time, 1)

    "from_state": self.config.state_degraded,
    "to_state": self.config.state_good,
    "duration_sec": duration_sec,
}
self.alert_engine.fire(
    "steering_recovered", "recovery", self.config.primary_wan, details
)
self._steering_activated_time = None
```

**Import:** Add `import time` at top of file if not already present.

**Alert types:**
- `steering_activated`: default severity "warning" (routing change is noteworthy but not an error)
- `steering_recovered`: default severity "recovery" (green in Discord -- consistent with congestion recovery)

**Severity is hardcoded in the fire() call** (not configurable per-rule severity override). Per-rule severity override in AlertEngine rules dict allows operators to override if desired, but the code passes the sensible default.

**Duration tracking approach:** Use `time.monotonic()` stored at activation time. On recovery, compute elapsed time. This is simpler and more accurate than parsing the ISO timestamp from state file. The `_steering_activated_time` resets to None on recovery.

**No additional delay on alert firing:** Alerts fire immediately after successful transition -- existing hysteresis in the state machine (degrade_threshold / recover_threshold samples) already provides delay.
