**WANController.__init__ changes (after alert_engine init, ~line 1127):**
- Add `self._dl_congestion_start: float | None = None` -- monotonic timestamp when DL entered RED/SOFT_RED
- Add `self._ul_congestion_start: float | None = None` -- same for UL

**run_cycle() changes (after `self._dl_zone = dl_zone` and `self._ul_zone = ul_zone`, ~line 1610):**
Insert a call to `self._check_congestion_alerts(dl_zone, ul_zone, dl_rate, ul_rate, delta)` right after zone assignment and before the log decision line.

**New method `_check_congestion_alerts(self, dl_zone, ul_zone, dl_rate, ul_rate, delta)`:**

For each direction (DL, UL) with its zone, congestion_start, sustained_fired:
1. Determine if zone is "congested" (RED or SOFT_RED for DL; RED for UL -- UL has 3-state so only RED is congested)
2. If congested and timer is None: set timer to `time.monotonic()`
3. If congested and timer is set: check if `(now - timer) >= sustained_sec`. If so and not already fired:
   - Look up per-rule sustained_sec override: `self.alert_engine._rules.get(alert_type, {}).get("sustained_sec", self._sustained_sec)`
   - Determine severity: RED = "critical", SOFT_RED = "warning"
   - Fire: `self.alert_engine.fire(alert_type, severity, self.wan_name, details)` where details = {"zone": zone, "dl_rate_mbps": dl_rate/1e6, "ul_rate_mbps": ul_rate/1e6, "rtt_ms": self.load_rtt, "delta_ms": delta, "duration_sec": round(now - start, 1)}`
   - If fire() returns True: set sustained_fired = True
4. If NOT congested and timer is not None (zone cleared):
   - If sustained_fired: fire recovery alert `congestion_recovered_{dl|ul}` with severity="recovery", details including zone that was active, duration, current rates
   - Reset timer to None, sustained_fired to False

**Alert types (6 total, all enabled by default per Claude's discretion -- operator opted into alerting):**
- `congestion_sustained_dl`: severity from zone (critical/warning)
- `congestion_sustained_ul`: severity from zone (critical)
- `congestion_recovered_dl`: severity "recovery"
- `congestion_recovered_ul`: severity "recovery"

Note: UL only has 3-state (GREEN/YELLOW/RED), so SOFT_RED is not applicable to UL. UL congested = RED only.

**Recovery gate (Claude's discretion):** Recovery alert only fires if sustained alert actually fired first (the `_sustained_fired` bool ensures this). This avoids spurious "recovered" alerts for brief congestion that never reached the sustained threshold.

**YELLOW handling (Claude's discretion):** YELLOW counts as "recovered" (not congested). YELLOW is a transitional state showing mild degradation -- it's not RED/SOFT_RED sustained congestion.

**Re-fire behavior (Claude's discretion):** Alert fires once per episode, then enters normal cooldown. If still congested when cooldown expires, re-checks duration and fires again. This is natural behavior from AlertEngine cooldown -- no special code needed.
