  <name>Task 1: Add _load_wan_state_config() with validation and startup logging</name>
  <files>src/wanctl/steering/daemon.py, tests/test_steering_daemon.py</files>
  <behavior>
    - Default: absent wan_state section -> wan_state_config = None, startup log "WAN awareness: disabled (enable via wan_state.enabled)"
    - Default: wan_state.enabled: false -> wan_state_config dict with enabled=False, startup log same as absent
    - Enabled: wan_state.enabled: true -> wan_state_config dict with all fields populated, startup log "WAN awareness: enabled (grace period: 30s, red_weight: 25, wan_override: false)"
    - Enabled but no autorate state file -> one-time warning (test separately in plan 02 at runtime)
    - Wrong type for enabled (e.g., string) -> warn, wan_state_config = None
    - Wrong type for red_weight (e.g., string) -> warn, wan_state_config = None
    - Out-of-range red_weight (e.g., 200) -> clamped to steer_threshold - 1 with warning "red_weight clamped to {max} (must be < steer_threshold)"
    - wan_override: true removes weight ceiling (red_weight >= steer_threshold allowed)
    - wan_override: true + enabled: false -> validation warning "override has no effect when disabled"
    - Unknown keys -> warning "wan_state: unrecognized keys {set} (possible typo)"
    - soft_red_weight derived as int(red_weight * 0.48), not independently configurable
    - staleness_threshold_sec defaults to 5, grace_period_sec defaults to 30
    - wan_state_config dict contains: enabled, red_weight, soft_red_weight, staleness_threshold_sec, grace_period_sec, wan_override
  </behavior>
  <action>
    Create `_load_wan_state_config()` method on SteeringConfig. Follow the `_load_confidence_config()` pattern (daemon.py:240-291) but with warn+disable instead of raise on error.

    **Do NOT add wan_state fields to SCHEMA class attribute** -- that would crash the daemon on invalid input. Validate manually in the method with try/except.

    Implementation steps:
    1. Add `_load_wan_state_config(self) -> None` method after `_load_confidence_config()` (~line 292).
    2. Read `wan_state = self.data.get("wan_state", {})`. If empty dict or missing, set `self.wan_state_config = None` and log disabled message.
    3. Check `enabled` field: `wan_state.get("enabled", False)`. Type-check (must be bool). If wrong type, warn + set None + return.
    4. Check for unknown keys: `known_keys = {"enabled", "red_weight", "staleness_threshold_sec", "grace_period_sec", "wan_override"}`. Warn on extras.
    5. Validate each numeric field with type checks. On wrong type for any field, warn + set None + return (disable entire feature).
    6. Load `red_weight` (default 25, must be int, min 1). Get `steer_threshold` from `self.data.get("confidence", {}).get("steer_threshold", 55)`.
    7. Clamping: if `wan_override` is False (or absent) and `red_weight >= steer_threshold`, clamp to `steer_threshold - 1` with warning.
    8. If `wan_override` is True, no clamping (allow red_weight >= steer_threshold).
    9. Derive `soft_red_weight = int(red_weight * 0.48)`.
    10. Cross-field: if `wan_override=True` and `enabled=False`, warn "override has no effect when disabled".
    11. If `wan_override=True` and `enabled=True`, startup warning "WAN override active -- WAN RED can trigger failover independently of CAKE signals".
    12. Build `self.wan_state_config` dict with all fields. Store clamped values in dict (do NOT mutate self.data).
    13. Add call `self._load_wan_state_config()` in `_load_specific_fields()` after `self._load_confidence_config()` (line 390).
    14. Startup logging: after building config, log appropriate message per CONTEXT.md decisions.

    For tests, create `TestWanStateConfig` class in test_steering_daemon.py. Use the existing `valid_config_dict` fixture as base, adding wan_state section. Fixture pattern: create helper that returns a config dict with wan_state, mock the YAML loading.
