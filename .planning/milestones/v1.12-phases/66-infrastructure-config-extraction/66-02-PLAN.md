Create `tests/test_deployment_contracts.py` with:

**Shared helpers at module top:**
1. `_load_pyproject()` -- uses `tomllib` (stdlib 3.11+) to parse `pyproject.toml` from project root (use `Path(__file__).parent.parent / "pyproject.toml"`)
2. `_parse_dependency(dep_str)` -- parses "requests>=2.31.0" into (package_name, operator, version). Handle the `>=` operator (the only one used in this project). Use `re.match(r"^([a-zA-Z0-9_-]+)([><=!]+)(.+)$", dep_str)`.
3. `_load_dockerfile()` -- reads `docker/Dockerfile` from project root, returns contents as string
4. `_extract_pip_install_deps(dockerfile_text)` -- extracts package specs from the `pip install --no-cache-dir` block. Parse the continuation-line block (lines ending with `\` after `pip install`). Return list of "package>=version" strings.
5. Map of pyproject package names to Python import names for packages where they differ: `{"pyyaml": "yaml", "icmplib": "icmplib"}` (most match).

**Class TestDockerfileDependencyContract:**
1. `test_all_pyproject_deps_in_dockerfile` -- every package name from pyproject.toml dependencies appears in Dockerfile pip install block
2. `test_version_specs_match` -- for each dep, the version spec in Dockerfile matches pyproject.toml exactly
3. `test_label_version_matches_pyproject` -- `LABEL version=` in Dockerfile matches `[project].version` in pyproject.toml
4. `test_copy_paths_resolve_to_files` -- for each COPY source glob in Dockerfile (src/wanctl/*.py, src/wanctl/backends/*.py, src/wanctl/steering/*.py), verify at least one file exists using Path.glob() from project root

**Class TestRuntimeDependencyVersions:**
1. `test_dependency_importable` -- parametrized over all deps from pyproject.toml. Uses the import name mapping. Calls `importlib.import_module(import_name)`.
2. `test_dependency_meets_version_spec` -- parametrized over all deps. Uses `importlib.metadata.version(package_name)` to get installed version, then `packaging.version.Version` to compare against the minimum spec. Note: `packaging` is available as a dependency of pip/setuptools (already in dev env). If not, use simple tuple comparison after splitting version on ".".

```python
    ...
```

For version comparison, use `from packaging.version import Version` if available, otherwise fall back to `importlib.metadata.version` string comparison with a simple helper. The dev environment has `packaging` (it's a setuptools dependency).
