<objective>
Teach `BackgroundRTTThread` to publish a current-cycle status on every iteration
(including zero-success cycles) via a new frozen/slots dataclass `RTTCycleStatus`
and a read-only accessor `get_cycle_status()`. The existing `_cached` / `get_latest()`
contract stays byte-identical; this plan only ADDS a parallel surface.

the only place that sees `len(successful_rtts)` before deciding whether to overwrite
`_cached`; it must publish that count so downstream consumers (Plan 187-02) can
distinguish a stale cache from a healthy current sample.
