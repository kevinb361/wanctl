<objective>
Add focused contract tests for the Phase 186 measurement-health surface to
`tests/test_health_check.py`. Tests call `_build_measurement_section`
directly with crafted `health_data` fixtures; they do not spin up an HTTP
server. Coverage is organized into three clearly-named test groups:

1. **Cross-product combinations** — exactly 6 tests pinning every legal
   `state × stale` combination (`{healthy, reduced, collapsed} × {fresh, stale}`).
   Parametrization is permitted where it improves clarity.
2. **Successful-count boundary partition** — parametrized over `range(0, 4)`
   pinning the state derivation (0 -> collapsed, 1 -> collapsed,
   2 -> reduced, 3 -> healthy). This is a partition WITHIN the `collapsed`
   state, not a separate axis of the cross-product.
3. **Stale boundary** — one focused test pinning `staleness_sec == 3 * cadence_sec`
   as inclusive-fresh (`stale=False`) and `staleness_sec > 3 * cadence_sec`
   as stale.

Plus two defensive tests pinning the D-14 and D-16 decisions, and a
backwards-compatibility test pinning the five existing fields.

Purpose: Phase 186 is a pure contract phase and this plan IS the phase's
validation surface (no RESEARCH.md, no separate VALIDATION.md). Contract
tests pin every decision from CONTEXT.md D-01..D-16 against code so
future refactors cannot silently drift.

Output: Additive test class in `tests/test_health_check.py`. No changes
to `src/wanctl/` in this plan.
</objective>
