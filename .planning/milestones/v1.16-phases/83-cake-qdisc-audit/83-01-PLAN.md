    **GREEN phase -- implement src/wanctl/check_cake.py:**

    Module structure (follow check_config.py conventions -- section dividers, docstrings):

    ```
    # CONSTANTS
    # CONFIG EXTRACTION (pure functions)
    _extract_router_config(data) -> dict
    _extract_queue_names(data, config_type) -> dict[str, str]
    _extract_ceilings(data, config_type) -> dict[str, int | None]
    _extract_mangle_comment(data) -> str | None

    # ENV VAR CHECK
    check_env_vars(data) -> list[CheckResult]
      Check if router.password contains ${VAR} and the env var is unset -> ERROR

    # CONNECTIVITY
    check_connectivity(client, transport, host, port) -> list[CheckResult]
      REST: client.test_connection()
      SSH: client.run_cmd("/system/resource/print", capture=True, timeout=5) -> rc==0

    # QUEUE AUDIT (CAKE-02, CAKE-03, CAKE-04)
    check_queue_tree(client, queue_names, ceilings, config_type) -> list[CheckResult]
      For each direction (download, upload):
        - GET queue stats via client.get_queue_stats(name)
        - Missing -> ERROR "queue not found"
        - Found -> PASS "queue exists"
        - Check queue field startswith("cake") -> PASS/ERROR (CAKE-03)
        - For autorate with ceiling: compare ceiling bps to int(max-limit) (CAKE-04)
          Equal -> PASS
          Different -> PASS with note: "max-limit {actual} (current), ceiling {expected} (config) -- max-limit changes dynamically during congestion"
        - For steering: skip max-limit comparison

    # MANGLE RULE (CAKE-05)
    check_mangle_rule(client, mangle_comment) -> list[CheckResult]
      Use client.get_queue_stats pattern but for mangle rules.
      For REST: use _find_mangle_rule_id() which already queries by comment.
      For SSH: run_cmd('/ip firewall mangle print where comment~"X"', capture=True) and check output.
      Found -> PASS, not found -> ERROR

    # ORCHESTRATOR
    run_audit(data, config_type, client) -> list[CheckResult]
      1. check_env_vars
      2. check_connectivity -- if ERROR, skip rest with "Skipped: router unreachable"
      3. check_queue_tree
      4. check_mangle_rule (steering only)

    # CLI
    create_parser() -> argparse.ArgumentParser
      Same flags as check-config: config_file, --type, --json, --no-color, -q/--quiet
    main() -> int
      Parse args, load YAML, detect type, create router client via SimpleNamespace + get_router_client,
      run audit, format output, return exit code.
    ```

    Key implementation details:
    - Use SimpleNamespace to wrap extracted router config dict for get_router_client() compatibility
    - Import Severity, CheckResult, format_results, format_results_json, detect_config_type from check_config
    - For SSH connectivity: hasattr(client, 'test_connection') check, fallback to run_cmd
    - For mangle rule check via REST: use client._find_mangle_rule_id(comment) if hasattr, else run_cmd
    - For SSH mangle: run_cmd('/ip firewall mangle print where comment~"X"', capture=True) and check if output contains data
    - All REST API values are strings -- convert with int() before comparison
    - Handle deprecated steering queue names: cake_queues.spectrum_download -> primary_download (replicate same defaulting as SteeringConfig._load_cake_queues)
    - Password env var check: if router.password starts with "${" and ends with "}", extract var name, check os.environ -- ERROR if unset, PASS if set
    - Category names: "Environment", "Connectivity", "Queue Tree", "CAKE Type", "Mangle Rule"
