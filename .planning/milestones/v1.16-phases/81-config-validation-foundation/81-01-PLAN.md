  <action>
Create `src/wanctl/check_config.py` following the `wanctl-history` CLI pattern (create_parser/main).

**Data model:**
- `Severity` enum: PASS, WARN, ERROR
- `CheckResult` dataclass: category (str), field (str), severity (Severity), message (str), suggestion (str | None)

**KNOWN_AUTORATE_PATHS set:** Build from BASE_SCHEMA paths + Config.SCHEMA paths + all imperatively-loaded paths documented in 81-RESEARCH.md Example 1. This is the comprehensive set of ~57 valid config paths. Include both legacy (floor_mbps) and modern (floor_green_mbps etc.) keys.

**6 category validators (each returns list[CheckResult]):**

1. `validate_schema_fields(data)` -- Iterate BASE_SCHEMA + Config.SCHEMA, call validate_field() per field in try/except, return PASS for valid fields and ERROR for invalid. This is Pattern 2 from research.

2. `validate_cross_fields(data)` -- Extract download/upload config dicts. For download: get ceiling, all floor values (handle both legacy floor_mbps and modern floor_green_mbps format as _load_download_config does), call validate_bandwidth_order() wrapped in try/except (PASS if valid, ERROR if raises). Same for upload. Extract threshold values (target_bloat_ms, warn_bloat_ms, hard_red_bloat_ms with default 15.0), call validate_threshold_order() wrapped. Also check ceiling >= floor for both directions.

3. `check_unknown_keys(data)` -- Walk all leaf paths in the config dict. Compare against KNOWN_AUTORATE_PATHS. For unknown keys, use `difflib.get_close_matches(key, KNOWN_AUTORATE_PATHS, n=1, cutoff=0.6)` and include "did you mean: X?" in suggestion if match found. WARN severity. Special handling: `alerting.rules.*` sub-keys are dynamic (per-alert-type), so skip deep validation under `alerting.rules`.

4. `check_paths(data)` -- Check parent dirs exist for logging.main_log, logging.debug_log (ERROR if missing, suggestion: `mkdir -p {parent}`). Check state_file parent dir if present. Check router.ssh_key exists (ERROR if missing), check permissions with stat module (WARN if group/other bits set, suggestion: `chmod 600 {path}`). If router.transport is "rest", SSH key check is informational only (PASS regardless).

5. `check_env_vars(data)` -- Walk all string values in config dict. Match `${VAR}` pattern with regex. If var set in os.environ: PASS. If not: WARN with message about env-specific nature. Per user decision: warning not error.

6. `check_deprecated_params(data)` -- Check for alpha_baseline (deprecated -> baseline_time_constant_sec) and alpha_load (deprecated -> load_time_constant_sec) in the thresholds dict. Show WARN with translated value display matching the format from CONTEXT.md decisions. Reuse the transform logic from deprecate_param() but do NOT call it directly (it modifies the dict and logs, we just want to detect and report). The transform for alpha->time_constant is: `round(1.0 / (value * 20), 1)` (cycle_interval=0.05, so 1/(alpha * 1/cycle_interval)).

**Output formatter:**
- `format_results(results, no_color, quiet)` -- Group results by category. Print category header (e.g., `=== Schema Validation ===`). Print each result with marker (checkmark PASS green, warning WARN yellow, X FAIL red). Skip PASS results when quiet=True. Print summary line: `Result: PASS/WARN/FAIL (N errors, N warnings)`.
- Color: Use raw ANSI codes (no new deps). Disable when no_color=True OR not sys.stdout.isatty() OR os.environ.get("NO_COLOR") is set.

**CLI:**
- `create_parser()` -- ArgumentParser with prog="wanctl-check-config", positional config_file arg, --no-color flag, -q/--quiet flag.
- `main()` -- Parse args, load YAML with yaml.safe_load, run all 6 validators, format output, return exit code. Exit code: 1 if any ERROR, 2 if any WARN but no ERROR, 0 otherwise.

**Anti-patterns to avoid:**
- Do NOT instantiate Config() or BaseConfig() -- only access SCHEMA class attributes
- Do NOT call validate_schema() directly (it raises with concatenated errors) -- iterate fields individually
- Do NOT short-circuit on first error in any validator
- Do NOT import daemon startup code paths
- Handle YAML parse errors gracefully (print error, exit 1)
