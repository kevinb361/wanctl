  <action>
    1. Add `import json` at top of check_config.py.

    2. Add `format_results_json(results: list[CheckResult], config_type: str) -> str` function that:
       - Counts errors (Severity.ERROR) and warnings (Severity.WARN)
       - Determines result word: "FAIL" if errors > 0, "WARN" if warnings > 0, else "PASS"
       - Groups results by category (preserving insertion order)
       - Each check result dict: {"field": r.field, "severity": r.severity.value, "message": r.message} plus optional "suggestion" key (only when r.suggestion is not None -- omit key entirely when None)
       - Top-level dict: {"config_type": config_type, "result": result_word, "errors": error_count, "warnings": warn_count, "categories": categories_dict}
       - Returns json.dumps(output, indent=2)
       - This function never touches ANSI codes -- it reads raw CheckResult.message which never contains escape sequences (colors are only applied in format_results)

    3. Add `--json` argument to create_parser(): `--json`, action="store_true", help="Output results as JSON". Store as `args.json` (note: argparse attribute will be `args.json` since the flag is `--json`).

    4. Update main() to check `args.json`: if true, call `format_results_json(results, config_type)` instead of `format_results()`. Print the JSON string. Exit code logic is unchanged (same has_errors/has_warnings check regardless of output mode). Per user decision: `--json` replaces text output entirely -- only JSON goes to stdout, no text headers, no summary line.

    5. Per user decision: `--json` and `--quiet` are independent. When --json is used, all results are always included regardless of -q. The --quiet flag only affects text output (format_results).
