## Milestone Summary

**Key Decisions:**

- Keep the controller link-agnostic and zero-diff: bypass is data-path/failure tooling, not `src/wanctl` control logic.
- Use `silicom-bypass` as the single operator composition layer; `silicom-test` does not call `bpctl_util` directly.
- Keep all Silicom deploy surfaces install-only/off-by-default; live arming and HIL scenarios remain explicit operator actions.
- Enforce watchdog stop discipline through W-INV/sentinel-clean helper surfaces rather than raw `systemctl stop/restart` examples.
- Use exactly the shipped pair allowlist (`att-modem`, `spec-modem`) for HIL pair-taking commands before filesystem paths, generated JSON/Python, live gates, or mutation verbs.

**Issues Resolved:**

- Stale native `wanctl@` watchdog coupling after the cake-autorate migration.
- Partial bpctl script surface converted into guarded CLI + boot baseline + documented deploy path.
- HIL result-path/pair traversal risk closed before milestone close.
- PATH-resolved live CLI gate bypass closed before milestone close.

**Issues Deferred:**

- Known deferred open artifacts at close: 11 (see `.planning/STATE.md` Deferred Items).
- ROLE-01 native-controller retirement remains time/event-gated; v1.52 supplies the rollback-drill harness but does not make the decision.
- TAIL-01, SEED-005, SEED-007, fping RTT backend evaluation, and Phase 218 watch-list remain future/backlog work.

**Technical Debt Incurred / Carried:**

- Legacy raw watchdog stop/restart examples remain in `docs/SILICOM-BYPASS.md`; W-INV implementation/tests pass, but docs cleanup is needed.
