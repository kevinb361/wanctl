<objective>
Make the deploy operator path explicit and repeatable without changing the deployment script into a service-orchestration tool.

Purpose: the audit gap is not that migration is impossible; it is that the repo's active flow omits the required migration checkpoint and leaves steering-related monitoring implicit. This plan aligns the printed next steps and the deployment/getting-started docs with the actual production sequence used for v1.35.
</objective>
