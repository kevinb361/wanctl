<task type="auto" tdd="true">
  <name>Task 1: Compare subcommand with grade delta and color-coded metrics</name>
  <files>src/wanctl/benchmark.py, tests/test_benchmark.py</files>
  <behavior>
    - compute_deltas(before: dict, after: dict) returns dict of numeric field deltas (after - before)
    - format_comparison(before: dict, after: dict, deltas: dict, color: bool) returns multi-line string: grade summary line ("C -> A+"), latency table with deltas, throughput with deltas, metadata line
    - Negative latency delta = green (improved), positive = red (worse)
    - Positive throughput delta = green (improved), negative = red (worse)
    - Grade improvement = green text, degradation = red text
    - run_compare(args) fetches runs via query_benchmarks: if args.ids has 2 IDs use those; if empty fetch latest 2 (limit=2)
    - run_compare prints comparability warning to stderr when server or duration differ between runs
    - run_compare returns 1 with clear error when fewer than 2 results available
    - run_compare returns 1 with clear error when specific ID not found
    - --json outputs structured {before, after, deltas} JSON
    - main() routes args.command == "compare" to run_compare()
  </behavior>
  <action>
  **benchmark.py compute_deltas():** Accept before/after dicts (rows from query_benchmarks). Compute after[field] - before[field] for all numeric latency and throughput fields plus baseline_rtt. Return dict keyed by field name.

  **benchmark.py format_comparison():** Build multi-line output per RESEARCH Pattern 5:
  - Line 1: "Grade: {before_dl_grade} -> {after_dl_grade}" with color on arrow direction (green if improved, red if worse). Use existing _colorize() for grade coloring.
  - Blank line, then "Download latency" section: Avg, P50, P95, P99 with Before/After/Delta columns. Delta values formatted with sign (+/-) and "ms" suffix. Color: negative delta (latency decreased) = green (\033[32m), positive delta = red (\033[31m).
  - "Upload latency" section: same format.
  - "Throughput" section: Download and Upload with Before/After/Delta. Positive delta = green, negative = red. Format with "Mbps" suffix.
  - Blank line, then metadata: "Baseline RTT: {before} -> {after}", "Server: {value} (both runs)" or "Server: {before_server} vs {after_server}" if different, "Duration: {value} (both runs)" or different.
  - Final line: "Run #{before_id} ({before_timestamp}) vs Run #{after_id} ({after_timestamp})"

  **benchmark.py run_compare(args):** Determine which runs to compare. If args.ids has exactly 2 values, fetch by ids via query_benchmarks(ids=args.ids, db_path=args.db). If args.ids is empty, fetch latest 2 via query_benchmarks(limit=2, db_path=args.db). If fewer than 2 results, print error to stderr and return 1. If specific IDs requested but not found, print error and return 1. Sort so earlier run is "before" and later run is "after" (by timestamp). Check comparability: if server or duration differ, print warning to stderr (don't block). Compute deltas, format output. If args.json, output structured JSON with before/after/deltas keys. Else print format_comparison().

  **benchmark.py main():** Replace compare stub with run_compare(args) call.

  **tests/test_benchmark.py:** Add TestComputeDeltas class: basic delta computation, all numeric fields present. Add TestFormatComparison class: contains grade arrow, contains latency deltas, contains throughput, contains metadata, no-color mode has no ANSI codes, improved latency shows negative delta. Add TestRunCompare class using real tmp_path SQLite: default compare (latest vs previous), compare with specific IDs, error on fewer than 2 results, error on missing ID, comparability warning on different server, comparability warning on different duration, --json output structure. Mock sys.argv and use capsys.
  </action>
  <verify>
    <automated>.venv/bin/pytest tests/test_benchmark.py -x -v -k "TestComputeDeltas or TestFormatComparison or TestRunCompare" && .venv/bin/ruff check src/wanctl/benchmark.py && .venv/bin/mypy src/wanctl/benchmark.py</automated>
  </verify>
  <done>wanctl-benchmark compare shows grade delta with color-coded latency/throughput improvements. Defaults to latest-vs-previous. Supports specific IDs. Warns on non-comparable runs. --json outputs structured comparison.</done>
</task>

<task type="auto" tdd="true">
  <name>Task 2: History subcommand with time-range filtering and table display</name>
  <files>src/wanctl/benchmark.py, tests/test_benchmark.py</files>
  <behavior>
    - format_history(rows: list[dict], color: bool) returns tabulated table with columns: ID, Timestamp, WAN, Grade, Avg Latency, DL Mbps, Label
    - Timestamp formatted as "YYYY-MM-DD HH:MM" (truncate seconds/tz for readability)
    - Grade column shows download_grade with color via _colorize()
    - Empty history returns "No benchmark results found."
    - run_history(args) fetches via query_benchmarks with filters: --last converts timedelta to ISO cutoff string, --from/--to convert to ISO strings, --wan filters by WAN name
    - run_history --json outputs list of result dicts
    - main() routes args.command == "history" to run_history()
    - Full regression: all 76+ existing tests pass, all new Plan 01 + Plan 02 tests pass
  </behavior>
  <action>
  **benchmark.py format_history():** Accept list[dict] rows (from query_benchmarks, already ordered DESC). Build table using tabulate library. Columns: ID (row["id"]), Timestamp (parse ISO string, format as "%Y-%m-%d %H:%M"), WAN (row["wan_name"]), Grade (_colorize(row["download_grade"], row["download_grade"], color)), Avg Latency (f'{row["download_latency_avg"]:.1f}ms'), DL Mbps (f'{row["download_throughput"]:.1f}'), Label (row.get("label") or ""). Use tabulate(table_data, headers=["ID", "Timestamp", "WAN", "Grade", "Avg Latency", "DL Mbps", "Label"], tablefmt="simple"). If rows empty, return "No benchmark results found."

  **benchmark.py run_history(args):** Build query params. If args has "last" attr and args.last is not None (timedelta from parse_duration), compute cutoff = datetime.now(UTC) - args.last, set start_ts = cutoff.isoformat(). If args has "from_ts" and it's not None, convert int timestamp to ISO string via datetime.fromtimestamp(args.from_ts, tz=UTC).isoformat(). Same for "to_ts" -> end_ts. wan = getattr(args, "wan", None) -- note: history subparser has its own --wan, which shadows the global --wan from run path; use the subparser-specific one. Call query_benchmarks(db_path=args.db, start_ts=start_ts, end_ts=end_ts, wan=wan). If args.json, output json.dumps([row for row in rows], indent=2). Else print(format_history(rows, color)).

  **benchmark.py main():** Replace history stub with run_history(args) call. Determine use_color once at top (before command routing) so compare and history can reuse it.

  **tests/test_benchmark.py:** Add TestFormatHistory class: formats table with all columns, empty list returns "No benchmark results found", grade is colorized when color=True, label column shows value or blank, timestamps formatted without seconds. Add TestRunHistory class using real tmp_path SQLite: lists all stored results, --last filters by time range, --wan filters by WAN name, --json outputs list, empty DB shows "No benchmark results found". Mock sys.argv and use capsys. Add TestFullRegression class: run full `.venv/bin/pytest tests/test_benchmark.py` to confirm all tests pass.
  </action>

  </verify>
