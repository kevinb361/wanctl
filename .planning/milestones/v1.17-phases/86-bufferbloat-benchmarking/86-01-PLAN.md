<task type="auto" tdd="true">
  <name>Task 2: Flent result parsing and statistics extraction</name>
  <files>src/wanctl/benchmark.py, tests/test_benchmark.py</files>
  <behavior>
    - parse_flent_results(gz_path) returns dict with "results" and "x_values" keys
    - extract_latency_stats(data, baseline_rtt) returns dict with avg/p50/p95/p99 as latency INCREASE over baseline
    - extract_latency_stats filters None values from ping series
    - extract_latency_stats returns zeros when ping series is empty
    - extract_latency_stats with baseline_rtt=20, mean_ping=45 returns avg ~25ms increase
    - extract_throughput(data) returns (download_mbps, upload_mbps) tuple
    - extract_throughput handles empty series returning (0.0, 0.0)
    - extract_throughput filters None values
    - build_result() assembles BenchmarkResult from parsed data, baseline RTT, server, duration
  </behavior>
  <action>
    Add to `src/wanctl/benchmark.py`:

    1. `parse_flent_results(gz_path: str) -> dict`:
       - `gzip.open(gz_path, "rt")` then `json.load(f)`
       - Return the parsed dict (contains "results", "x_values", "metadata")

    2. `extract_latency_stats(data: dict, baseline_rtt: float) -> dict`:
       - Get `data["results"].get("Ping (ms) ICMP", [])`
       - Filter `None` and `<= 0` values
       - If empty, return `{"avg": 0.0, "p50": 0.0, "p95": 0.0, "p99": 0.0}`
       - Compute mean latency under load: `statistics.mean(values)`
       - Latency increase = mean - baseline_rtt (floor at 0)
       - For percentiles: use `statistics.quantiles(values, n=100)` which returns 99 cut points. P50 = index 49, P95 = index 94, P99 = index 98. Each percentile is also expressed as increase over baseline (value - baseline_rtt, floor 0).
       - Return `{"avg": increase, "p50": p50_increase, "p95": p95_increase, "p99": p99_increase}`

    3. `extract_throughput(data: dict) -> tuple[float, float]`:
       - Get `data["results"].get("TCP download sum", [])` and `"TCP upload sum"`
       - Filter `None` values
       - Return `(statistics.mean(dl_values), statistics.mean(ul_values))` or `(0.0, 0.0)` if empty

    4. `build_result(data: dict, baseline_rtt: float, server: str, duration: int) -> BenchmarkResult`:
       - Call extract_latency_stats and extract_throughput
       - Compute grade from avg latency increase via compute_grade()
       - Both download_grade and upload_grade use the SAME latency increase (RRUL is bidirectional per Pitfall 1)
       - Both directions get the SAME latency percentile values (same ping series)
       - Per-direction throughput is different (TCP download sum vs TCP upload sum)
       - timestamp = `datetime.now(UTC).isoformat()`
       - Return assembled BenchmarkResult

    Add to `tests/test_benchmark.py`:
    - TestFlentParsing class: create a gzipped JSON fixture with sample flent data, verify parse_flent_results reads it
    - TestLatencyStats class: test with known values, test None filtering, test empty series, test baseline subtraction
    - TestThroughput class: test extraction, test None filtering, test empty
    - TestBuildResult class: verify full assembly produces correct BenchmarkResult fields
    - Use `tmp_path` pytest fixture for gzip test files
  </action>
