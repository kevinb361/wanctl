<task type="auto" tdd="true">
  <name>Task 1: Prerequisite checks, server connectivity, and daemon warning</name>
  <files>src/wanctl/benchmark.py, tests/test_benchmark.py</files>
  <behavior>
    - check_prerequisites(server) returns list of (name, passed, detail) tuples
    - When flent is missing: ("flent", False, "not found -- install with: sudo apt install flent")
    - When netperf is missing: ("netperf", False, "not found -- install with: sudo apt install netperf")
    - When both present and server reachable: 3 items all True, server detail includes baseline RTT
    - When server unreachable: ("server", False, "... unreachable")
    - Server check skipped if either binary missing
    - check_server_connectivity(server) returns (reachable: bool, baseline_rtt_ms: float)
    - check_server_connectivity uses netperf -H server -t TCP_STREAM -l 1 with 3s timeout
    - check_server_connectivity measures baseline RTT via ping -c 5 -i 0.2
    - check_daemon_running() returns (running: bool, detail: str) by checking /run/wanctl/*.lock
    - Daemon running = warning printed to stderr, does NOT block test
  </behavior>
  <action>
    Add to `src/wanctl/benchmark.py`:

    1. `check_server_connectivity(server: str, timeout: int = 3) -> tuple[bool, float]`:
       - Run `netperf -H server -t TCP_STREAM -l 1` with subprocess.run, timeout=timeout, capture_output=True
       - `# nosec B603` annotation on subprocess.run (hardcoded netperf invocation)
       - If returncode != 0 or TimeoutExpired or FileNotFoundError: return (False, 0.0)
       - If reachable, measure baseline RTT: `ping -c 5 -i 0.2 server` with subprocess.run
       - Parse ping output with `parse_ping_output()` from `wanctl.rtt_measurement`
       - Return (True, min(rtts)) if rtts else (True, 0.0)

    2. `check_prerequisites(server: str) -> list[tuple[str, bool, str]]`:
       - Check `shutil.which("flent")` -- if None: ("flent", False, "not found -- install with: sudo apt install flent")
       - Check `shutil.which("netperf")` -- if None: ("netperf", False, "not found -- install with: sudo apt install netperf")
       - If both present: call check_server_connectivity(server)
         - If reachable: ("server", True, f"reachable ({baseline:.0f}ms baseline)")
         - If not: ("server", False, f"{server} unreachable (3s timeout)")
       - Return list of all check tuples

    3. `check_daemon_running() -> tuple[bool, str]`:
       - Glob `/run/wanctl/*.lock` files
       - For each lock file, use `read_lock_pid()` and `is_process_alive()`
       - If any alive daemon found: return (True, f"wanctl daemon is running (PID {pid})")
       - Else: return (False, "")

    4. `_print_prerequisites(checks: list[tuple[str, bool, str]], color: bool) -> None`:
       - Print checklist to stderr:
         - Pass: `[OK] flent ......... found at /usr/bin/flent`
         - Fail: `[FAIL] flent ...... not found -- install with: sudo apt install flent`
       - Use green/red ANSI colors when color=True

    Add to `tests/test_benchmark.py`:
    - TestPrerequisites: mock shutil.which to test all combinations (both present, flent missing, netperf missing, both missing)
    - TestServerConnectivity: mock subprocess.run for netperf probe success/failure/timeout, mock ping for baseline RTT
    - TestDaemonWarning: mock glob and lock_utils to test running/not-running detection
    - TestPrintPrerequisites: capture stderr, verify checklist format with pass/fail markers
  </action>
  <verify>
    <automated>.venv/bin/pytest tests/test_benchmark.py::TestPrerequisites tests/test_benchmark.py::TestServerConnectivity tests/test_benchmark.py::TestDaemonWarning -x -v</automated>
  </verify>
  <done>Prerequisite checks detect missing binaries with apt install instructions. Server connectivity verified via netperf probe with 3s timeout. Baseline RTT measured via ping. Daemon detection warns but does not block.</done>

<task type="auto" tdd="true">
  <name>Task 2: CLI orchestration, flent subprocess, output formatting, entry point</name>
  <files>src/wanctl/benchmark.py, tests/test_benchmark.py, pyproject.toml</files>
  <behavior>
    - run_benchmark(server, duration) invokes flent rrul -H server -l duration -o tmpfile, returns BenchmarkResult or None
    - run_benchmark passes timeout=duration+30 to subprocess.run
    - --quick sets duration to 10s, default is 60s
    - --server sets server host, default is netperf.bufferbloat.net
    - --json outputs BenchmarkResult as JSON dict to stdout
    - --no-color disables ANSI colors in output
    - Normal output shows large grade prominently with color, then latency/throughput detail
    - Exit 0 when test produces grades, exit 1 when prerequisites fail or flent errors
    - create_parser() has --server, --quick, --json, --no-color arguments
    - main() orchestrates: parse args -> prerequisites -> daemon warning -> run test -> display results
    - pyproject.toml has wanctl-benchmark = "wanctl.benchmark:main"
  </behavior>
  <action>
    Add to `src/wanctl/benchmark.py`:

    1. `run_benchmark(server: str, duration: int) -> BenchmarkResult | None`:
       - Create tempfile.mkdtemp(prefix="wanctl-benchmark-")
       - Build cmd: `["flent", "rrul", "-H", server, "-l", str(duration), "-o", output_path]`
       - subprocess.run with capture_output=True, text=True, timeout=duration+30
       - `# nosec B603` annotation
       - Print a brief status line to stderr before test starts: `Running RRUL test ({duration}s)...`
       - If returncode != 0: print flent stderr to stderr, clean up tmpdir, return None
       - If TimeoutExpired: print timeout message, clean up, return None
       - Call parse_flent_results(output_path), then build_result() to get BenchmarkResult
       - Clean up tmpdir (shutil.rmtree), return result
       - Important: parse BEFORE rmtree -- the gz file is in the tmpdir

    2. `format_grade_display(result: BenchmarkResult, color: bool) -> str`:
       - Large prominent grade display per user decision:
         ```
         Download: A+   Upload: B
         ```
       - Color: A+/A = green (ANSI 32), B = yellow (33), C = yellow (33), D = red (31), F = red (31)
       - Below the grade line, show latency detail:
         ```
         Latency under load (increase over baseline):
           Avg: 3.2ms   P50: 2.8ms   P95: 8.1ms   P99: 12.4ms
         ```
       - Then throughput:
         ```
         Throughput:
           Download: 94.2 Mbps   Upload: 11.3 Mbps
         ```
       - Then baseline info: `Baseline RTT: 23.1ms   Server: netperf.bufferbloat.net   Duration: 60s`
       - If --quick was used, add note: `Note: Quick mode (10s) -- grades may be less accurate than full 60s test`

    3. `format_json(result: BenchmarkResult) -> str`:
       - `json.dumps(dataclasses.asdict(result), indent=2)`

    4. `create_parser() -> argparse.ArgumentParser`:
       - prog="wanctl-benchmark"
       - description="Run RRUL bufferbloat test and grade results"
       - `--server`: default="netperf.bufferbloat.net", help="Netperf server host"
       - `--quick`: action="store_true", help="Run 10s quick test instead of 60s full test"
       - `--json`: action="store_true", help="Output results as JSON"
       - `--no-color`: action="store_true", help="Disable colored output"
       - No positional args (standalone tool, no config file needed)

    5. `main() -> int`:
       - Parse args
       - Determine duration: 10 if args.quick else 60
       - Determine color: not args.no_color and sys.stderr.isatty()
       - Run check_prerequisites(args.server) -> print checklist to stderr
       - If any prerequisite failed: print install instructions, return 1
       - Extract baseline_rtt from server check result detail (parse the number from "reachable (23ms baseline)")
         - Better approach: store baseline_rtt as a module-level variable during check_prerequisites, or have check_prerequisites return it separately
         - Cleanest: refactor check_prerequisites to also return baseline_rtt as second return value, e.g., `-> tuple[list[tuple[str, bool, str]], float]`
       - Run check_daemon_running() -> if running, print warning to stderr
       - Run run_benchmark(args.server, duration)
       - If None (test failed): return 1
       - If args.json: print format_json(result) to stdout
       - Else: print format_grade_display(result, color) to stdout
       - Return 0

    6. Update `pyproject.toml`:
       - Add `wanctl-benchmark = "wanctl.benchmark:main"` under `[project.scripts]`

    Add to `tests/test_benchmark.py`:
    - TestRunBenchmark: mock subprocess.run for flent, provide sample gzipped JSON as output file, verify BenchmarkResult returned. Test failure case (returncode != 0). Test timeout case.
    - TestQuickMode: mock subprocess, verify `-l 10` passed to flent cmd
    - TestServerFlag: mock subprocess, verify `-H custom-server` passed to flent cmd
    - TestFormatGradeDisplay: verify output contains grade letters, latency percentiles, throughput, baseline info
    - TestFormatJson: verify valid JSON output with all BenchmarkResult fields
    - TestMain: mock subprocess and shutil.which, test full main() flow returns 0 on success, 1 on prerequisite failure, 1 on flent failure
    - TestCreateParser: verify parser has all expected arguments with correct defaults
  </action>

  </verify>
