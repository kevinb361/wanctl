    - TestColorFlags::test_256_color_sets_env: When main() called with --256-color, os.environ["TEXTUAL_COLOR_SYSTEM"] is set to "256" before app.run()
    - TestColorFlags::test_flags_mutually_exclusive_priority: --no-color takes priority over --256-color (NO_COLOR set, TEXTUAL_COLOR_SYSTEM not set)
    - TestColorFlags::test_no_flags_no_env_change: When neither flag passed, NO_COLOR and TEXTUAL_COLOR_SYSTEM are not set by main()
  </behavior>
  <action>
    **Tests first** in `tests/test_dashboard/test_layout.py` (append TestColorFlags class):

    Use `from wanctl.dashboard.app import parse_args` for parsing tests. For env var tests, use `unittest.mock.patch.dict(os.environ, {}, clear=False)` and `unittest.mock.patch("wanctl.dashboard.app.DashboardApp")` to prevent actual app launch. Call `main(["--no-color"])` and assert `os.environ.get("NO_COLOR") == "1"`.

    **Then modify `src/wanctl/dashboard/app.py`:**

    1. Add `import os` at top of file.
    2. In `parse_args()`, add two arguments:
       ```python
       parser.add_argument("--no-color", action="store_true",
                           help="Disable all color output")
       parser.add_argument("--256-color", action="store_true", dest="color_256",
                           help="Use 256-color mode (for limited terminals)")
       ```
    3. In `main()`, after `args = parse_args(argv)` but before `app = DashboardApp(config)` and `app.run()`, add:
       ```python
       if args.no_color:
           os.environ["NO_COLOR"] = "1"
       elif args.color_256:
           os.environ["TEXTUAL_COLOR_SYSTEM"] = "256"
       ```

    This is the standard Textual/Rich pattern. `NO_COLOR` is the universal convention (no-color.org). `TEXTUAL_COLOR_SYSTEM` is Textual's own env var for forcing color depth.
  </action>
