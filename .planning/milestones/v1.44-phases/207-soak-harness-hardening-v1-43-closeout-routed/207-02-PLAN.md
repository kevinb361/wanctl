<objective>
HRDN-02: make scripts/soak-capture.sh survive transient `curl`/`jq` blips during a 24h soak without aborting on the first error. Bounded failure tolerance with a documented threshold; postmortem sidecar TSV; no change to the NDJSON schema consumed downstream by `scripts/soak_summary_aggregate.py`.

The current loop runs `curl ... | jq ...` under `set -euo pipefail`. Any transient HTTP/curl/jq failure aborts the whole soak. v1.43 closeout (WR-02) waived this; v1.44 closes it.

Tolerance model (CONTEXT.md D-08..D-12):
- Per-row failure bypass using subshell or explicit `|| true` capture — NOT a global `set +e`. The `set -euo pipefail` posture stays intact.
- Aggregate `row_failed` counter (incremented when the pipeline produced no valid NDJSON row that iteration). This is the value compared to threshold.
- Distinct mode counters tracked alongside, for the postmortem: `curl_exit_nonzero`, `curl_http_nonzero` (HTTP code != 200), `jq_parse_error`, `empty_body`.
- Threshold: lifetime failure-rate cap, default `0.01` (1% of attempted rows). Configurable via `SOAK_FAIL_RATE_THRESHOLD`.
- Evaluated only AFTER `MIN_SAMPLES_BEFORE_EVAL=60` attempted rows (avoids spurious abort in first minute).
- Evaluated once per iteration after the sleep.
- Sidecar TSV at `${CAPTURE_DIR}/soak-capture-errors.tsv` with columns `t_wall \t failure_mode \t last_curl_exit \t last_message`. NOT a sentinel NDJSON row — keep the NDJSON consumer-contract clean.
- Final summary line on abort goes to stderr (mode breakdown + lifetime totals).

**Denominator semantics (Codex review clarification):** the failure rate is `row_failed / row_total` where `row_total` counts ATTEMPTED iterations, not expected wall-clock slots. A slow curl that holds the loop for ~11 seconds still increments `row_total` by exactly 1. The script header MUST document this. HRDN-02's scope is closing the original abort-on-first-blip gap (D-08), not wall-clock slot accounting — wall-clock missed-slot counting would be a future enhancement (out of scope for v1.44).

**Edge-case hardening (Codex review M-4):**
- **Env-var validation:** before the loop, validate `SOAK_FAIL_RATE_THRESHOLD` is a numeric in `[0.0, 1.0]` (use awk) and `MIN_SAMPLES_BEFORE_EVAL` is a positive integer. Either invalid → fail-closed exit with a clear stderr message.
- **`SIDECAR_TSV` ordering:** assignment goes AFTER `CAPTURE_DIR` is set AND AFTER `mkdir -p "$CAPTURE_DIR"`. Under `set -u` an earlier assignment would abort the script.
- **Per-iteration temp-file truncation:** body and out temp files are truncated (`: > "$body_tmp"`, `: > "$out_tmp"`) at the START of each iteration. Stale content from a prior successful iteration must not leak into the next failure-mode classification.
- **TSV `last_message` scrubbing:** strip `\t`, `\n`, `\r` from `last_message` before writing to TSV (e.g., `tr -d '\t\n\r'`). Otherwise the column delimiter breaks.
- **Counter increments:** use `counter=$((counter + 1))` form. The `((counter++))` form returns exit status `1` when counter was `0`, which would trip errexit on the first failure.

Test strategy (D-12):
- pytest with fake `curl` shim on PATH that reads a configurable failure schedule from a file. Real `jq` from PATH (not shimmed).
- One transient → soak continues; sustained → soak aborts above threshold.
- Slow integration variants marked `@pytest.mark.slow` with truncated `SOAK_DURATION_SEC` and lowered `MIN_SAMPLES_BEFORE_EVAL`.
- Env-var validation failure cases covered explicitly.

Output:
- Updated `scripts/soak-capture.sh` with per-row failure capture, threshold gate, sidecar TSV, env-var validation, temp-file truncation, TSV scrubbing, arithmetic counter increments, and no NDJSON schema change.
- New `tests/test_soak_capture_transient_tolerance.py` using PATH-shim approach with env-var-validation coverage.

        path.write_text(body, encoding="utf-8")

    def _setup_shims(tmp_path: Path, schedule: list[str]) -> Path:
        """Schedule entries: 'ok', 'curl_exit', 'http_503', 'empty', 'bad_json'."""
        shim_dir = tmp_path / "shim"
        shim_dir.mkdir()
        schedule_file = tmp_path / "schedule.txt"
        schedule_file.write_text("\n".join(schedule) + "\n", encoding="utf-8")
        counter_file = tmp_path / "counter"
        counter_file.write_text("0", encoding="utf-8")

        curl_body = textwrap.dedent(f"""\
            #!/usr/bin/env bash
            set -u
            SCHEDULE="{schedule_file}"
            COUNTER="{counter_file}"

            while [ "$#" -gt 0 ]; do
              case "$1" in
                --output) out_file="$2"; shift 2 ;;
                --write-out) write_out="$2"; shift 2 ;;
                *) shift ;;
              esac

            i=$(cat "$COUNTER"); echo $((i+1)) > "$COUNTER"

            [ -z "$action" ] && action="ok"

              curl_exit) exit 7 ;;
              http_503)

                [ "$write_out" = "%{{http_code}}" ] && printf '503'

                exit 0

            }}

                fi
