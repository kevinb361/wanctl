> **Revised 2026-05-16 incorporating codex review feedback — see 208-REVIEWS.md.**
> Changes vs original:
> - **HIGH:** exception scope split. The previous plan put `sqlite3.OperationalError` around
>   both `sqlite3.connect()` AND `_query_digest_rows()`. Query-time OperationalError
>   includes "no such table" and malformed-SQL — catching it would mask real schema
>   corruption as a permission skip, directly contradicting D-13. Now:
>   - `(sqlite3.OperationalError, OSError)` is caught ONLY around the `sqlite3.connect()` call.
>   - `_query_digest_rows()` runs OUTSIDE that try block; query-time exceptions bubble to
>     the existing main()-level `(sqlite3.DatabaseError, json.JSONDecodeError, ...)` handler
>     and exit rc=1 with the original error text.
> - **MEDIUM:** `print_digest()` now returns a structured count object
>   `{"readable": int, "printed": int, "read_skipped": int, "write_skipped": int}` instead
>   of a bare int. `main()` decides exit hint based on `readable` vs `printed`:
>   - `readable == 0` → "no readable WAN DBs - try sudo" + rc=0
>   - `readable > 0 and printed == 0` → "operator-summary digest: all output writes failed" + rc=1
> - **LOW:** added discovery-level OSError guard around `discover_wan_dbs()` with stable
>   prefix `operator-summary digest: discovery failed (<error>)` and rc=1.
> - New regression test pins the HIGH fix: DB exists, lacks `alerts` table → rc != 0 AND
>   stderr does NOT contain `operator-summary digest: skipped`.

        are NOT silently masked as permission skips.

            print("no WAN DBs discovered")
            return counts
