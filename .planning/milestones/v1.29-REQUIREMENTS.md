# Requirements: wanctl v1.29

**Defined:** 2026-04-05
**Core Value:** Sub-second congestion detection with 50ms control loops

## v1.29 Requirements

Requirements for Code Health & Cleanup milestone. Each maps to roadmap phases.

### Dead Code & Cruft

- [ ] **DEAD-01**: All unused imports and dead functions/methods are identified and removed
- [ ] **DEAD-02**: Orphaned modules (files no longer imported anywhere) are identified and removed
- [ ] **DEAD-03**: Unused pip dependencies are identified and removed from pyproject.toml
- [ ] **DEAD-04**: Stale TODO comments, outdated docstrings, and dead config references are cleaned up

### Module Complexity

- [ ] **CPLX-01**: Files over ~500 LOC are split into focused, single-responsibility modules
- [ ] **CPLX-02**: Long methods (>50 lines) are extracted into smaller, testable functions
- [ ] **CPLX-03**: Tight coupling between modules is identified and reduced with cleaner interfaces
- [ ] **CPLX-04**: High cyclomatic complexity functions are refactored (nested if/else, long chains)

### Test Quality

- [ ] **TEST-01**: Redundant tests that duplicate coverage or test implementation details are removed
- [ ] **TEST-02**: Brittle mocks are replaced with better patterns (fakes, fixtures, dependency injection)
- [ ] **TEST-03**: Slow tests are profiled and optimized, test parallelization improved
- [ ] **TEST-04**: Test directories are restructured, conftest fixtures consolidated, naming improved

### Type Safety & Linting

- [ ] **TYPE-01**: Missing type annotations are added to all public function signatures
- [ ] **TYPE-02**: Stricter mypy rules enabled (disallow_untyped_defs, warn_return_any, etc.)
- [ ] **TYPE-03**: Additional ruff rules enabled (complexity, naming, import sorting)
- [ ] **TYPE-04**: Interfaces use proper Protocol/ABC patterns, unnecessary Any types removed

## Future Requirements

### Deferred

- **OBSV-01 through OBSV-04**: Prometheus/Grafana export -- infrastructure not yet deployed
- **ATT-TUNE-01**: RRUL A/B tuning sweep for ATT WAN -- requires late-night testing window

## Out of Scope

| Feature | Reason |
|---------|--------|
| New features or capabilities | This milestone is purely internal quality -- no behavioral changes |
| Architecture redesign | Refactor within existing structure, don't reimagine it |
| Performance optimization | v1.27 already optimized hot path; this is code quality, not speed |
| Configuration format changes | Backward compatibility constraint |
| Test coverage increase | Already at 90%+; focus on quality, not quantity |
