# Plan 248.2-01 Summary — fping Freshness / Staleness Repair

## Outcome

Implemented and verified the targeted fping freshness repair discovered by Phase 248.1.

The controller now derives RTT snapshot freshness limits from the active background producer cadence while preserving the existing D-04 floors for normal icmplib behavior:

- soft stale limit: `max(0.5, cadence_sec)`
- hard stale limit: `max(5.0, cadence_sec * 1.5)`

For the default fping 10s background cadence, a healthy cached fping sample is no longer rejected at the old fixed 5s hard limit while waiting for the next expected fping burst.

## Files changed

- `src/wanctl/wan_controller.py`
  - Added `WANController._rtt_staleness_limits()`.
  - `measure_rtt()` now uses cadence-aware soft/hard stale limits.
- `tests/test_rtt_backend_factory.py`
  - Added regression test that a 6s-old fping sample under 10s cadence remains usable.
  - Added regression test that a 16s-old fping sample under 10s cadence is still rejected.
