# Plan 248.3-01 Summary — Native Spectrum CAKE Parity

## Outcome

Aligned native `wanctl@spectrum` CAKE config with the external cake-autorate Spectrum production envelope so future native fping canaries do not conflate RTT backend behavior with unrelated qdisc/rate drift.

Changed `configs/spectrum.yaml`:

- `cake_params.rtt: "100ms"` → `"25ms"`
- removed global `cake_params.ack_filter: false`
  - existing direction-aware defaults now apply:
    - download: `no-ack-filter`
    - upload: `ack-filter`
- `continuous_monitoring.download.ceiling_mbps: 920` → `600`
- retained `continuous_monitoring.download.floor_green_mbps: 550`
- retained upload ceiling 18M

This matches the external cake-autorate trial shape:

- external qdisc init download: 550Mbit, `rtt 25ms`, no-ack-filter
- external qdisc init upload: 18Mbit, `rtt 25ms`, ack-filter
- external cake-autorate download envelope: 500M min / 550M base / 600M max

## Tests added

`tests/test_cake_params.py` now locks native Spectrum parity:

- config download green floor is 550M and ceiling is 600M;
- config CAKE RTT is `25ms`;
- global `ack_filter` override is absent;
- built download qdisc params have no ack-filter;
- built upload qdisc params have ack-filter;
- both directions keep `wash` and `rtt 25ms`;
- `25ms` RTT readback conversion is covered.
