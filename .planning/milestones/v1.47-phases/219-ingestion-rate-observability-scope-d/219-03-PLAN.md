<objective>
Integrate the Phase 219 ingestion block into `wanctl-operator-summary --digest`. The block renders one compact
line per WAN summarizing total rows/sec and the dominant-metric_name table for a 1-hour rolling window. This
satisfies INGEST-04 and resolves the folded todo `2026-04-17-ingestion-rate-tool` (the smoothed tail-rate
window use case is covered by the 3600s window from `per_wan_ingestion_rate_bucketed`).

Purpose: One-glance per-WAN ingestion visibility in the operator digest — the surface Phase 218 audits will use
if they fire during v1.47. Tolerance semantics mirror the existing hard-red block (skip failed WAN, increment
read_skipped, never abort). **Critically (D-22, resolves Codex H5): ingestion data is pre-gathered at function
entry BEFORE the existing hard-red loop, so a query-time DB error on the hard-red path cannot suppress the
ingestion line for the same WAN.**

Output: `print_digest` emits a second block of lines prefixed
`operator-summary digest: ingestion-rate wan=X total_rps=Y top=Z`. Tie-break uses `mixed:t1/t2` (no space, D-20).
Return dict gains `"ingestion_printed"` key (D-26). Test coverage for all four tie-break branches PLUS the
independence guarantee.
</objective>

        counts = {"readable": 0, "printed": 0, "read_skipped": 0, "write_skipped": 0, "ingestion_printed": 0}

            return counts

        # This ensures hard-red query failures cannot suppress ingestion-line emission.
        now = int(time.time())

            try:
                rows, _failures = per_wan_ingestion_rate_bucketed(
                    [db_path], start_ts=now - 3600, end_ts=now, wan=None,
                )
                ingestion_data[db_path] = rows
            except (sqlite3.OperationalError, sqlite3.DatabaseError, OSError) as exc:
                print(
                    f"{_DIGEST_SKIP_PREFIX} (ingestion) wan={wan_name} db={db_path}: {exc}",
                    file=sys.stderr,
                )
                counts["read_skipped"] += 1
                ingestion_data[db_path] = None

        # D-22 STEP 2: existing hard-red loop runs UNCHANGED in intent — but per-WAN
        # query failures are caught locally so they cannot abort the function.
        # (Codex H5 fix: the original behavior aborted on _query_digest_rows raising.)

            try:
                conn = sqlite3.connect(f"file:{db_path}?mode=ro", uri=True)
            except (sqlite3.OperationalError, OSError) as exc:
                print(
                    f"{_DIGEST_SKIP_PREFIX} wan={wan_name} db={db_path}: {exc}",
                    file=sys.stderr,
                )
                counts["read_skipped"] += 1
                continue

            counts["readable"] += 1
            try:
                try:
                    rows = _query_digest_rows(conn)
                except (sqlite3.OperationalError, sqlite3.DatabaseError) as exc:

                        file=sys.stderr,
                    )
                    counts["read_skipped"] += 1
                    continue
            finally:
                conn.close()
            line = _format_digest_line(wan_name, rows)

                print(line)
            except OSError as exc:
