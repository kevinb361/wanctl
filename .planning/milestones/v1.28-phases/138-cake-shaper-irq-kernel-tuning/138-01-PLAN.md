```bash
# Spectrum DL NIC (ens16) - heaviest single-NIC IRQ load
```

Remove the old `SPECTRUM_CPU=0` constant. Keep `SPECTRUM_NICS` array for use in the tune_nic loop (ring buffers + GRO still apply to both).

Replace the "Pin IRQ affinity" section at the bottom of main. Instead of looping all SPECTRUM_NICS with SPECTRUM_CPU, pin each Spectrum NIC individually:

```bash
# Pin IRQ affinity -- Spectrum DL (ens16) to CPU 0
```

Add sysctl tuning section AFTER the IRQ affinity section but BEFORE the final log line. Apply sysctls at runtime (belt-and-suspenders with sysctl.d):

```bash
# Kernel network stack tuning (per D-04, D-05, D-06)
# These complement the sysctl.d drop-in for immediate effect on service restart
apply_sysctl() {
    local key="$1"
    local value="$2"
    if sysctl -w "${key}=${value}" &>/dev/null; then
        logger -t "$TAG" "sysctl: ${key}=${value}"
    else
        logger -t "$TAG" "WARNING: sysctl ${key}=${value} failed"
    fi
}

apply_sysctl net.core.netdev_budget 600
apply_sysctl net.core.netdev_budget_usecs 4000
apply_sysctl net.core.netdev_max_backlog 10000
```

Sysctl values rationale:
- `netdev_budget=600`: 2x default (300). Bridge+CAKE doubles packet processing -- each packet traverses RX NIC, bridge, CAKE qdisc, TX NIC. 600 lets softirq process more packets per poll cycle without starving other softirqs.
- `netdev_budget_usecs=4000`: Half the default (8000). Tighter time bound prevents any single softirq from holding the CPU too long, improving latency. Combined with doubled budget, this means "process more packets but in shorter bursts."
- `netdev_max_backlog=10000`: Already at 10000 (10x kernel default 1000). Keep as-is -- confirmed adequate for 1 Gbps bridged CAKE.

Update the script header comment to mention 3-core IRQ distribution and sysctl tuning.

**B) Create deploy/sysctl/99-wanctl-network.conf** (per success criteria #3):

Create new directory `deploy/sysctl/` and file `99-wanctl-network.conf`:

```ini
# wanctl network stack tuning for bridge + CAKE workload
# Deployed to: /etc/sysctl.d/99-wanctl-network.conf
# Applied by:  sysctl --system (at boot) + wanctl-nic-tuning.sh (runtime)
#
# Tuned for cake-shaper VM: 4 NICs (2 bridges), 3 vCPUs, igb driver,
# 1 Gbps Spectrum + 40 Mbps ATT bridged through CAKE qdiscs.

# Double the default (300) -- bridge+CAKE means each packet is processed
# twice (ingress NIC + egress NIC). 600 keeps up without starving other
# softirqs.
net.core.netdev_budget = 600

# Halve the default (8000us) -- tighter time bound improves tail latency
# by preventing softirq from monopolizing a core. Combined with doubled
# budget: "more packets, shorter bursts."
net.core.netdev_budget_usecs = 4000

# 10x kernel default (1000). Already set via prior tuning. Ensures the
# per-CPU backlog queue doesn't drop packets during traffic bursts.
net.core.netdev_max_backlog = 10000
```
