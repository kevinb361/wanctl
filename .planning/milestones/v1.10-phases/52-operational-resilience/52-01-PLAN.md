    - Test: YAML with tab indentation error includes line number in error message
  </behavior>
  <action>
    1. In `BaseConfig.__init__`, wrap `yaml.safe_load(f)` in try/except yaml.YAMLError
    2. On yaml.YAMLError, check for `problem_mark` attribute which has `.line` and `.column`
    3. Re-raise as ConfigValidationError with message like: "YAML parse error at line {line+1}, column {col+1}: {error.problem}" (problem_mark.line is 0-indexed, so +1 for human-readable)
    4. If no problem_mark (rare), fall back to str(error)
    5. Import yaml.YAMLError at top of file (yaml already imported)

    Pattern:
    ```python
    try:
        self.data = yaml.safe_load(f)
    except yaml.YAMLError as e:
        if hasattr(e, 'problem_mark') and e.problem_mark is not None:
            mark = e.problem_mark
            raise ConfigValidationError(
                f"YAML parse error in {config_path} at line {mark.line + 1}, "
                f"column {mark.column + 1}: {e.problem}"
            ) from e
        raise ConfigValidationError(f"YAML parse error in {config_path}: {e}") from e
    ```
  </action>
  <verify>
    <automated>.venv/bin/pytest tests/test_config_base.py -x -v</automated>
  </verify>
