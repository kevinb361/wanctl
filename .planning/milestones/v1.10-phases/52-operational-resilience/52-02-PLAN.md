    - Test: MetricsWriter._get_connection on corrupted DB (write garbage bytes to file) detects corruption, renames to .corrupt, creates fresh DB
    - Test: After auto-rebuild, write_metric succeeds on fresh DB
    - Test: If integrity_check itself throws (locked DB, I/O error), log warning and proceed (don't crash daemon)

  <action>
    1. In `MetricsWriter._get_connection()`, after connecting and enabling WAL/PRAGMA but BEFORE `create_tables()`, run integrity check:

    ```python
    # Check database integrity at startup
    try:
        result = self._conn.execute("PRAGMA integrity_check").fetchone()
        if result[0] != "ok":
            logger.error(
                "Database integrity check failed: %s. Rebuilding database.",
                result[0],
            )
            self._conn.close()
            self._conn = None
            # Rename corrupt file for forensics
            corrupt_path = self._db_path.with_suffix(".db.corrupt")
            self._db_path.rename(corrupt_path)
            logger.warning("Corrupt database moved to %s", corrupt_path)
            # Reconnect to fresh database
            self._conn = sqlite3.connect(
                self._db_path,
                timeout=30.0,
                check_same_thread=False,
                isolation_level=None,
            )
            self._conn.execute("PRAGMA journal_mode=WAL")
            self._conn.execute("PRAGMA synchronous=NORMAL")
            self._conn.row_factory = sqlite3.Row
    except Exception as e:
        logger.warning("Integrity check error (proceeding anyway): %s", e)
    ```

    2. The integrity check runs only once per connection (singleton pattern means _get_connection only creates once).
    3. If the .corrupt file already exists from a previous corruption, overwrite it (or append timestamp). Use a simple approach: `corrupt_path = self._db_path.with_suffix(f".db.corrupt")` and if it exists, just overwrite -- one backup is sufficient for forensics.
    4. The key safety property: daemon NEVER crashes on corruption. It logs, rebuilds, and continues.

  <action>
    1. Create a shared helper function `_get_disk_space_status(path: str, threshold_bytes: int)` in `health_check.py`:

    ```python
    import shutil

    # Default: warn when less than 100MB free on data partition
    _DISK_SPACE_WARNING_BYTES = 100 * 1024 * 1024  # 100 MB

    def _get_disk_space_status(
        path: str = "/var/lib/wanctl",
        threshold_bytes: int = _DISK_SPACE_WARNING_BYTES,
    ) -> dict[str, Any]:
        """Check disk space for the given path.

        Returns dict with path, free_bytes, total_bytes, free_pct, status.
        Status is "ok", "warning", or "unknown" (if path inaccessible).
        """
        try:
            usage = shutil.disk_usage(path)
            free_pct = round((usage.free / usage.total) * 100, 1) if usage.total > 0 else 0.0
            status = "ok" if usage.free >= threshold_bytes else "warning"
            return {
                "path": path,
                "free_bytes": usage.free,
                "total_bytes": usage.total,
                "free_pct": free_pct,
                "status": status,
            }
        except OSError:
            return {
                "path": path,
                "free_bytes": 0,
                "total_bytes": 0,
                "free_pct": 0.0,
                "status": "unknown",
            }
    ```

    2. In `HealthCheckHandler._get_health_status()`, add `health["disk_space"] = _get_disk_space_status()` before the return. Place it after the router_reachable line and before the status determination.

    3. In `steering/health.py`, import the helper: `from wanctl.health_check import _get_disk_space_status` and add `health["disk_space"] = _get_disk_space_status()` in `SteeringHealthHandler._get_health_status()` at equivalent location.

       ```python
       is_healthy = self.consecutive_failures < 3 and all_routers_reachable and not disk_warning
       ```
       Same pattern in steering health.

    5. The threshold (100MB) is a module-level constant `_DISK_SPACE_WARNING_BYTES` for easy tuning. No config file change needed -- this is an operational constant, not user-facing config.
