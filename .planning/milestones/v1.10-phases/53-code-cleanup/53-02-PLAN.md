    **CLEAN-05 -- Scope disable_warnings to REST session:**

    In `src/wanctl/routeros_rest.py`:
    1. Remove the process-global line 50: `requests.packages.urllib3.disable_warnings(InsecureRequestWarning)`
    2. Remove the module-level `from urllib3.exceptions import InsecureRequestWarning` import (line 45) -- move it into the `__init__` method
    3. In `__init__`, after creating `self._session` (around line 105-107), add session-scoped warning suppression using `warnings` module:

    ```python
    import warnings
    # ... (add to module-level imports)

    # In __init__, after self._session.verify = verify_ssl:
    if not verify_ssl:
        import urllib3
        urllib3.disable_warnings(urllib3.exceptions.InsecureRequestWarning)
    ```

    Actually, the cleanest approach per the requirement: use `warnings.filterwarnings` scoped to the specific warning category, applied only when `verify_ssl=False`. This avoids the process-global call at import time.

    Better approach -- suppress only during requests by configuring the session's HTTPAdapter with a custom context, OR simply move the `disable_warnings` call into `__init__` conditional on `verify_ssl=False`:

    ```python
    # Remove line 50 entirely (the global disable_warnings call)
    # Keep the InsecureRequestWarning import at module level

    # In __init__, after self._session.verify = verify_ssl:
    if not verify_ssl:
        # Suppress InsecureRequestWarning only when SSL verification is disabled
        import urllib3
        urllib3.disable_warnings(InsecureRequestWarning)
    ```

    This scopes suppression to only instances that actually disable SSL (not all REST clients), and only runs when explicitly opted into. Not perfectly per-session isolated (urllib3 warnings are global by nature), but the suppression only fires when a client is configured with `verify_ssl=False`, which is the correct semantic.
