<objective>
Trace DSCP marks hop-by-hop through the MikroTik-to-CAKE bridge path to find exactly where they are lost or preserved. Produce a documented analysis with fix strategy for Phase 134.

Purpose: v1.26 testing showed CAKE diffserv4 tins not separating traffic. All packets land in Best Effort despite diffserv4 config. Research indicates CAKE reads DSCP from the IP header directly (not skb->priority), so bridges should not strip marks. The root cause is likely upstream -- either MikroTik has no mangle rules setting DSCP, or the DSCP marks exist but CAKE is misconfigured. This audit pinpoints the exact failure point.

Output: `.planning/phases/133-diffserv-bridge-audit/133-ANALYSIS.md` with hop-by-hop results table, failure point identification, and fix strategy recommendation.
</objective>
