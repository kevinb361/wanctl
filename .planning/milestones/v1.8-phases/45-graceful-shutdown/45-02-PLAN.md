# Plan 45-02: Shutdown Timeout Safety

## Goal
Ensure shutdown completes within a bounded time, preventing hung daemons from requiring SIGKILL.

## Truths (must be verifiable after implementation)

1. A shutdown timeout constant exists (e.g., SHUTDOWN_TIMEOUT_SECONDS = 30)
2. If any cleanup step hangs, the overall shutdown still completes within the timeout
3. Router close operations have individual timeouts (not unbounded)
4. Health server shutdown has explicit timeout (already exists: 5s thread join)
5. Tests verify that cleanup timeout is bounded
6. Shutdown timeout is less than systemd's default TimeoutStopSec (90s) to allow clean exit before SIGKILL

## Tasks

### Task 1: Add SHUTDOWN_TIMEOUT_SECONDS constant
- **File:** `src/wanctl/signal_utils.py`
- **Change:** Add `SHUTDOWN_TIMEOUT_SECONDS = 30` constant
- Rationale: 30s gives ample time for cleanup, well under systemd's 90s SIGKILL
- Export from signal_utils so both daemons can reference it

### Task 2: Add timeout to router close operations
- **Files:** `src/wanctl/routeros_ssh.py`, `src/wanctl/routeros_rest.py`
- **Change:** Add timeout parameter to close() methods
- SSH: paramiko close should be fast, but add 5s safety timeout via threading
- REST: requests session close is synchronous, add timeout wrapper
- Default timeout: 5 seconds per connection

### Task 3: Add bounded shutdown wrapper to autorate main
- **File:** `src/wanctl/autorate_continuous.py`
- **Change:** Wrap finally block cleanup in a timed sequence
- Track elapsed time during cleanup, log warning if approaching timeout
- If individual step exceeds its budget, log error and continue to next step
- Do NOT use threading.Timer (dangerous with signal handlers) — use monotonic clock checks

### Task 4: Add bounded shutdown wrapper to steering main
- **File:** `src/wanctl/steering/daemon.py`
- **Change:** Same pattern as Task 3 for steering daemon

### Task 5: Add tests for shutdown timeout behavior
- **Files:** `tests/test_signal_utils.py`, `tests/test_steering_daemon.py`
- Tests to add:
  - test_shutdown_timeout_constant_exists
  - test_shutdown_timeout_less_than_systemd_default
  - test_cleanup_continues_after_slow_router_close
  - test_cleanup_logs_warning_on_slow_step
