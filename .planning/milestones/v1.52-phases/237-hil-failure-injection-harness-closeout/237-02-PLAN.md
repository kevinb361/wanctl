<objective>
Build `scripts/silicom-test` — the HIL failure-injection orchestrator — and two seed chaos scenario files, turning the RED Plan-01 tests GREEN. The orchestrator composes the proven `silicom-bypass` verbs (NEVER `bpctl_util`), registers restore handlers BEFORE any mutation, and uses TWO distinct handlers: an EXIT handler that restores every touched pair while preserving the original rc, and an INT/TERM handler that restores then EXITS with a signal-derived code (130 for INT, 143 for TERM) so a Ctrl-C or kill can never let the script continue. It reuses the `phase213-*` capture scripts (deployed defaults pointing at absolute installed paths, overridable for tests) and writes structured results to `tests/silicom/<timestamp>-<scenario>/`.

Purpose: This is the safety-critical deliverable — the harness drives live-WAN Silicom bypass pairs. The handlers are the keeper mechanism; a pair must NEVER be left in bypass/disconnect after a crash, error, or Ctrl-C, and a signal must TERMINATE the run (not let a returning trap resume it).
Output: `scripts/silicom-test` (failover/ab-cake/chaos), `scripts/silicom-test-scenarios/*.sh`.
</objective>
