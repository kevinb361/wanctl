# Plan 251-01 — Route Ownership Decision + Read-Only Inventory

<objective>
Make WAN default-route ownership explicit before any wanctl route-management code exists: document Netwatch interim ownership, define the future single-owner contract, collect a read-only live RouterOS inventory, and produce Snapshot-A rollback evidence for later canary work.
</objective>

<must_haves>
<truths>
- Netwatch remains the interim active WAN route owner at Phase 251 start.
- Exactly one component may mutate WAN default routes at a time.
- Phase 251 must not mutate RouterOS routes, Netwatch entries, scripts, wanctl configs, services, CAKE qdiscs, or controller thresholds.
- Any active route mutation belongs to Phase 254 and requires explicit operator approval at that time.
</truths>
<outputs>
- `.planning/phases/251-route-ownership-decision-read-only-inventory/251-ROUTE-OWNERSHIP-DECISION.md`
- `.planning/phases/251-route-ownership-decision-read-only-inventory/evidence/routeros-route-ownership-inventory-<timestamp>.md`
- `.planning/phases/251-route-ownership-decision-read-only-inventory/evidence/snapshot-a-<timestamp>.json`
- `.planning/phases/251-route-ownership-decision-read-only-inventory/251-01-SUMMARY.md`
</outputs>
</must_haves>

<threat_model>
T-251-01 simultaneous-route-owners: If Phase 251 leaves ownership ambiguous, later phases may enable wanctl route mutation while Netwatch remains active. Mitigation: decision artifact must state single-owner contract, Netwatch interim owner, and future guard requirement.

T-251-02 accidental-router-mutation: Live inventory could accidentally run mutating RouterOS commands. Mitigation: command list must be written to `evidence/routeros-readonly-commands-<timestamp>.txt` and pass the automated whitelist in task `251-01-02` before any live command runs; allowed live commands are identity reads and explicit `print`/`export hide-sensitive` inspection only. No `enable`, `disable`, `set`, `add`, `remove`, or script `run` commands are allowed in Phase 251.

T-251-03 insufficient-rollback-anchor: A later canary could lack enough data to restore Netwatch ownership. Mitigation: Snapshot-A must capture Netwatch entries, route-mutating scripts, default routes, host identity, timestamps, and restore notes.

T-251-04 secret-leakage: RouterOS script/export output may contain sensitive values. Mitigation: prefer `export hide-sensitive`; redact secrets before writing evidence; if redaction is needed, include hashes/structure rather than values.
</threat_model>

Capture a timestamped read-only inventory in `routeros-route-ownership-inventory-<timestamp>.md` and a machine-readable Snapshot-A in `snapshot-a-<timestamp>.json`.

Before running any live command, the command file must pass this exact validation pattern:

- Skip blank lines and lines beginning with `#`.
- Permit only commands matching one of these regexes:
  - `^/system/identity/print$`
  - `^/tool/netwatch/print(\s+detail)?(\s+where\s+[^;]+)?$`
  - `^/system/script/print(\s+detail)?(\s+where\s+[^;]+)?$`
  - `^/system/script/export\s+hide-sensitive(\s+where\s+[^;]+)?$`
  - `^/ip/route/print(\s+detail)?(\s+where\s+[^;]+)?$`
- Reject every command containing any of these tokens as standalone words: `enable`, `disable`, `set`, `add`, `remove`, `run`, `import`, `reset`, `comment`, `policy`.
- Reject semicolons and shell metacharacters: `;`, `&&`, `||`, `>`, `<`, `` ` ``, `$(`.

If a needed inventory item cannot be collected by the allowlist, stop and record the gap in the evidence. Do not broaden the allowlist during execution without explicit operator approval.

Inventory must attempt to capture:
- Router identity and timestamp.
- Netwatch entries expected as `Monitor-Spectrum` and `Monitor-ATT`, plus broader Spectrum/ATT matches.
- Route-mutating scripts matching `Enable-*` and `Disable-*`, including policy, owner, last-start/run-count if available, and either source text with secrets redacted or source hash plus non-sensitive structure.
- Default routes with `dst-address="0.0.0.0/0"`, plus route comments/IDs/gateways/distances/disabled/active flags/routing-table/check-gateway.
- A mapping from each route-mutating script to the route(s) it appears to mutate.
- A current owner conclusion.
- The live command transcript must record issued commands with a `COMMAND:` prefix. RouterOS output must not use that prefix.

If current `RouterOSREST.run_cmd()` cannot read `/tool/netwatch`, `/system/script`, or `/ip/route`, existing operator SSH access may be used only after the command-file whitelist passes. This is evidence collection only, not an implementation precedent; record that REST route/Netwatch support remains Phase 252 implementation work.
</action>
