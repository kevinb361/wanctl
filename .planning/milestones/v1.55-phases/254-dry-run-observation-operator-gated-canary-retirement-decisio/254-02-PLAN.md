# Plan 254-02 — Operator-Gated One-WAN Active Canary / Rollback / Netwatch Retirement Decision

<objective>
Use the Wave 1 dry-run evidence and approval packet to either stop safely with Netwatch retained, or, only after explicit operator approval, run a bounded one-WAN active route-management canary with rollback proof and produce a final keep/rollback/retire route ownership decision.
</objective>

<must_haves>
<truths>
- This plan is not autonomous past the approval checkpoint.
- Active route mutation must not begin without explicit operator approval recorded during execution of this plan.
- If approval is declined, evidence is stale, rollback proof is incomplete, or dry-run observation is weak, the correct outcome is `keep-netwatch` and no mutation.
- Any approved canary must be one-WAN, bounded, observable, and rollback-ready before mutation.
- Netwatch is disabled/retired/converted to alert-only only after canary evidence and explicit acceptance support that decision.
</truths>
<outputs>
- Approval record or explicit declined/no-canary record.
- Canary runbook with exact scope, stop criteria, rollback commands/procedure, and observation commands.
- Canary observation artifact if approved and run.
- Final route ownership decision artifact choosing `keep-netwatch`, `keep-wanctl-for-approved-scope`, `retire-or-alert-only-netwatch`, or `rollback-to-netwatch`.
</outputs>
</must_haves>

<threat_model>
T-254-06 unapproved-mutation: executor starts active route mutation because dry-run looked good. Mitigation: `autonomous: false`, explicit `CHECKPOINT REQUIRED`, and approval record required before any mutation-shaped step.

T-254-07 rollback-not-ready: canary changes route ownership with no tested rollback path. Mitigation: canary preflight verifies Snapshot-A/drift status, rollback procedure, stop criteria, and operator abort path before approval.

T-254-08 canary-scope-creep: both WANs or Netwatch retirement happen at once. Mitigation: canary scope must name exactly one WAN/default-route scope; retirement/alert-only conversion is a final decision after evidence, not a precondition for canary.

T-254-09 stale-evidence-canary: approval uses old dry-run evidence after live state drift. Mitigation: require current Wave 1 observation artifact and block if drift/evidence gaps exist.

T-254-10 ambiguous-rollback: code/config/route ownership rollback are conflated. Mitigation: runbook and final decision split rollback into code, config, and route/Netwatch ownership layers.

T-254-11 SAFE-19-breach: production config, systemd, CAKE/qdisc, Netwatch, or routes are mutated outside the approved window. Mitigation: mutation commands are listed in the approval record, bounded to approved scope, and transcript/summary records exact actions.
</threat_model>
