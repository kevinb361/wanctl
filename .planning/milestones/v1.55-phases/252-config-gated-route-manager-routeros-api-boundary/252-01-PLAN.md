# Plan 252-01 — Route-Management Config Schema, Validation, and Dry-Run Surface

<objective>
Add an inert route-management configuration and dry-run helper for `steering.service`: safe/off by default, explicit route mappings, fail-closed validation for unsafe/malformed configs, and no live RouterOS route mutation in Phase 252.
</objective>

<must_haves>
<truths>
- Netwatch remains the interim WAN route owner.
- Phase 252 must not enable active route mutation.
- Missing `route_management` must behave as disabled/off.
- `route_management.mode: active` must fail closed by default in Phase 252.
- Dry-run/observe mode must not call RouterOS route mutation methods.
</truths>
<outputs>
- Steering config can parse and expose `route_management` defaults.
- Offline config validation accepts disabled/off and dry-run examples but rejects unsafe active/malformed route mappings.
- A small route-manager/helper seam emits intended actions for dry-run without mutating router state.
- Example/docs mention route management as disabled/dry-run only.
</outputs>
</must_haves>

<threat_model>
T-252-01 active-route-mutation-by-default: A missing or partial config accidentally enables route mutation. Mitigation: absent config loads as `enabled=false`, `mode="off"`; tests assert no route manager mutation calls in off mode.

T-252-02 unsafe-active-mode: An operator sets `mode: active` before Phase 253 guard/Phase 254 canary. Mitigation: config validation fails closed for active mode unless a deliberately named future migration escape exists; no example uses active mode.

T-252-03 dry-run-side-effect: Dry-run logs intended action but still calls route enable/disable. Mitigation: route-manager unit tests use a fake router client and assert no mutation method or `run_cmd()` route mutation command is called in dry-run.

T-252-04 config-drift-blindness: New config keys cause unknown-key warnings or malformed route maps are ignored. Mitigation: update `KNOWN_STEERING_PATHS` and add negative/positive validator tests.

T-252-05 SAFE-19-scope-creep: Executor touches live configs/services/router state while implementing config plumbing. Mitigation: only repo code/docs/tests/examples are modified; no live RouterOS or systemd commands are part of this plan.
</threat_model>
