    - src/wanctl/logging_utils.py (log rotation config)
    - Makefile (existing security target for pip-audit)
  </read_first>
  <action>
1. Install one-shot audit tools into the project venv:
   `.venv/bin/uv pip install vulture deptry pytest-deadfixtures radon`
   (Install vulture+radon now too so Plan 04 does not need to reinstall.)

2. Run pip-audit (FSCAN-01):
   `.venv/bin/pip-audit`
   Record full output. If critical/high CVEs found, document them. Goal is zero critical/high.

3. Run deptry (FSCAN-02):
   `.venv/bin/deptry src/`
   This checks for: unused deps (declared but never imported), missing deps (imported but undeclared), transitive deps used directly.
   Record full output. For each unused dep found:
   - Verify it is truly unused by checking all 8 CLI entry points in pyproject.toml [project.scripts]
   - Check if it is used transitively (e.g., cryptography via paramiko)
   - Remove confirmed unused deps from pyproject.toml
   NOTE: pyflakes is known redundant with ruff F rules (per STACK.md). Remove `"pyflakes>=3.4.0"` from [dependency-groups] dev list.
   After removal, run `uv sync` to update the lockfile.

4. Run pytest-deadfixtures (FSCAN-07):
   `.venv/bin/pytest --dead-fixtures tests/`
   Record all orphaned/duplicated fixtures. Do NOT remove them -- catalog only.

5. Verify log rotation (FSCAN-08):
   Grep for RotatingFileHandler or TimedRotatingFileHandler in src/wanctl/.
   Check: handler class, max bytes, backup count, log file paths.
   Also SSH to production VM to verify log directory sizes:
   `ssh kevin@10.10.110.223 'ls -lh /var/log/wanctl/ && du -sh /var/log/wanctl/'`

6. Write all findings to `.planning/phases/112-foundation-scan/112-01-findings.md` with sections:
   ```
   ## pip-audit (FSCAN-01)
   [Full output, disposition of any findings]

   ## deptry (FSCAN-02)
   [Full output, what was removed, what was kept and why]

   ## pytest-deadfixtures (FSCAN-07)
   [Full output, catalog of orphaned fixtures]

   ## Log Rotation (FSCAN-08)
   [Handler config, retention settings, production log sizes]
   ```
  </action>
