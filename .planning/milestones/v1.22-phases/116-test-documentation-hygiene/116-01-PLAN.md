Scan all test files in tests/ for three categories of test quality issues:

**1. Assertion-Free Tests (per D-01):**
Run: `.venv/bin/pytest tests/ --collect-only -q 2>/dev/null | grep "::" | cut -d: -f1 | sort -u` to get all test files.
For each test file, search for test functions that have NO assert statement, NO pytest.raises, NO pytest.warns, NO mock.assert_called, NO mock.assert_called_with, NO mock.assert_called_once, NO mock.assert_not_called. A test function that only calls code without verifying the result is assertion-free.

Use grep-based heuristic: extract test function bodies and check for assertion patterns. Focus on test functions where the body contains only setup + function call with no verification.

**2. Tautological Tests (per D-01):**
Search for tests that assert on their own setup rather than production code behavior:
- `assert mock_obj.return_value == mock_obj.return_value` patterns
- Tests that only assert on hardcoded values matching hardcoded expectations
- Tests where the assertion is on a local variable set 1-2 lines above with the same literal

**3. Over-Mocked Tests (document only per D-02):**
Search for tests where >3 mocks patch out core production behavior (e.g., mocking both the function under test AND its dependencies). Flag tests where `@patch` decorators exceed 4 per function or where the function under test is itself mocked.

**Input from Phase 112 (per D-03):** Incorporate the 8 orphaned fixtures from 112-01-findings.md into the report under a "Previously Identified" section.

From the test quality audit catalog (Task 1 output), fix all tests marked with Action="fix":

**Assertion-Free Test Fixes (per D-02 -- highest risk, false confidence):**
For each assertion-free test, add meaningful assertions that verify the actual behavior of the code under test. Do NOT just add `assert True` -- add assertions that would fail if the production code broke. Examples:
- If test calls a function but doesn't check return value: add `assert result == expected`
- If test calls a method for side effects: add `mock.assert_called_once_with(expected_args)`
- If test exercises error path: add `with pytest.raises(ExpectedError):`

**Tautological Test Fixes (per D-02 -- highest risk, false confidence):**
For each tautological test, replace the self-referential assertion with one that tests actual production behavior. The test should fail if the production code changes unexpectedly.

**Do NOT fix over-mocked tests** -- per D-02 these are lower risk and document-only.

After fixes, run the full test suite to confirm no regressions: `.venv/bin/pytest tests/ -x -q`

Update the audit catalog to mark fixed items as "FIXED" in the Action column.
