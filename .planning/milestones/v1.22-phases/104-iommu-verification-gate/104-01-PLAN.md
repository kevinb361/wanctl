**Section 3 -- VFIO Readiness:**
Two tables. "Already Working" table:

"Phase 109 Must Configure" table:
- VFIO modules: Add `vfio vfio_iommu_type1 vfio_pci` to /etc/modules, run update-initramfs -u -k all
- vfio-pci binding: Add device IDs to /etc/modprobe.d/vfio.conf: `options vfio-pci ids=8086:1533,8086:1521` with `softdep igb pre: vfio-pci`
- Kernel cmdline: Add `iommu=pt` to /etc/kernel/cmdline (optional, DMA-FQ already active)
- VM passthrough: `qm set <vmid> --hostpciN <addr>` for each NIC

**Section 4 -- Kernel Warnings:**
- VFIO regression on kernels 6.17.13-1-pve and 6.17.13-2-pve (DMAR faults after VMs run with VFIO passthrough). Odin is on 6.17.2-1-pve (safe, pre-regression). Do not upgrade past 6.17.9-1-pve until fix confirmed. Consider pinning: `apt-mark hold proxmox-kernel-6.17.2-1-pve`.
- IOMMU group numbering may change after kernel upgrades. Re-verify before starting VM after any kernel update.
