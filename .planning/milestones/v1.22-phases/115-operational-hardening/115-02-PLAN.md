Write a backup/recovery runbook covering all items per D-09 and D-10. This is documentation only -- no automated backup implementation.

**Structure the runbook with these sections:**

1. **Backup Inventory** -- what to back up and where it lives:
   - `/etc/wanctl/*.yaml` (spectrum.yaml, att.yaml, steering.yaml)
   - `/etc/wanctl/secrets` (router password, webhook URLs)
   - `/var/lib/wanctl/metrics.db` (historical performance data, SQLite)
   - `/etc/systemd/system/wanctl@.service` (template unit)
   - `/etc/systemd/system/steering.service`
   - `/etc/systemd/system/wanctl-nic-tuning.service`
   - `/opt/wanctl/` (application code, deployed from git)

2. **Quick Backup Commands** -- single-command backup to a tarball:
   ```
   ssh kevin@10.10.110.223 "sudo tar czf /tmp/wanctl-backup-$(date +%Y%m%d).tar.gz /etc/wanctl/ /var/lib/wanctl/metrics.db /etc/systemd/system/wanctl@.service /etc/systemd/system/steering.service /etc/systemd/system/wanctl-nic-tuning.service"
   scp kevin@10.10.110.223:/tmp/wanctl-backup-*.tar.gz ~/backups/
   ```

3. **Proxmox VM Snapshot** -- how to snapshot VM 206:
   - Via web UI: Datacenter > odin > VM 206 > Snapshots > Take Snapshot
   - Via CLI (from Proxmox host): `qm snapshot 206 <name> --description "<reason>"`
   - When to snapshot: before any systemd/config/code changes, before version upgrades

4. **Recovery Procedures:**
   - **Config restore:** Untar backup, copy to /etc/wanctl/, verify permissions (0640 root:wanctl)
   - **Metrics DB restore:** Stop services, copy metrics.db to /var/lib/wanctl/, set perms (0644 wanctl:wanctl), start services
   - **Service unit restore:** Copy unit files, daemon-reload, restart services
   - **Code restore:** `cd /opt/wanctl && sudo git checkout v1.XX` (deployed from git tags)
   - **Full VM rollback:** Restore Proxmox snapshot (loses all changes since snapshot)
   - **From scratch rebuild:** Create VM, install Debian 12, create wanctl user, deploy code, copy configs, enable services

5. **Rollback from Phase 115 changes specifically (per D-14):**
   - systemd units: `sudo cp *.bak.pre115 <original-name> && sudo systemctl daemon-reload && sudo systemctl restart ...`
   - NIC tuning: revert wanctl-nic-tuning.service from .bak.pre115
   - Resource limits: remove MemoryMax/TasksMax/LimitNOFILE lines from units
   - Nuclear option: revert Proxmox snapshot taken in Plan 01

6. **Verification after recovery:**
   - `systemctl is-active wanctl@spectrum wanctl@att steering`
   - `curl -s http://127.0.0.1:9101/health`
   - `journalctl -u wanctl@spectrum --since '2 min ago'` (check for errors)
   - `ethtool -k ens16 | grep rx-udp-gro-forwarding` (check NIC tuning)
