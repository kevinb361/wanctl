Per D-11, D-12, D-13:

1. Per D-11, document the full signal chain with data flow:
   ```
   Raw ICMP RTT (icmplib, median-of-3)
     -> SignalProcessor.process() [Hampel outlier filter]
       -> filtered_rtt (outliers replaced with window median)
     -> Fusion (if enabled): weighted average of ICMP + IRTT
       -> fused_rtt = w_icmp * icmp_rtt + w_irtt * irtt_rtt
     -> update_ewma(): load_rtt = alpha * sample + (1-alpha) * load_rtt
     -> BaselineRTTManager: baseline updates only when delta < threshold (idle condition)
     -> delta = load_rtt - baseline_rtt (drives state machine)
   ```
   Read autorate_continuous.py to trace exactly where each step happens in the control loop.
   Document: which WANController methods call which, in what order, per cycle.

2. Per D-12, document IRTT vs ICMP measurement paths:
   - ICMP path: icmplib.ping() -> RTTMeasurement -> median_of_three -> SignalProcessor
     * Correctness: Sub-millisecond precision, kernel ICMP stack, CAP_NET_RAW
     * Limitation: ICMP can be deprioritized by ISP, Spectrum has blocked it before (v1.1.0)
   - IRTT path: irtt binary subprocess -> IRTTResult -> IRTTThread (background, cached)
     * Correctness: Isochronous timing, OWD measurement, server-side timestamps
     * Limitation: Requires dedicated IRTT server (Dallas 104.200.21.31:2112), single point
   - TCP fallback path: socket.connect_ex() timing to port 443
     * Correctness: Works when ICMP blocked, but higher variance from TLS handshake
   - Document when each path is used and why
