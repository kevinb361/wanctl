<objective>
Add sustained IRTT packet loss alerting to the autorate daemon. When upstream or downstream loss exceeds 5% for 60+ seconds, fire a Discord alert via the existing AlertEngine. Fire a recovery alert when loss clears. Respect per-event cooldown to prevent alert storms.

Purpose: Operators currently see IRTT loss in logs and SQLite but get no proactive notification. This adds the missing alerting layer on top of existing loss data.
Output: Modified autorate_continuous.py with _check_irtt_loss_alerts(), modified webhook_delivery.py with loss unit mapping, new test file.
</objective>
